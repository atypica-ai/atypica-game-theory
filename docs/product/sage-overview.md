# Sage (专家智能体) 产品文档

## 产品定位

**Sage** 是 atypica.AI 推出的可进化 AI 专家系统，通过知识管理、持续学习和主动补全机制，帮助用户构建具备深度领域专长的 AI 顾问。

### 核心价值主张

- **记忆即专家**: 基于结构化的知识文档（Memory Document）构建专家能力
- **持续进化**: 通过对话发现知识盲点，主动补充和更新知识库
- **可追溯性**: 完整的知识来源追踪和版本历史管理
- **成本可控**: 手动触发处理流程，精细控制 AI 调用成本

## 核心功能

### 1. 专家创建与知识导入

用户通过多种方式为 AI 专家注入领域知识：

- **文件上传**: 支持 PDF、TXT、Markdown、Audio、DOCX 等格式
- **文本粘贴**: 直接输入文本内容
- **URL 导入**: 从网页抓取内容（使用 Jina Reader API）

每个专家最多支持 10 个知识源，系统会自动解析和提取文本内容。

### 2. 三步处理流程

创建专家后，需要手动触发以下三个独立步骤：

#### Step 1: 解析知识源
- 处理上传的文件、文本和 URL
- 提取纯文本内容（extractedText）
- 每个知识源独立处理，失败不影响其他源

#### Step 2: 提取知识并构建记忆文档
- 使用 Claude Sonnet 4.5 分析所有知识源
- 生成专家简介（bio）、专长领域（expertise）和推荐问题
- 构建结构化的 Markdown 记忆文档（Memory Document）
- 创建版本 1 的知识库

#### Step 3: 知识分析
- 使用 GPT-4o 分析记忆文档
- 识别知识空白（Knowledge Gaps）
- 标注严重程度（critical / important / nice-to-have）
- 说明影响和建议补充方向

### 3. 知识空白（Knowledge Gap）管理

系统通过三种方式识别专家的知识盲点：

#### 来源 1: 初始知识分析
- 在 Step 3 中由 AI 主动分析记忆文档
- 系统性地评估知识完整性
- 标记为 `source.type: "analysis"`

#### 来源 2: 对话中发现（核心创新）
- 用户与专家对话后，AI 异步分析回答质量
- 识别专家回答薄弱或无法准确回应的领域
- 自动创建 Knowledge Gap，关联原始对话
- 在 Gap 列表中显示用户提问引用，可跳转查看完整对话
- 标记为 `source.type: "conversation"`

#### 来源 3: 系统建议（预留）
- 系统定期分析或管理员手动添加
- 标记为 `source.type: "system_suggestion"`

### 4. 补充访谈（Supplementary Interview）

当存在未解决的 Knowledge Gaps 时，用户可以创建补充访谈：

**自动流程**:
1. 系统基于所有 pending gaps 使用 Claude Sonnet 4 生成访谈计划
2. 创建包含预设问题和追问方向的结构化访谈
3. AI 访谈员与用户（扮演专家）进行深度对话
4. AI 判断何时结束访谈（调用 `endInterview` tool）

**访谈结束后的自动更新**:
- AI 分析访谈内容，更新记忆文档（创建新版本）
- 自动识别哪些 Knowledge Gaps 已被解决
- 标记解决状态，关联到具体访谈记录
- 完整的知识演进追踪

### 5. 专家咨询对话

#### 公开专家主页
- 展示专家简介、专长领域
- 显示 AI 生成的推荐问题列表
- 用户可直接开始对话或点击推荐问题快速提问

#### 对话功能
- 基于最新记忆文档的精准回答
- 支持文件附件上传（作为对话上下文）
- 对话后自动分析，可能产生新的 Knowledge Gap
- 私有对话仅专家所有者可见

### 6. 版本控制与知识追溯

#### Memory Document 版本管理
- 每次知识更新都会创建新版本（version++）
- 保留最近 20 个版本的完整历史
- 记录变更来源：
  - `initial`: 首次创建
  - `interview`: 补充访谈更新
  - `manual`: 手动编辑

#### Knowledge Gap 追溯
- 每个 Gap 记录创建来源（分析/对话/系统建议）
- Conversation 类型显示用户提问引用，可跳转查看
- 解决后记录解决方式（访谈/手动）和关联记录

## 技术特性

### AI 模型策略

不同任务使用针对性的 AI 模型，平衡质量与成本：

| 任务 | 模型 | 原因 |
|------|------|------|
| 专家资料生成 | Claude Sonnet 4.5 | 需要高质量的文本生成 |
| 记忆文档构建 | Claude Sonnet 4.5 | 结构化知识整理 |
| 知识空白分析 | GPT-4o | 快速识别知识盲点 |
| 专家对话 | Claude Sonnet 4.5 | 高质量的交互体验 |
| 补充访谈 | Claude Sonnet 4.5 | 深度对话能力 |
| 对话质量分析 | Gemini 2.5 Flash | 低成本异步分析 |
| 访谈计划生成 | Claude Sonnet 4 | 结构化计划生成 |

### 成本优化设计

- **手动触发机制**: 三步处理流程均需用户手动触发，避免意外消耗
- **异步处理**: 对话质量分析、访谈后更新等使用后台异步处理
- **轻量模型**: 非关键任务使用成本更低的模型（如 Gemini 2.5 Flash）
- **增量更新**: 支持新增知识源后的增量处理

### 数据安全与隐私

- **私有知识库**: 每个专家的知识库完全私有
- **访问控制**: 仅专家所有者可管理和查看详细信息
- **公开控制**: 用户可选择是否公开专家主页（预留功能）

## 用户界面

### 专家管理界面

采用左右分栏布局，提供清晰的管理体验：

**左侧面板 (1/3 宽度)**:
- 专家头像和基本信息
- 知识源列表（显示处理状态）
- 批量处理和添加源按钮

**右侧面板 (2/3 宽度)**:
- Tab 导航: Memory / Gaps / Chats
- Memory Tab: 显示记忆文档，提供提取和分析按钮
- Gaps Tab: 展示知识空白列表，创建补充访谈
- Chats Tab: 查看所有用户对话记录

### 公开专家主页

- 简洁的专家介绍卡片
- 专长领域标签展示
- AI 生成的推荐问题列表
- 一键开始对话按钮

## 典型使用场景

### 场景 1: 企业内部专家知识传承

**背景**: 某企业的资深 UX 设计师即将退休，需要将多年经验传承下来。

**操作流程**:
1. 创建 "UX Design Expert" Sage
2. 上传设计师的项目文档、设计规范、演讲 PPT 等
3. 处理知识源，构建初始记忆文档
4. 分析知识空白，发现缺少"最新 AI 工具使用经验"
5. 创建补充访谈，让设计师回答 AI 辅助设计相关问题
6. 新员工可随时与该 AI 专家对话，获得设计指导

**价值**: 将隐性知识显性化，永久保存专家经验。

### 场景 2: 个人学习助手

**背景**: 学生正在学习机器学习课程，有大量课件和笔记。

**操作流程**:
1. 创建 "Machine Learning Tutor" Sage
2. 上传课程 PDF、笔记 Markdown、相关论文
3. 处理并构建知识库
4. 学习过程中随时向专家提问
5. AI 发现学生反复问某些问题，自动识别知识空白
6. 学生可创建补充访谈，补充相关知识点

**价值**: 将静态学习资料转化为可交互的智能导师。

### 场景 3: 行业研究专家

**背景**: 分析师需要跟踪特定行业的最新动态和趋势。

**操作流程**:
1. 创建 "FinTech Industry Expert" Sages
2. 添加行业报告 PDF、公司公告 URL、研究论文
3. 定期补充新的行业资讯（增量添加知识源）
4. 通过对话快速获取行业洞察
5. 系统发现知识盲点，提醒补充特定领域研究

**价值**: 构建持续更新的行业知识库，提升研究效率。

## 与 NotebookLM 的对比

| 维度 | Sage | Google NotebookLM |
|------|------|-------------------|
| **核心定位** | 可进化的 AI 专家代理系统 | AI 研究助手和知识管理工具 |
| **核心价值** | 构建持续学习的专业 AI 顾问 | 快速理解和利用已有文档 |
| **知识管理** | 版本化记忆文档 + 主动学习 | 基于上传内容的静态分析 |
| **学习机制** | 主动识别知识盲点并通过访谈补充 | 被动响应用户查询 |
| **独特功能** | Knowledge Gap 追踪、补充访谈、知识演进 | Audio Overview（文档转播客）、Deep Research |
| **适用场景** | 需要深度专业咨询和知识传承 | 快速消化大量文档、学习研究 |
| **成本控制** | 手动触发处理，精细控制 | Google 托管，使用门槛低 |
| **源追溯** | 详细的知识变更历史和 Gap 来源 | 引用来源文档位置 |
| **知识更新** | 版本控制，可追踪每次变化 | 重新上传文档后覆盖 |
| **交互模式** | 构建专家 → 识别缺口 → 补充知识 → 提供咨询 | 上传内容 → 即时使用 → 生成不同输出 |

### 核心差异

**Sage 强调"培养专家"**:
- 知识是动态演进的
- 系统主动发现知识盲点
- 通过结构化访谈补全知识
- 完整的知识演进追踪

**NotebookLM 强调"理解材料"**:
- 知识是静态的（基于上传文档）
- 被动响应用户查询
- 创新的输出形式（播客、视频总结）
- 快速上手，使用门槛低

## 技术架构

### 数据模型

#### 核心实体

- **Sage**: 专家基本信息（名称、领域、专长、头像）
- **SageSource**: 知识源（文件/文本/URL）
- **SageMemoryDocument**: 版本化的记忆文档
- **SageKnowledgeGap**: 独立的知识空白记录
- **SageChat**: 用户与专家的对话会话
- **SageInterview**: 补充访谈记录

#### 数据关系

```
Sage (1) ──< (N) SageSource
     (1) ──< (N) SageMemoryDocument [版本历史]
     (1) ──< (N) SageKnowledgeGap
     (1) ──< (N) SageChat
     (1) ──< (N) SageInterview

SageKnowledgeGap ──> SageChat [可选关联，conversation 类型]
SageKnowledgeGap ──> SageInterview [可选关联，解决方式]
SageInterview ──> SageMemoryDocument [创建新版本]
```

### 关键技术点

#### 1. 独立 Knowledge Gap 表设计

采用独立数据表而非 JSON 字段存储，支持：
- 高效的状态查询（pending / resolved）
- 灵活的来源追踪（analysis / conversation / system_suggestion）
- 详细的解决记录（interview / manual）
- 索引优化查询性能

#### 2. 版本控制机制

- 自增版本号（乐观锁保证串行）
- 保留最新 20 个版本
- 每个版本记录变更来源和说明
- 联合索引优化查询：`(sageId, version DESC)`

#### 3. Jina Reader API 集成

- 支持 PDF、DOCX 等多种文件格式解析
- 支持网页内容抓取和清洗
- 为上传文件生成 S3 签名 URL 后传递给 Jina
- 失败重试机制

#### 4. After Pattern 异步分析

- 使用 Next.js 的 `after()` API
- 对话结束后异步分析知识空白
- 不阻塞主流程，提升响应速度
- 失败不影响用户体验

#### 5. Tool-based Interview 控制

- AI 通过调用 `endInterview` tool 结束访谈
- Tool 触发后台更新流程（waitUntil）
- 自动更新记忆文档和解决相关 Gaps
- 完整的事件驱动处理链

## 最佳实践

### 创建高质量专家的建议

1. **提供多样化的知识源**
   - 结合结构化文档（PDF、DOCX）和非结构化文本
   - 包含不同深度的内容（概览 + 详细案例）
   - 适当包含实际对话记录或 Q&A

2. **充分利用补充访谈**
   - 及时创建访谈填补知识空白
   - 在访谈中提供具体案例和场景
   - 回答问题时尽量详细和结构化

3. **持续观察对话质量**
   - 定期查看 Gaps Tab，了解用户提问趋势
   - 关注 conversation 类型的 Gap，反映真实需求
   - 主动补充用户常问但专家回答不佳的领域

4. **合理使用推荐问题**
   - 在公开主页展示最能体现专家价值的问题
   - 引导用户提出高质量问题
   - 根据对话反馈优化推荐问题列表

### 成本优化建议

1. **分批处理知识源**
   - 先处理核心文档，观察效果
   - 逐步添加补充材料
   - 避免一次性上传大量低质量内容

2. **选择性创建访谈**
   - 优先解决 critical 和 important 级别的 Gap
   - 合并相关领域的 Gap 在一次访谈中解决
   - Nice-to-have 的 Gap 可以暂缓处理

3. **充分利用对话分析**
   - 对话分析使用低成本模型（Gemini 2.5 Flash）
   - 通过真实对话发现的 Gap 更有针对性
   - 比主动分析更贴近实际使用需求

## 总结

Sage 是一个创新的 AI 专家系统，通过**记忆文档、知识空白追踪、补充访谈**三大机制，实现了 AI 专家的持续进化。与传统的静态知识库工具不同，Sage 能够主动发现知识盲点并通过结构化访谈补充知识，真正做到"越用越聪明"。

无论是企业知识传承、个人学习助手，还是行业研究专家，Sage 都能帮助用户构建一个专业、可靠、不断进化的 AI 顾问。
