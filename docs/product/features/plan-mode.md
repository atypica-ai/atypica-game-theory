# Plan Mode vs 直接执行：什么时候用哪个？

> **文档类型**：功能对比
> **更新日期**：2026-01-15
> **适用人群**：所有用户（特别是新手和不确定研究方向的用户）

---

## 一句话差异

Plan Mode 是意图澄清层，通过对话帮你明确"做什么研究"后展示完整计划；直接执行是跳过澄清，直接开始研究（适合明确需求的用户）。

---

## 核心差异

| 维度 | Plan Mode（意图澄清） | 直接执行（立即开始） |
|------|---------------------|-------------------|
| **适用场景** | 需求模糊、不确定方向、需要建议 | 需求明确、知道怎么做、直接开始 |
| **交互方式** | 多轮对话澄清（灵活轮数） | 一次性输入，直接执行 |
| **AI 角色** | 顾问（帮你想清楚） | 执行者（按你说的做） |
| **输出** | 完整研究计划 → 用户确认 → 执行 | 直接执行 → 边做边展示 |
| **用户控制** | 确认后才开始（可修改计划） | 开始后无法修改计划 |
| **自动判断** | AI 自动判断研究类型和方法 | 用户自己指定或 AI 执行中判断 |
| **背景调研** | 可以使用 webSearch 了解背景 | 执行中也可以搜索，但不会提前调研 |
| **时长** | 2-5 分钟澄清 + 研究执行时间 | 直接执行（节省澄清时间） |
| **适合人群** | 新手、探索阶段、不确定的用户 | 熟练用户、明确需求、重复性研究 |
| **步数限制** | 最多 15 步（澄清过程） | 根据研究类型不同（10-20 步） |

## 使用建议

### ✅ 用 Plan Mode 的场景

#### 1. 需求模糊，不确定具体要研究什么
- **例子**："我想了解年轻人对咖啡的看法"（太宽泛）
- **Plan Mode 帮你**：
  - 澄清"年轻人"的年龄段（18-22？23-28？）
  - 确认"看法"的维度（品牌偏好？价格敏感度？使用场景？）
  - 建议合适的研究方法（Interview？Discussion？）
- **结果**：明确的研究计划 → "18-28 岁一线城市年轻人的日常咖啡消费决策研究（重点关注品牌偏好和价格敏感度），使用 Interview 访谈 8 个 AI 人设"

#### 2. 第一次使用 atypica.AI，不熟悉功能
- **例子**："我想做用户研究，但不知道怎么开始"
- **Plan Mode 帮你**：
  - 通过对话了解你的研究目标
  - 自动判断适合的研究类型（testing/insights/creation 等）
  - 推荐合适的方法和人设数量
- **结果**：新手友好，像有个顾问在旁边指导

#### 3. 不确定应该用哪种研究方法
- **例子**："我想了解用户对新产品的反应，用 Interview 还是 Discussion？"
- **Plan Mode 帮你**：
  - 根据你的研究目标自动判断（需要深度动机 → Interview，需要意见分布 → Discussion）
  - 展示判断逻辑，帮你理解为什么
- **结果**：学习到方法论，下次可以自己判断

#### 4. 需要背景调研后才能确定研究方向
- **例子**："我想研究电动汽车市场，但不太了解最新趋势"
- **Plan Mode 帮你**：
  - 使用 webSearch 快速了解行业动态
  - 根据调研结果建议研究方向
  - 在计划中融入最新背景信息
- **结果**：研究计划基于最新市场动态，更有针对性

#### 5. 想看到完整研究计划，确认后再执行
- **例子**："我想先看看会做什么，确认后再开始"
- **Plan Mode 帮你**：
  - 展示完整的研究意图、方法、预期产出
  - 用户可以修改或取消
  - 确认后才开始执行，避免浪费
- **结果**：心里有数，明确预期

### ✅ 直接执行的场景

#### 1. 需求非常明确，知道具体要做什么
- **例子**："请访谈 10 个 25-30 岁的咖啡爱好者，了解他们选择咖啡品牌的决策因素"
- **为什么直接执行**：需求已经很清晰（目标人群、人数、研究问题），不需要澄清
- **结果**：节省澄清时间，直接进入执行

#### 2. 熟练用户，多次使用过类似研究
- **例子**：每周都做竞品对比研究的产品经理
- **为什么直接执行**：已经知道怎么做，重复性研究
- **结果**：提高效率，快速完成

#### 3. 需要快速验证，不想花时间澄清
- **例子**："快速测试一下用户对这 3 个文案的反应"
- **为什么直接执行**：时间紧迫，直接开始
- **结果**：最快速度获得结果

#### 4. 已经在其他地方做好了研究计划
- **例子**：团队已经讨论好了研究方案，只需要执行
- **为什么直接执行**：计划已定，不需要 AI 建议
- **结果**：按照既定方案执行

#### 5. 简单的单一任务
- **例子**："生成一个关于 AI 技术趋势的播客"
- **为什么直接执行**：需求单一，不需要复杂规划
- **结果**：快速完成简单任务

## 真实案例：气泡咖啡市场研究

### 场景
某饮品公司产品经理想了解消费者对气泡咖啡的态度，但对研究方法不熟悉。

### 方案 A：使用 Plan Mode

#### 初始输入：
"我想了解用户对气泡咖啡的看法"

#### Plan Mode 澄清过程（3 轮对话）：

**Round 1**：
- **Plan Mode 问**："你说的用户，具体是哪个人群？"
- **用户答**："年轻人吧，喜欢尝试新东西的"

**Round 2**：
- **Plan Mode 问**："年轻人的年龄段大概是？"
- **用户答**："18-28 岁，一线城市的"

**Round 3**：
- **Plan Mode 问**："你主要想了解哪些方面？"
- **用户答**："他们为什么会买，能接受什么价格"

#### Plan Mode 输出完整计划：

```markdown
# 研究计划确认

## 📋 研究意图
**研究对象**: 18-28 岁一线城市年轻消费者（喜欢尝试新产品）
**研究场景**: 日常咖啡消费决策，特别是首次购买气泡咖啡的场景
**关注维度**: 购买动机、价格敏感度、决策因素

## 🔬 研究方法
**分析框架**: JTBD（Jobs to Be Done）- 理解用户"雇佣"气泡咖啡的原因
**研究方式**:
- Interview（一对一深度访谈）：挖掘个体购买动机和决策路径
- Discussion（群体讨论）：验证价格接受度的意见分布
**人设配置**:
- Interview: 10 个 AI 人设（深访人设）
- Discussion: 8 个 AI 人设（社媒人设）

## 📊 预期产出
- 购买动机图谱（核心驱动因素）
- 价格敏感度分析（可接受的溢价范围）
- 用户决策路径（从认知到购买）
- 产品定位建议

---

是否开始执行？
```

#### 用户确认后执行：
- 时间：澄清 3 分钟 + 执行 28 分钟 = **总共 31 分钟**
- 成本：约 100 元
- 结果：完整的研究报告

### 方案 B：直接执行

#### 初始输入（需要用户自己明确所有细节）：
"请对 18-28 岁一线城市年轻消费者进行气泡咖啡购买决策研究。使用 JTBD 框架，通过 Interview（10 个深访人设）挖掘购买动机和价格敏感度，然后通过 Discussion（8 个社媒人设）验证价格接受度分布。重点关注首次购买场景，输出购买动机图谱、价格敏感度分析、决策路径和产品定位建议。"

#### 直接执行：
- 时间：28 分钟（无澄清）
- 成本：约 100 元
- 结果：完整的研究报告

### 对比分析

| 维度 | Plan Mode | 直接执行 |
|------|----------|---------|
| **初始输入** | "我想了解用户对气泡咖啡的看法"（模糊） | 需要完整详细的研究方案（140+ 字） |
| **澄清过程** | 3 轮对话，AI 自动判断方法 | 无澄清，用户自己决定一切 |
| **总时间** | 31 分钟（3 澄清 + 28 执行） | 28 分钟（直接执行） |
| **用户体验** | 新手友好，有指导 | 需要专业知识 |
| **计划可见** | 完整计划，确认后执行 | 执行中才知道具体做什么 |
| **适合人群** | 不熟悉方法论的用户 | 专业研究人员 |

### 实际选择建议

**第一次研究**：用 Plan Mode
- 学习 atypica.AI 的研究方法论
- 理解如何设计研究
- 建立信任（看到计划再确认）

**第二次及以后**：可以直接执行
- 已经熟悉流程和方法
- 可以参考第一次的研究设计
- 提高效率

## 技术实现差异

### Plan Mode
- **代码位置**：`src/app/(study)/agents/configs/planModeAgentConfig.ts`
- **系统提示词**：`src/app/(study)/prompt/planMode.ts`（524 行）
- **核心工具**：
  - `requestInteraction`: 对话澄清
  - `makeStudyPlan`: 提交完整研究计划
  - `webSearch/webFetch`: 背景调研
  - `reasoningThinking`: 深度分析
- **步数限制**：最多 15 步（planMode.ts:15）
- **决策逻辑**：三层决策树
  - **第一层**：Agent 选择（fastInsight / productRnD / Study）
  - **第二层**：Study 子类型（testing / insights / creation / planning / misc）
  - **第三层**：研究细节（框架、方法、人设）

### 直接执行（Study Agent / Fast Insight Agent / Product R&D Agent）
- **代码位置**：`src/app/(study)/agents/configs/studyAgentConfig.ts` 等
- **系统提示词**：`src/app/(study)/prompt/study.ts`（660 行）
- **核心工具**：
  - `planStudy/planPodcast`: 内层战术规划
  - `interviewChat/discussionChat`: 研究执行
  - `generateReport`: 报告生成
  - 16+ 专业工具
- **步数限制**：
  - Study Agent: 20 步
  - Fast Insight Agent: 10 步
  - Product R&D Agent: 15 步
- **执行逻辑**：
  - 5 阶段工作流（主题澄清 → 准备 → 执行 → 报告 → 完成）
  - Messages as Source of Truth（消息驱动）

### 架构关系

```
用户输入
    ↓
[Plan Mode: 意图澄清层]
    ├→ 澄清需求（多轮对话）
    ├→ 背景调研（webSearch）
    ├→ 自动判断（kind + method）
    └→ 展示计划（makeStudyPlan）
         ↓
    [用户确认]
         ↓
    [执行层: Study/FastInsight/ProductRnD Agent]
         ├→ 内层规划（planStudy）
         ├→ 执行研究（interview/discussion/scout）
         └→ 生成报告（generateReport）
              ↓
         [研究完成]
```

### 为什么这样设计？

**推理-执行分离**：
1. **Plan Mode**：战略层，决定"做什么研究"
   - 灵活对话，没有固定流程
   - 自动判断最合适的研究类型
   - 展示完整计划，用户确认后才执行

2. **执行 Agent**：战术层，规划"怎么做研究"
   - 固定工作流（5 阶段）
   - 专注执行，不做意图判断
   - 消息驱动，边做边展示

**好处**：
- 新手有指导（Plan Mode）
- 熟练用户有捷径（直接执行）
- 关注点分离（意图 vs 执行）
- 各自优化（澄清 vs 研究）

## 常见问题

### Q1: 我一定要用 Plan Mode 吗？
**A**: 不一定，取决于你的需求明确度和熟练程度。

**必须用 Plan Mode**：
- ❌ 没有这种情况，Plan Mode 始终是可选的

**强烈建议用 Plan Mode**：
- 第一次使用 atypica.AI
- 不确定研究方向
- 需要背景调研
- 想看到计划再确认

**可以直接执行**：
- 需求非常明确
- 熟练用户，重复性研究
- 时间紧迫，快速验证

### Q2: Plan Mode 会不会浪费时间？
**A**: 取决于你的需求明确度。

**不会浪费时间的情况**（80% 用户）：
- 需求模糊：Plan Mode 2-3 轮澄清（2-5 分钟）节省了你思考和试错的时间
- 第一次使用：澄清过程是学习过程，理解方法论后下次可以直接执行

**可能多花时间的情况**（20% 熟练用户）：
- 需求极其明确：澄清过程可能多花 2-3 分钟
- 重复性研究：每次澄清类似内容

**建议**：
- 第一次：用 Plan Mode，学习方法论
- 第二次及以后：如果需求明确，可以直接执行

### Q3: Plan Mode 澄清了几轮后，我可以跳过吗？
**A**: 不可以中途跳过，但可以快速完成。

**原因**：
- Plan Mode 必须完成完整意图组装
- 最后调用 makeStudyPlan 提交计划
- 用户确认后才能路由到执行 Agent

**加速方法**：
- 在第一轮对话时提供尽可能多的信息
- 例如："我想对 18-28 岁一线城市年轻人进行气泡咖啡购买决策研究，重点关注购买动机和价格敏感度"
- Plan Mode 可能只需要 1-2 轮补充澄清

### Q4: Plan Mode 会自动判断研究类型，我可以不同意吗？
**A**: 可以，在确认计划时可以要求修改。

**流程**：
1. Plan Mode 展示完整计划（包含判断的研究类型和方法）
2. 用户选择：
   - **确认执行**：按计划开始
   - **修改计划**：告诉 Plan Mode 哪里需要调整
   - **取消研究**：不执行

**修改示例**：
- 用户："我觉得用 Discussion 更合适，不要 Interview"
- Plan Mode：重新组装计划，调用 makeStudyPlan 展示修改后的计划
- 用户：再次确认

### Q5: 直接执行时，AI 会不会判断错研究方向？
**A**: 可能会，因为没有意图澄清层。

**风险**：
- 用户输入模糊："了解用户对产品的看法"
- 执行 Agent 可能理解为"用户画像研究"，但你想要的是"竞品对比"
- 执行到一半才发现方向错了

**降低风险方法**：
1. **输入尽可能详细**：明确研究对象、场景、维度、方法
2. **参考之前的研究**：如果做过类似研究，直接参考那次的设计
3. **使用 Plan Mode**：不确定时，让 AI 先澄清再执行

### Q6: Plan Mode 可以使用 webSearch，直接执行可以吗？
**A**: 可以，但使用时机和目的不同。

**Plan Mode 的 webSearch**：
- **时机**：意图澄清后，执行前
- **目的**：了解行业背景，辅助研究设计
- **例子**：研究电动汽车前，先搜索最新市场动态

**执行 Agent 的 webSearch**：
- **时机**：研究执行中，需要补充信息时
- **目的**：获取具体数据、案例、事实
- **例子**：报告中需要引用行业数据

**区别**：
- Plan Mode: 宏观背景 → 指导研究设计
- 执行 Agent: 微观细节 → 丰富研究内容

### Q7: 我可以先用 Plan Mode 澄清，然后保存计划，之后再执行吗？
**A**: 目前不支持保存计划稍后执行。

**当前流程**：
- Plan Mode 展示计划 → 用户确认 → 立即开始执行
- 一旦确认，就会创建研究并开始执行

**未来可能支持**：
- 保存研究模板
- 定时执行研究

**当前解决方法**：
- 复制 Plan Mode 生成的计划内容
- 下次直接执行时，粘贴作为输入

### Q8: Plan Mode 判断研究类型的准确率如何？
**A**: 基于三层决策树，准确率很高（内部测试 > 90%）。

**决策逻辑**（从 planMode.ts 提取）：

**第一层：Agent 选择**
1. 扫描关键词："播客"/"快速"/"音频" → fastInsight Agent
2. 扫描关键词："创新"/"灵感"/"新产品" → productRnD Agent
3. 其他深度研究 → Study Agent

**第二层：Study 子类型**
- "比较"/"测试"/"验证" → testing
- "了解"/"发现"/"分析" → insights
- "设计"/"创意"/"头脑风暴" → creation
- "制定"/"规划"/"战略" → planning
- 无法归类 → misc

**第三层：研究细节**
- 自动选择合适框架（JTBD / KANO / STP 等）
- 判断研究方式（interview / discussion）
- 确定人设数量（5-10 个）

**边界情况处理**：
- 使用 reasoningThinking 工具深度分析
- 不确定时默认 insights（最通用）

### Q9: 如果我对 AI 判断的框架不满意怎么办？
**A**: 可以在确认计划时要求修改，或在直接执行时自己指定。

**方法 1：在 Plan Mode 中修改**
1. Plan Mode 展示计划（包含框架）
2. 用户："我想用 KANO 模型，不要 JTBD"
3. Plan Mode 重新组装计划

**方法 2：直接执行时指定**
"请使用 KANO 模型分析用户对 X 功能的优先级..."

**方法 3：执行中调整**
- 执行 Agent 在 planStudy 阶段也会规划框架
- 可以在执行中要求调整

### Q10: Plan Mode 和直接执行的成本一样吗？
**A**: Plan Mode 略贵，但差异很小（< 10%）。

**成本构成**：
- **Plan Mode**：意图澄清（2-3 分钟，约 5-10 元）+ 研究执行（28 分钟，约 90-100 元）= **总计 95-110 元**
- **直接执行**：研究执行（28 分钟，约 90-100 元）

**差异原因**：
- Plan Mode 多了澄清对话（requestInteraction、webSearch、makeStudyPlan）
- 但澄清成本很低（简单对话 + 一次搜索）

**建议**：
- 不要因为成本选择（差异 < 10 元）
- 根据需求明确度选择（新手用 Plan Mode 避免试错浪费更多）
