# Plan Mode：让 AI 先想清楚再动手

**一句话总结**：Plan Mode 是意图澄清层，AI 自动生成完整研究计划，展示方法和步骤，你确认后再执行。

---

## 为什么需要 Plan Mode？

### 传统 AI 的问题

**直接执行，黑盒操作**：
- 用户："我想了解 Z 世代对可持续时尚的态度"
- AI：立即开始执行（用户不知道它会做什么）
- 结果：可能方向不对，浪费时间和成本

### Plan Mode 的解决方案

**先展示计划，确认后执行**：
- 用户："我想了解 Z 世代对可持续时尚的态度"
- AI：生成 5 步研究计划
  1. 定义目标用户（18-25 岁 Z 世代）
  2. 观察社交媒体（小红书/B站讨论）
  3. 群体讨论（8 人讨论可持续时尚话题）
  4. 深度访谈（5 人深挖支付意愿）
  5. 生成研究报告
- 用户：审核计划 → 确认或修改
- AI：开始执行

---

## 核心价值

### 1. 透明化：看到 AI 的思考过程

**用户能看到**：
- AI 会调研哪些人
- 用什么研究方法
- 分几个步骤
- 预计产出什么

**好处**：
- 不再是黑盒
- 可以提前发现问题
- 避免方向错误

---

### 2. 可控性：你可以修改计划

**灵活调整**：
- 不满意人设选择 → 修改筛选条件
- 不需要某个步骤 → 删除
- 想增加研究深度 → 添加更多访谈

**vs. 传统 AI**：
- 传统：执行后才发现不对，已经浪费成本
- Plan Mode：执行前调整，零成本修正

---

### 3. 协作性：团队可以一起审核

**团队使用场景**：
- 研究员提需求
- Plan Mode 生成计划
- 团队成员审核计划
- 讨论后确认执行

**价值**：
- 确保研究方向符合团队共识
- 避免执行后的争议

## 真实案例：气泡咖啡产品验证

### 用户输入（初始需求）

```
我想做一款气泡咖啡，目标用户是年轻白领，
但不确定应该主打健康还是社交属性。
```

**问题**：需求模糊，不知道怎么开始

---

### Plan Mode 生成计划

AI 自动分析需求，生成以下计划：

**研究计划：气泡咖啡产品定位验证**

**目标**：确定产品定位（健康 vs 社交 vs 其他）

**步骤**：

1. **定义目标用户特征**
   - 人群：25-35 岁一线城市白领
   - 筛选条件：经常喝咖啡，愿意尝试新产品
   - 预计人数：8 人

2. **市场趋势分析**
   - 使用 Product R&D Agent
   - 分析咖啡市场趋势
   - 输出 3 个可能的定位方向

3. **群体讨论测试定位**
   - 使用 Discussion
   - 8 人讨论 3 个定位方向
   - 快速筛选最受欢迎的方向

4. **深度访谈验证动机**
   - 使用 Interview
   - 1v1 访谈 5 个高意向用户
   - 深挖为什么喜欢这个定位

5. **生成研究报告**
   - 产品定位建议
   - 目标用户画像
   - 定价和包装建议

**预计时间**：8-12 小时
**预计输出**：完整产品定位报告

---

### 用户审核计划

**用户看到计划后**：

✅ **满意的部分**：
- 研究步骤清晰
- 先讨论后访谈的顺序合理

⚠️ **需要调整**：
- "25-35 岁太宽泛，我想重点关注 28-32 岁女性"
- "想增加包装设计测试"

**用户修改**：
1. 调整目标用户：28-32 岁女性白领
2. 增加步骤：上传 3 个包装设计稿，Discussion 测试反馈

---

### AI 更新计划

根据用户反馈，AI 自动更新计划：

**研究计划：气泡咖啡产品定位验证（v2）**

**目标**：确定产品定位 + 包装设计验证

**步骤**：

1. 定义目标用户特征
   - 人群：**28-32 岁一线城市女性白领**（已修改）
   - 筛选条件：经常喝咖啡，注重产品颜值
   - 预计人数：8 人

2. 市场趋势分析（同上）

3. 群体讨论测试定位（同上）

4. 深度访谈验证动机（同上）

5. **包装设计测试**（新增）
   - 上传 3 个包装设计稿
   - Discussion 讨论哪个更吸引人
   - 输出包装建议

6. 生成研究报告（更新：包含包装建议）

**预计时间**：10-14 小时
**预计输出**：产品定位报告 + 包装设计建议

---

### 用户确认执行

**用户**："计划看起来不错，开始执行吧"

**AI**：立即开始执行，按照确认的计划进行研究

## Plan Mode 的 4 种使用场景

### 场景 1：初次使用，不确定怎么提需求

**用户困境**：
- 知道要做研究，但不知道怎么描述需求
- 担心提需求不够专业

**Plan Mode 价值**：
- 用户只需简单描述想法
- AI 自动补全研究细节
- 用户看到计划后理解"原来可以这样做"

**案例**：
> 用户："我想知道年轻人对我们品牌的看法"
>
> Plan Mode 生成：
> 1. 定义"年轻人"（18-30 岁）
> 2. Scout Agent 观察社交媒体讨论
> 3. Interview 深度访谈品牌认知
> 4. Discussion 测试新定位方向
> 5. 生成品牌认知报告

---

### 场景 2：复杂项目，需要确认研究路径

**用户困境**：
- 研究项目复杂，涉及多个环节
- 担心 AI 理解错误，执行错方向

**Plan Mode 价值**：
- 展示完整研究路径
- 用户可以提前发现遗漏或错误
- 确保 AI 理解了需求

**案例**：
> 用户："我想做一个新产品从创意到验证的完整研究"
>
> Plan Mode 生成 8 步计划：
> 1. 市场机会分析
> 2. 跨领域创意生成
> 3. 创意讨论筛选
> 4. 深度访谈验证
> 5. 包装设计测试
> 6. 定价测试
> 7. 渠道策略讨论
> 8. 生成完整产品方案
>
> 用户看到后："太完整了，这正是我要的"

---

### 场景 3：团队协作，需要让同事看到计划

**用户困境**：
- 需要向老板或团队说明研究方案
- 传统 AI 直接执行，无法提前展示

**Plan Mode 价值**：
- 生成可视化研究计划
- 团队可以一起讨论和审核
- 确认后再执行，避免争议

**案例**：
> 品牌经理："我们想重新定位品牌，但需要先给老板看研究方案"
>
> 使用 Plan Mode：
> 1. 生成 5 步研究计划
> 2. 导出计划 PDF
> 3. 团队会议审核
> 4. 老板确认后执行
> 5. 避免"执行后才发现老板不满意"

---

### 场景 4：预算有限，需要控制研究范围

**用户困境**：
- 预算或时间有限
- 需要选择性执行研究步骤

**Plan Mode 价值**：
- 看到完整计划后，可以删减步骤
- 优先执行最重要的部分
- 节省成本

**案例**：
> 用户："我想做用户画像研究，但预算有限"
>
> Plan Mode 生成：
> 1. Scout Agent 观察（10 轮）
> 2. Discussion 验证（8 人）
> 3. Interview 深度访谈（10 人）
> 4. 生成画像报告
>
> 用户调整：
> - 删除 Interview（最耗时）
> - 只保留 Scout + Discussion
> - 节省 50% 时间和成本

## Plan Mode vs 直接执行

| 维度 | Plan Mode | 直接执行 |
|------|----------|---------|
| **透明度** | 完全透明，看到所有步骤 | 黑盒，不知道 AI 在做什么 |
| **可控性** | 可修改计划，灵活调整 | 无法调整，只能重新开始 |
| **风险** | 提前发现问题，零成本修正 | 执行后才发现，浪费成本 |
| **协作** | 团队可以审核计划 | 难以向他人说明 |
| **适用场景** | 复杂项目、初次使用、团队协作 | 简单任务、熟练用户 |
| **时间** | +5-10 分钟审核时间 | 立即执行 |

**推荐**：大多数情况下使用 Plan Mode，避免方向错误

## 常见问题

### Q1：Plan Mode 会增加多少时间？

**审核计划时间**：5-10 分钟
**价值**：避免方向错误，节省数小时到数天

**计算**：
- 不用 Plan Mode：执行 8 小时后发现方向错，重新开始，总计 16+ 小时
- 用 Plan Mode：审核 10 分钟 + 执行 8 小时，总计 8.17 小时

**结论**：Plan Mode 反而节省时间

---

### Q2：生成的计划可以完全自定义吗？

**可以**：
- 修改研究步骤
- 调整目标用户
- 增删研究方法
- 改变人数和范围

**不可以**：
- 不能让 AI 做它不支持的研究方法
- 基本框架受限于 atypica.AI 的能力

---

### Q3：如果我不满意计划，可以重新生成吗？

**可以**：
- 点击"重新生成"
- AI 会基于需求生成新的计划
- 可以多次重新生成

**建议**：
- 如果只是小调整，直接修改更快
- 如果方向完全错误，重新生成

---

### Q4：Plan Mode 是必须的吗？

**不是必须**：
- 熟练用户可以跳过 Plan Mode
- 简单任务可以直接执行

**但强烈推荐**：
- 初次使用时用 Plan Mode 理解流程
- 复杂项目用 Plan Mode 避免错误
- 团队协作用 Plan Mode 达成共识

---

### Q5：生成计划后多久需要确认？

**没有时间限制**：
- 计划保存在系统中
- 可以随时回来审核
- 可以分享给团队讨论后再确认

**建议**：
- 当天确认，避免忘记上下文
- 重要项目可以讨论 1-2 天

## 实战建议

### 1. 初次使用必用 Plan Mode

**原因**：
- 理解 atypica.AI 的工作方式
- 学习如何提更好的需求
- 避免因不熟悉而浪费成本

**效果**：
- 第 1 次：看到计划后"原来是这样工作的"
- 第 3 次：开始理解如何提需求
- 第 10 次：可以考虑跳过 Plan Mode

---

### 2. 复杂项目必用 Plan Mode

**什么是复杂项目**：
- 涉及 3+ 个研究步骤
- 需要多种研究方法
- 预算或时间有限需要精确控制

**价值**：
- 提前优化研究路径
- 避免中途发现方向错误
- 节省时间和成本

---

### 3. 团队协作必用 Plan Mode

**原因**：
- 需要向老板/同事说明研究方案
- 需要团队讨论和共识
- 避免执行后的争议

**工作流**：
1. 研究员用 Plan Mode 生成计划
2. 导出计划文档
3. 团队会议审核
4. 确认后执行

---

### 4. 简单任务可以跳过

**什么是简单任务**：
- 只需要 1-2 个步骤
- 研究方向非常明确
- 熟练用户

**例子**：
- "8 个人讨论这个产品概念"
- "观察小红书讨论露营的用户"

**可以直接执行**，不需要 Plan Mode
