# MCP Integration：团队自定义工具

**一句话总结**：通过 MCP 协议接入团队自有数据源和工具，无需改代码。

## 支持场景

- 接入公司内部数据库（如"查询我们的用户行为数据"）
- 集成第三方 API（如"调用我们的 CRM 系统"）
- 自定义分析工具（如"运行我们的市场模型"）

## 工作机制

1. AI 自动从团队配置的工具中选择合适工具
2. 创建临时执行单元，使用选中的工具完成任务
3. 返回结果并继续研究

## vs. 固定工具

- 固定工具：所有用户共享，无法定制
- MCP Integration：每个团队配置自己的工具，数据隔离

## 真实案例

**某数据分析团队**：

**需求**：想用 atypica.AI 做用户研究，但需要查询公司的 Snowflake 数据库

**解决方案**：
1. 配置 MCP 连接 Snowflake
2. 提问："分析上季度 25-35 岁女性用户的留存情况"
3. AI 自动选择 Snowflake 工具 → 生成 SQL 查询 → 返回分析结果
4. 基于数据分析，AI 再进行 Discussion 探索用户流失原因

**价值**：打通内部数据和 AI 研究能力，形成完整闭环。
