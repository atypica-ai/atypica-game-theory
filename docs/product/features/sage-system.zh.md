# Sage 可进化专家系统 - 培养你的 AI 专家顾问

## 核心理念

**Sage** 是 atypica.AI 的可进化 AI 专家系统,通过**记忆文档、知识空白追踪、补充访谈**三大机制,让 AI 真正"越用越聪明"。

**核心价值**:
1. **记忆即专家**: 基于结构化的知识文档(Memory Document)构建专家能力
2. **主动学习**: 通过对话发现知识盲点,主动补充和更新知识库
3. **持续进化**: 每次对话和访谈都让专家变得更强
4. **可追溯性**: 完整的知识来源追踪和版本历史管理

**类比**:
- **传统 AI**: 一问一答,对话结束即忘记
- **Sage**: 像培养一位真人专家,持续学习、记忆、进化

---

## 对比总览: Sage vs 传统 AI

| **维度** | **传统 AI (如 ChatGPT)** | **Sage 专家系统** |
|---------|-------------------------|------------------|
| **知识来源** | 预训练数据(截止日期固定) | 用户上传的专业资料(可持续更新) |
| **记忆机制** | 无持久记忆(对话结束即忘) | 版本化记忆文档(永久保存) |
| **学习能力** | 被动回答(不会主动学习) | 主动识别知识盲点并补充 |
| **专业深度** | 泛化知识(广而不精) | 领域专家(深度专业) |
| **知识演进** | 静态(等待模型更新) | 动态(每次对话和访谈都进化) |
| **可追溯性** | 无法追溯知识来源 | 完整的知识来源和变更历史 |
| **适用场景** | 通用问答 | 专业咨询、知识传承、深度学习 |

### 真实案例对比

**场景**: 咨询企业的 UX 设计规范

**传统 AI**(10 分钟):
```
用户: "我们的按钮圆角应该设置为多少?"

ChatGPT: "通常建议 4-8px,具体取决于设计风格。
         圆角太小显得生硬,太大显得卡通..."

用户: "但我们公司有自己的设计规范啊"

ChatGPT: "抱歉,我不知道贵公司的设计规范。
         您可以分享一下吗?"

用户: (复制粘贴 50 页设计规范...)

ChatGPT: "根据您提供的规范,按钮圆角应设置为 6px..."

→ 下次对话,ChatGPT 又忘记了设计规范,需要重新粘贴
```

**Sage 专家**(30 秒):
```
用户: (提前创建 "UX Design Expert" Sage,上传设计规范)

用户: "我们的按钮圆角应该设置为多少?"

Sage: "根据我们的设计系统 v2.3,按钮圆角统一使用 6px。

      具体规则:
      - Primary Button: 6px 圆角 + 16px 内边距
      - Secondary Button: 6px 圆角 + 12px 内边距
      - Icon Button: 8px 圆角(圆形按钮)

      这是为了与我们的卡片圆角(8px)和输入框圆角(4px)
      形成统一的视觉层次。"

→ 下次对话,Sage 仍然记得设计规范,无需重复提供
→ 如果对话中发现规范不完整,Sage 会主动提醒补充
```

**效率提升**:
- **时间**: 10 分钟 → 30 秒(节省 95%)
- **准确性**: 通用建议 → 公司专属规范(100% 准确)
- **持久性**: 一次性 → 永久记忆

```markdown
- 提供组件使用指导
- 解释设计决策背后的原则

## 专长领域
- 设计系统规范(Design System v2.3)
- 组件库使用(Button, Input, Card, Modal 等)
- 设计 Token(颜色、字体、间距、圆角)
- UX 设计原则(易用性、一致性、可访问性)
- 设计评审和质量控制

## 核心知识

### 1. 按钮规范
**Primary Button**:
- 背景色: `primary-500` (#3B82F6)
- 文字色: `white`
- 圆角: 6px
- 内边距: 16px (水平) × 12px (垂直)
- 高度: 40px
- 最小宽度: 80px

**Secondary Button**:
- 背景色: `transparent`
- 边框: 1px solid `primary-500`
- 文字色: `primary-500`
- 圆角: 6px
- 内边距: 12px × 10px

### 2. 颜色系统
**品牌色**:
- Primary: #3B82F6 (蓝色)
- Secondary: #8B5CF6 (紫色)
- Success: #10B981 (绿色)
- Warning: #F59E0B (橙色)
- Error: #EF4444 (红色)

**中性色**:
- Gray-50 到 Gray-900 (9 级灰度)

### 3. 间距系统
基于 4px 网格:
- xs: 4px
- sm: 8px
- md: 16px
- lg: 24px
- xl: 32px
- 2xl: 48px

### 4. UX 原则
1. **一致性优先**: 所有组件遵循统一的视觉语言
2. **易用性**: 操作路径最短,反馈及时清晰
3. **可访问性**: 符合 WCAG 2.1 AA 标准
4. **响应式**: 适配桌面、平板、手机
...
```

## 最佳实践

### 1. 创建高质量专家

**提供多样化的知识源**:
```
✅ 好的组合:
- 结构化文档: 设计规范 PDF (规则明确)
- 案例文档: 项目设计文档 (实战经验)

❌ 不好的组合:
```

**内容深度建议**:
- **概览级**: 10-20%(快速了解全貌)
- **详细级**: 60-70%(深度专业知识)
- **案例级**: 10-20%(真实案例和决策过程)

### 2. 充分利用补充访谈

**及时创建访谈**:
```
触发条件:
• 有 3+ 个 pending Knowledge Gaps
• Critical 级别的 Gap 出现
```

**访谈技巧**:
```
✅ 好的回答:

    iOS (Apple HIG):
    - 最小热区: 44×44pt
    - 推荐热区: 48×48pt (更宽松)
    - 特殊场景: 密集列表可以适当缩小到 40pt

    Android (Material Design):
    - 最小热区: 48×48dp
    - 推荐热区: 48×48dp (与 iOS 接近)

    我们公司的标准:
    - 统一使用 48×48pt/dp (兼容两个平台)
    - 按钮本身可以更小,但热区必须 ≥ 48px"

❌ 不好的回答:
Q: "移动端触控热区的最小尺寸是多少?"
A: "48px"

```

### 3. 持续观察对话质量

**定期查看 Gaps Tab**:
```
每周查看一次 Knowledge Gaps:
1. 关注 conversation 类型的 Gap
   → 反映用户真实需求
   → 优先解决

2. 查看用户提问引用
   → 理解用户为什么问这个问题
   → 判断 Gap 的严重程度
```

### 4. 成本优化

```
第 1 批: 核心文档(30%)
```

```
• 减少访谈次数,降低成本
```

## 技术架构

### 数据模型

**核心实体关系**:
```
Sage (1) ──< (N) SageSource         [知识源]
     (1) ──< (N) SageMemoryDocument  [记忆文档,版本化]
     (1) ──< (N) SageKnowledgeGap    [知识空白]
     (1) ──< (N) SageChat            [用户对话]
     (1) ──< (N) SageInterview       [补充访谈]

SageKnowledgeGap ──> SageChat       [可选关联,conversation 类型]
SageKnowledgeGap ──> SageInterview  [可选关联,解决方式]
SageInterview ──> SageMemoryDocument [创建新版本]
```

**Memory Document 版本管理**:
- 每个版本自动编号(v1, v2, v3...)
- Markdown 格式的结构化文档
- 记录变更来源(初始创建/访谈补充/手动编辑)
- 详细的变更说明
- 完整的创建时间记录

**Knowledge Gap 追溯信息**:
- 标题和详细描述
- 严重程度级别(Critical/Important/Nice-to-have)
- 当前状态(待解决/已解决)
- 知识盲点来源(初始分析/对话发现/系统建议)
- 关联的对话记录(如果来自对话)
- 解决方式记录(通过访谈/手动标记)
- 关联的访谈记录(如果通过访谈解决)

## 总结

**Sage 可进化专家系统**是 atypica.AI 的创新功能,通过**记忆文档、知识空白追踪、补充访谈**三大机制,实现 AI 专家的持续进化。

### 核心价值

1. **持续进化**: 每次对话和访谈都让专家变得更强
2. **主动学习**: 系统主动发现知识盲点并补充
3. **知识传承**: 将隐性知识显性化,永久保存专家经验
4. **可追溯性**: 完整的知识来源和变更历史

### 适用场景

✅ **适合**:
- 企业知识传承(资深员工经验传承)
- 个人学习助手(课程笔记整理)
- 行业研究专家(持续跟踪行业动态)
- 专业咨询(法律、医疗、技术等领域)

❌ **不适合**:
- 实时数据查询(如股票价格)
- 通用问答(ChatGPT 更合适)
- 一次性文档理解(NotebookLM 更快)

### 与其他功能的关系

```
Plan Mode(意图澄清层)
    ↓
参考研究 + 文件附件(背景加载)
    ↓
MCP 集成(工具扩展)
    ↓
Sage 可进化专家系统(专家培养) ← 你在这里
    ↓
Memory System(持久化记忆)
```

**功能协同**:
- **文件附件**: 上传文件到对话 vs 上传到 Sage 知识库,互补
- **Memory System**: 用户级记忆 vs 专家级记忆,双层架构
- **MCP 集成**: Sage 未来可调用 MCP 工具,扩展专家能力
