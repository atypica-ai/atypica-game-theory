# Scout Agent：社交媒体深度观察

本文档详细说明 Scout Agent（用户发现智能体）的核心能力、工作流程、技术实现和应用场景。

---

## 什么是 Scout Agent？

Scout Agent 是 atypica.AI 的社交媒体观察智能体，**像人类学家做田野调查**，通过沉浸式观察社交媒体内容，理解一群人的生活方式、价值观和精神世界。

**核心理念**：
> 不是收集数据和统计特征，而是**观察和理解**一个群体的社会心理和文化特征。

---

## 核心差异：Scout vs 传统社交聆听工具

| 维度 | Scout Agent（atypica.AI） | 传统社交聆听工具（Brandwatch等） |
|------|-------------------------|----------------------------|
| **目标** | 理解群体的社会心理和文化特征 | 监控品牌提及和情感分析 |
| **方法** | 人类学家式田野调查 | 数据抓取和统计分析 |
| **关注点** | "这群人为什么这样想、这样说、这样做" | "品牌被提及了多少次、正面还是负面" |
| **输出** | 社会心理分析 + AI 人设构建 | 数据报表 + 情感趋势图 |
| **数据源** | 5个平台（小红书、抖音、Instagram、TikTok、Twitter） | 100万+ 网站（覆盖更广但不够深） |
| **工作流程** | 3阶段（观察 → 推理 → 验证） | 持续监控 + 实时报警 |
| **最终产物** | 可用于访谈的 AI 人设 | 品牌监控报告 |

## 三阶段工作流程

### Phase 1: 沉浸式观察阶段（前 5 次工具调用）

**目标**：不急于下结论，先大量浏览和观看

**AI 行为**：
- 关注"有意思"、"有共鸣"、"有冲突"的内容
- 注意语言的微妙之处：语气、措辞、表达习惯
- 记录反复出现的主题、情绪、价值观表达

**可用工具**（15个社交媒体工具）：
- **搜索工具**：`xhsSearch`, `dySearch`, `tiktokSearch`, `insSearch`, `twitterSearch`
- **用户内容**：`xhsUserNotes`, `dyUserPosts`, `tiktokUserPosts`, `insUserPosts`, `twitterUserPosts`
- **评论互动**：`xhsNoteComments`, `dyPostComments`, `tiktokPostComments`, `insPostComments`, `twitterPostComments`

**示例观察过程**（气泡咖啡案例）：
```
第1次：小红书搜索 "职场女性 咖啡 提神"
- 发现：大量"下午3点续命咖啡"的内容
- 语言：用户常用"续命"、"救急"、"醒脑"
- 情绪：对"糖分焦虑"反复提及

第2次：小红书搜索 "0糖饮料 健康"
- 发现：用户对"0糖"概念高度敏感
- 语气：既有"求安慰"也有"自我怀疑"
- 关键词："代糖安全吗"、"真的0卡吗"

第3次：抖音搜索 "气泡水 咖啡"
- 发现：对气泡+咖啡混搭表现出好奇
- 评论："会不会很怪"、"想尝试但怕踩雷"
- 注意：颜值和打卡高频出现

第4次：Instagram搜索 "sparkling coffee"
- 发现：国外已有类似产品，主打"energizing"
- 视觉：偏向"清爽"、"活力"
- 差异：用户关注"口感"多于"健康"

第5次：小红书搜索 "便利店 新品 咖啡"
- 发现：尝新意愿强，但对价格敏感
- 价格：18-22元处于"纠结区间"
- 复购：取决于"好不好喝"而非概念
```

### Phase 2: 深度理解阶段（第 5 次后，强制触发）

**触发机制**：
- 代码实现：当工具调用次数 >= 5 且未使用 `reasoningThinking` 时，强制调用
- 模型切换：临时切换到 `gemini-2.5-pro` 进行推理

**reasoningThinking 工具调用**：

```typescript
{
  background: "观察总结（简要概述现象）",
  question: "需要深层理解的社会心理问题"
}
```

**示例推理过程**：

**输入**：
```
background: "观察了约40条小红书和抖音内容。职场女性（25-35岁）在讨论咖啡和提神饮料时，反复使用'续命'、'救急'、'醒脑'等词汇，语气中既有自嘲也有焦虑。她们对'0糖'概念高度关注但充满怀疑（'代糖安全吗'）。对气泡+咖啡混搭表现出好奇但担心'踩雷'。价格18-22元被视为'有点贵但可以试试'的纠结区间。"

question: "这群人反复使用'续命'和'救急'表达下午提神需求，同时又对'0糖'充满怀疑和焦虑，这反映了什么样的价值观冲突？她们为什么既想尝新又怕'踩雷'？这种矛盾心理如何影响她们对18-22元新品的购买决策？"
```

**输出**（AI 推理结果）：
```markdown
## 社会心理分析

### 1. 身份焦虑与自我管理的冲突
- "续命"背后是职场高压下的无力感
- "0糖"焦虑源于对身体管理的强迫性需求
- 她们试图通过"健康选择"重获控制感

### 2. 尝新欲望与风险规避的平衡
- 好奇心驱动尝试，但"怕踩雷"反映消费理性化
- 18-22元不是"买不起"，而是"值不值"的判断
- 需要"理由"来说服自己：颜值、打卡、朋友推荐

### 3. 群体认同与个体表达的张力
- 社交媒体上的"晒"是为了获得认同
- 但她们抗拒"被营销"、"交智商税"的标签
- 产品需要平衡"潮流感"与"实用性"
```

### Phase 3: 继续观察验证（第 6-15 次工具调用）

**目标**：带着推理的洞察，继续观察更多细节

**AI 行为**：
- 验证或修正之前的理解
- 关注矛盾的、意外的、复杂的表现
- 进一步细化用户画像

**停止条件**：
- 工具调用次数达到 15-17 次
- Token 消耗超过 150,000
- 强制输出总结（第 15 次后，AI 被要求描述理解的这群人）

## 观察维度框架

Scout Agent 遵循以下观察维度（而非数据维度）：

### 1. 生活方式观察
- 他们如何描述自己的日常生活？用什么样的语气？
- 他们分享的生活片段透露出什么样的价值追求？
- 他们的表达方式体现了什么样的审美和品味？
- 他们在什么情境下表达快乐、焦虑、期待？

### 2. 社交互动观察
- 他们在社交媒体上如何与他人互动？语气是亲密的、疏离的、还是某种特定的风格？
- 他们如何回应不同观点？是包容的、对抗的、还是避而不谈？
- 在评论区他们如何表达共鸣？用什么样的话语方式？
- 他们如何定义"我们"和"他们"？群体边界在哪里？

### 3. 价值观与身份观察
- 他们在消费决策中透露出什么样的价值排序？
- 他们如何平衡理想与现实的冲突？
- 什么样的内容能引起他们强烈的情感共鸣？
- 他们认同什么样的身份标签？抗拒什么样的标签？

### 4. 表达与语言观察
- 他们习惯用什么样的词汇和句式？
- 他们的表达是直接的、委婉的、还是某种特定的修辞风格？
- 他们如何使用流行语、梗、emoji？
- 他们的语言风格在不同场景下如何变化？

## 技术实现细节

### 模型选择策略

| 阶段 | 模型 | 原因 |
|------|------|------|
| **观察阶段** | Gemini 2.5 Flash | 成本低、速度快、支持 Google Search grounding |
| **推理阶段** | Gemini 2.5 Pro | 推理能力强、支持 thinkingConfig（includeThoughts） |
| **强制总结** | Gemini 2.5 Pro | 生成质量高、能够整合多轮观察 |

### 工具调用控制（prepareStep 机制）

```typescript
prepareStep: async ({ messages }) => {
  const toolUses = 统计已调用工具次数;

  // 规则1：超过 15+2 次，强制输出总结
  if (toolUses.length >= 17) {
    return {
      model: llm("gemini-2.5-pro"),
      toolChoice: "none",  // 禁用工具
      activeTools: [],
      messages: [...messages, {
        role: "user",
        content: "观察已充分，请描述你理解的这群人。"
      }]
    };
  }

  // 规则2：第 5 次后，强制调用 reasoningThinking
  if (toolUses.length >= 5 && 未使用 reasoningThinking) {
    return {
      model: llm("gemini-2.5-pro"),
      toolChoice: "required",
      activeTools: [StudyToolName.reasoningThinking]
    };
  }

  // 规则3：正常观察阶段，排除 reasoningThinking
  return {
    activeTools: 社交媒体工具列表（不含 reasoningThinking）
  };
}
```

### Token 消耗控制

```typescript
reduceTokens: { model: "gemini-2.5-flash", ratio: 10 }

stopWhen: async ({ steps }) => {
  // 条件1：Token 消耗超过 15万
  if (tokensConsumed > 150_000) {
    logger.error(`Token 消耗超限，结束观察`);
    return true;
  }

  // 条件2：工具调用次数达到 20 次
  return steps.length >= 20 ||
         steps.reduce((acc, step) => acc + step.toolCalls.length, 0) >= 20;
}
```

## 我们能做什么

### ✅ 沉浸式社交媒体观察

**能力说明**：
- 同时覆盖 5 个主流社交平台
- 不是简单的关键词搜索，而是理解语境和情绪
- 识别语言微妙之处：语气、措辞、表达习惯

**技术支撑**：
- 15 个社交媒体工具（搜索、用户内容、评论互动）
- Google Search grounding（Gemini 2.5 Flash 内置）
- 自动记录平台覆盖情况（stats 统计）

**示例**：
> 搜索"职场女性 咖啡"时，不只是找到"咖啡"关键词，而是理解"续命咖啡"、"下午3点救急"背后的职场焦虑和身份认同。

### ✅ 社会心理深度分析

**能力说明**：
- 强制在第 5 次观察后进行 reasoningThinking
- 不是统计频率，而是理解"为什么"和"如何"
- 将观察现象提升到价值观、认同、文化符号层面

**技术支撑**：
- reasoningThinking 工具（Gemini 2.5 Pro）
- 支持 thinkingConfig（includeThoughts: true）
- 5-10 分钟深度推理时间

**输出质量**：
- 提炼价值观冲突（如"身份焦虑 vs 自我管理"）
- 识别矛盾心理（如"尝新欲望 vs 风险规避"）
- 揭示群体认同机制（如"社交晒图 vs 抗拒被营销"）

### ✅ 智能平台选择

**能力说明**：
- AI 自动判断全球性问题 vs 本地化问题
- 动态选择搜索平台组合

**策略**：
```
全球性问题（如"Z世代对元宇宙态度"）：
- 中文环境：小红书 + 抖音 + Instagram + Twitter
- 英文环境：Twitter + Instagram + TikTok + 小红书

本地化问题（如"北京职场女性咖啡习惯"）：
- 中文环境：仅小红书 + 抖音
- 英文环境：仅Twitter + Instagram + TikTok
```

### ✅ 构建可用于访谈的 AI 人设

**能力说明**：
- 基于观察结果，构建典型用户 AI 人设（Tier 1）
- 人设包含社会心理分析和行为模式
- 可直接用于后续的 Interview 或 Discussion

**数据来源**：
- 30-50 条社交媒体内容
- 社会心理分析结论
- 用户画像（7 维度框架）

**人设输出**（示例）：
```
Linda - 续命咖啡依赖者
- 年龄：28岁，运营岗位
- 典型行为：每天下午3点崩溃，对提神效果要求高
- 语言风格："续命"、"救急"、"没心情冒险"
- 价值观冲突：职场高压 vs 健康焦虑
- 决策逻辑：功效承诺 > 健康概念，需要"值"的理由
```

## 我们不能做什么

### 技术限制（物理上做不到或效果不好）

#### ❌ 不能替代定量的大样本社交聆听

**原因**：
- Scout 观察 30-50 条内容，无法达到统计显著性
- 不提供精确的百分比、趋势图、数据报表

**对比**：
- **Brandwatch**：1.6 万亿历史对话、100万+ 网站、实时监控
- **Scout Agent**：30-50 条深度理解、5 个平台、定性分析

**适用场景**：
- ✅ Scout 适合：理解"为什么"，构建 AI 人设
- ❌ Scout 不适合：品牌监控、危机预警、舆情趋势

**替代方案**：
- 如需定量监控，使用 Brandwatch / Sprinklr 等工具
- Scout 用于补充深度理解和人设构建

#### ❌ 不能覆盖所有社交平台

**原因**：
- 只覆盖 5 个主流平台（小红书、抖音、Instagram、TikTok、Twitter）
- 没有 Facebook、LinkedIn、YouTube、B站、知乎等

**对比**：
- **Brandwatch**：100万+ 网站和平台
- **Scout Agent**：5 个精选平台（但理解更深）

**适用场景**：
- ✅ Scout 适合：5 个平台的深度理解
- ❌ Scout 不适合：全网覆盖的舆情监控

#### ❌ 不能实时监控和报警

**原因**：
- Scout 是一次性观察任务，不是持续监控
- 没有实时报警机制（品牌危机、负面舆情）

**对比**：
- **Brandwatch**：实时监控、情感分析、自动报警
- **Scout Agent**：一次性深度观察（6-8 小时）

**适用场景**：
- ✅ Scout 适合：产品概念验证、用户群体理解
- ❌ Scout 不适合：品牌监控、危机预警

### 战略选择（可以做但刻意不做）

#### ❌ 不做浅层的关键词匹配

**原因**：
- 刻意不做"品牌被提及多少次"的简单统计
- 专注于理解"这群人是什么样的人"

**示例**：
- ❌ 不做："咖啡"被提及 500 次，正面情感占 70%
- ✅ 做："这群人用'续命'表达焦虑，用'悦己'合理化消费"

#### ❌ 不做历史数据回溯

**原因**：
- 刻意不做"回溯到 2010 年"的历史分析
- 专注于当前的用户心理和行为模式

**对比**：
- **Brandwatch**：回溯到 2010 年的历史数据
- **Scout Agent**：当前实时搜索结果

#### ❌ 不做多任务并行观察

**原因**：
- 每次 Scout 任务专注于一个群体或一个问题
- 不支持同时观察多个不相关的群体

**替代方案**：
- 如需观察多个群体，启动多个独立的 Scout 任务

## 与其他 atypica 功能的配合

### Scout → buildPersona → Interview

**最佳实践流程**：

1. **Phase 1 - Scout 观察**（2-3 小时）：
   - 观察社交媒体，理解群体特征
   - 输出：社会心理分析 + 平台覆盖统计

2. **Phase 2 - buildPersona**（自动）：
   - 基于 Scout 观察结果
   - 构建 3-5 个典型 AI 人设（Tier 1）
   - 保存到 Persona 库

3. **Phase 3 - Interview**（3-4 小时）：
   - 使用构建的 AI 人设进行深度访谈
   - 验证 Scout 观察的结论
   - 获得更细致的决策链条

**优势**：
- Scout 提供群体画像，Interview 提供个体深度
- 从"理解是什么样的人"到"理解为什么这样决策"
- 可追溯的洞察链条：社交媒体观察 → 社会心理分析 → 深度访谈

## 实际案例

### 案例：气泡咖啡目标用户发现

**研究目标**：
理解 25-35 岁都市职场女性对"气泡咖啡"新品的潜在反应。

**Scout 观察过程**：

**第 1-5 次观察**（沉浸阶段）：
```
1. 小红书搜索"职场女性 咖啡 提神"
   → 发现："下午3点续命咖啡"话题浏览量500万+

2. 小红书搜索"0糖饮料 健康"
   → 发现："代糖焦虑"相关帖子2000+条

3. 抖音搜索"气泡水 咖啡"
   → 发现：评论中"会不会很怪"、"想尝试但怕踩雷"

4. Instagram搜索"sparkling coffee"
   → 发现：国外产品主打"energizing"，用户关注口感

5. 小红书搜索"便利店 新品 咖啡"
   → 发现：18元以上价格让用户犹豫"值不值"
```

**第 5 次后**（推理阶段）：
```
reasoningThinking 输入：
"这群人反复使用'续命'和'救急'表达下午提神需求，同时又对'0糖'充满怀疑和焦虑，这反映了什么样的价值观冲突？"

AI 推理输出：
1. 身份焦虑与自我管理的冲突
   - "续命"背后是职场高压下的无力感
   - "0糖"焦虑源于对身体管理的强迫性需求

2. 尝新欲望与风险规避的平衡
   - 好奇心驱动尝试，但"怕踩雷"反映消费理性化
   - 18-22元不是"买不起"，而是"值不值"的判断

3. 群体认同与个体表达的张力
   - 社交媒体上的"晒"是为了获得认同
   - 但她们抗拒"被营销"、"交智商税"的标签
```

**第 6-12 次观察**（验证阶段）：
```
6. 小红书搜索"下午茶 打卡 办公室"
   → 验证：社交动机确实存在

7. 小红书搜索"代糖 焦虑 真相"
```

## 常见问题

### Q1: Scout 观察的内容是实时的吗？

**回答**：
- **部分实时**：搜索结果基于平台的当前数据
- **有延迟**：社交平台 API 可能有几小时到几天的延迟
- **不是历史回溯**：不像 Brandwatch 可以回溯到 2010 年

**建议**：
- 如需了解当前趋势，Scout 足够及时
- 如需历史对比分析，建议补充传统社交聆听工具

### Q2: Scout 的 reasoningThinking 是否可以跳过？

**回答**：
- **不可以**：第 5 次工具调用后，系统强制触发 reasoningThinking
- **设计原因**：确保观察不只是"看了看"，而是"理解了"
- **质量保证**：推理阶段是将现象提升到社会心理层面的关键

### Q3: Scout 观察 30-50 条内容是否太少？

**回答**：
- **不是统计研究**：Scout 的目标不是"X%的人这样认为"
- **是定性理解**：30-50 条足以识别语言模式、价值观冲突、群体认同
- **对比**：人类学家做田野调查，也不需要数千个样本

**质量保证**：
- AI 人设一致性：79分（社媒数据），接近 81% 人类基线
- 足以构建有代表性的 AI 人设用于后续访谈

### Q4: Scout 如何避免"筛选偏差"（selection bias）？

**回答**：
- **AI 主动多样化**：观察不同平台、不同关键词、不同用户群体
- **显性vs隐性表达**：不只看"我喜欢X"，也看语气、措辞、矛盾表述
- **推理阶段校验**：reasoningThinking 会识别"是否只看到了部分人"

**局限性**：
- 承认社交媒体用户 ≠ 全部目标用户
- 建议：Scout 用于探索性研究，高风险决策需补充真人调研

### Q5: Scout 的 Token 消耗如何控制？

**回答**：
- **模型选择**：观察阶段使用 Gemini 2.5 Flash（成本低）
- **Token 限制**：超过 15 万 tokens 自动停止
- **Ratio 优化**：`reduceTokens: { ratio: 10 }` 降低成本

**实际消耗**（气泡咖啡案例）：
- 12 次工具调用：约 80,000 tokens
- 成本：约 $1-2（远低于传统调研的 5-10 万元）
