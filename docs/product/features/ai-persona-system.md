> **文档类型**：功能深度解析
> **更新日期**：2026-01-15
> **适用人群**：产品经理、研究人员、所有用户

## 一句话总结

AI Persona 三层体系是基于 7 维度评分的人设质量分层系统，从 Tier0（普通）到 Tier3（私有），支持社媒观察、深度访谈、用户导入三种构建方式，提供 30 万+ AI 人设池供研究使用。

---

## 核心概念

### 什么是 AI Persona？

AI Persona 是模拟真实用户的 AI 智能体，具备完整的人设信息（人口统计、心理特征、行为模式等），可以参与 Interview、Discussion 等研究活动，提供接近真人的反馈。

### 为什么需要 AI Persona？

1. **降低成本**：不需要招募真人参与研究（省去招募费用和时间）
2. **提高速度**：30 分钟完成研究 vs 2-4 周真人研究
3. **规模化**：30 万+ 人设池，覆盖多样化人群，小众人群也能快速找到
4. **可重复**：同一个人设可以多次参与研究，保持一致性
5. **隐私保护**：不涉及真人隐私，无需伦理审批

### AI Persona 的质量保证

- **深访人设一致性**：85 分（超过人类重测基线 81%）
- **社媒人设一致性**：79 分（接近人类基线）
- **真人重合度**：80%+（AI 人设和真人的答案重合度）

## 三层体系架构

| Tier | 名称 | 评分范围 | 人设数量 | 来源 | 质量特征 | 适用场景 |
|------|------|---------|---------|------|---------|---------|
| **Tier 0** | 普通 AI Persona | 0-3 分 | - | 简单描述构建 | 信息不完整，质量较低 | 快速原型测试 |
| **Tier 1** | 合成智能体 | 4-5 分 | ~30 万 | 社交媒体观察（Scout） | 高质量合成，信息完整 | 常规研究、快速验证 |
| **Tier 2** | 真人模拟智能体 | 6-7 分 | ~1 万 | 深度访谈数据 | 接近真人，一致性 85 分 | 深度研究、关键决策 |
| **Tier 3** | 私有智能体 | - | 用户自定 | 用户导入（PDF 访谈等） | 用户私有，不可搜索 | 企业专用研究 |

### Tier 0：普通 AI Persona

**定义**：基于简单描述构建的 AI 人设，信息不完整。

**评分标准**：7 维度总分 < 4 分（满分 7 分）

**构建方式**：
- 直接输入简单描述（如："25 岁职场女性"）
- 系统自动生成基础人设信息

**质量特征**：
- 信息稀疏：只有基本的人口统计信息
- 一致性低：缺少心理特征和行为模式，回答可能前后矛盾
- 深度不足：无法提供深入的洞察

**适用场景**：
- 快速原型测试：验证研究流程是否可行
- 概念演示：向团队展示 AI Persona 功能
- **不适合**：正式研究、产品决策

**示例**：
```
名称：年轻职场女性
描述：25 岁，在一线城市工作，关注时尚和美妆
评分：2 分（demographic: 1, psychological: 1, 其他: 0）
```

### Tier 1：合成智能体（社媒人设）

**定义**：通过社交媒体观察构建的高质量 AI 人设。

**评分标准**：7 维度总分 4-5 分

**构建方式**：
- 使用 Scout Agent 观察社交媒体（小红书、抖音、Twitter 等）
- 提取用户的生活方式、价值观、语言风格等
- 自动评分并打标签（tags）

**质量特征**：
- 信息完整：包含 4-5 个维度的信息
- 一致性：79 分（接近人类基线 81%）
- 真实感：基于真实社交媒体内容构建

**人设数量**：~30 万（持续增长）

**适用场景**：
- 常规用户研究：理解目标用户群体
- 快速验证：产品概念测试、文案测试
- 大规模研究：需要多样性样本

**示例**：
```
名称：一线城市职场女性 - 30 岁困境
来源：小红书观察（100+ 帖子）
标签：职场焦虑、内耗、躺平、悦己消费、价格敏感
评分：4 分（demographic: 1, geographic: 1, psychological: 1, behavioral: 1, needs: 0, tech: 0, social: 0）
一致性：79 分
```

### Tier 2：真人模拟智能体（深访人设）

**定义**：基于深度访谈数据构建的高级 AI 人设，接近真人水平。

**评分标准**：7 维度总分 6-7 分

**构建方式**：
- 基于深度访谈记录（文字稿、音频转录等）
- atypica 团队手工审核和优化
- 包含完整的 7 维度信息

**质量特征**：
- 信息极其完整：包含所有 7 个维度
- 高一致性：85 分（**超过人类重测基线 81%**）
- 深度洞察：可以提供深层动机和决策路径

**人设数量**：~1 万（精选高质量人设）

**适用场景**：
- 深度用户研究：理解复杂决策过程
- 关键产品决策：新功能是否开发、定价策略等
- 战略规划：市场定位、品牌策略等

**示例**：
```
名称：30 岁一线城市独立女性 - 职场与生活平衡者
来源：深度访谈（45 分钟完整访谈记录）
标签：职场中层、独居、健身爱好者、悦己消费、理性决策
评分：6 分（所有 7 个维度都有信息，其中 6 个维度完整）
一致性：85 分
7 维度信息：
  - Demographic: 30 岁，女性，硕士，产品经理，年收入 40-50 万
  - Geographic: 北京，独居，公司附近租房
  - Psychological: 追求自我价值实现，平衡工作和生活，不被外界定义
  - Behavioral: 工作日规律作息，周末健身+阅读，每月旅行 1 次
  - Needs & Pain Points: 职场天花板焦虑，社交压力，寻找生活意义
  - Tech Acceptance: 早期采用者，愿意尝试新产品，注重效率工具
  - Social Relations: 小而精的社交圈，注重深度连接，家人关系良好但保持距离
```

### Tier 3：私有智能体（用户导入）

**定义**：用户通过 personaimport 功能导入的私有 AI 人设。

**评分标准**：自动设为 Tier 3（不参与公开评分）

**构建方式**：
- 用户上传真人访谈记录（PDF、音频转录等）
- 系统自动分析并生成人设
- 如果信息不完整，系统会生成补充问题建议

**质量特征**：
- 质量取决于导入数据：如果是完整的深度访谈，质量可能达到或超过 Tier 2
- 私有性：只有导入的用户可以使用，不会被其他用户搜索到
- 可更新：用户可以补充访谈数据，持续优化人设

**人设数量**：用户自定（每个用户可以导入任意数量）

**适用场景**：
- 企业专用研究：使用公司自己的真人访谈数据
- 保密性要求高：不希望人设信息被其他用户看到
- 高度定制：需要特定行业、特定人群的人设

**示例**：
```
名称：某企业客户 - 张女士
来源：用户导入（2024 年 9 月真人访谈记录）
标签：企业客户、大客户、决策者
评分：Tier 3（私有，不公开评分）
特征：
  - 只有上传的用户可以使用
  - 不会出现在公开人设搜索结果中
  - 可以持续更新和优化
```

## 7 维度评分系统

### 为什么是 7 个维度？

基于斯坦福小镇（Stanford Smallville）框架和用户研究最佳实践，7 个维度覆盖了理解一个人所需的关键信息。

### 7 个维度详解

| 维度 | 英文 | 说明 | 示例 | 权重 |
|------|------|------|------|------|
| 1. 人口统计学 | Demographic | 年龄、性别、教育、职业、收入等 | 30 岁，女性，硕士，产品经理 | 1 分 |
| 2. 地理 | Geographic | 城市、区域、居住环境等 | 北京，独居，公司附近租房 | 1 分 |
| 3. 心理特征 | Psychological | 价值观、生活态度、性格特质等 | 追求自我价值，平衡工作生活 | 1 分 |
| 4. 行为 | Behavioral | 消费习惯、社交方式、决策模式等 | 理性决策，注重性价比 | 1 分 |
| 5. 需求与痛点 | Needs & Pain Points | 核心需求、未被满足的需求、焦虑来源 | 职场天花板焦虑，寻找生活意义 | 1 分 |
| 6. 技术接受度 | Tech Acceptance | 对新技术的态度、使用习惯、学习能力 | 早期采用者，愿意尝试新产品 | 1 分 |
| 7. 社会关系 | Social Relations | 群体归属感、社交圈层、人际关系模式 | 小而精的社交圈，注重深度连接 | 1 分 |

### 评分逻辑

```typescript
// 每个维度评分 0-1 分（有信息 = 1 分，无信息 = 0 分）
const totalScore =
  demographic +      // 0 or 1
  geographic +       // 0 or 1
  psychological +    // 0 or 1
  behavioral +       // 0 or 1
  needsPainPoints +  // 0 or 1
  techAcceptance +   // 0 or 1
  socialRelations;   // 0 or 1

// 根据总分确定 Tier
if (totalScore >= 6) {
  tier = Tier2; // 真人模拟智能体
} else if (totalScore >= 4) {
  tier = Tier1; // 合成智能体
} else {
  tier = Tier0; // 普通 AI Persona
}
```

## 使用建议

### 如何选择合适的 Tier？

| 研究目标 | 推荐 Tier | 原因 |
|---------|----------|------|
| 快速原型测试 | Tier 0 | 成本最低，速度最快 |
| 常规用户研究 | Tier 1 | 性价比最高，30 万人设池 |
| 深度决策支持 | Tier 2 | 高一致性（85 分），接近真人 |
| 企业专用研究 | Tier 3 | 私有性强，可定制 |

### Tier 1 vs Tier 2：何时用哪个？

**用 Tier 1（合成智能体）的场景**：
- 需要快速验证：产品概念测试、文案测试
- 需要多样性：覆盖不同人群，发现模式
- 预算有限：成本更低
- 探索阶段：不确定具体研究方向

**用 Tier 2（真人模拟智能体）的场景**：
- 关键决策：新功能是否开发、定价策略
- 深度洞察：理解复杂决策过程和深层动机
- 高一致性要求：需要前后一致的反馈
- 战略规划：市场定位、品牌策略

**组合使用**：
1. Tier 1 快速探索（10 个人设，15 分钟）
2. Tier 2 深度验证（5 个人设，20 分钟）

### 如何构建自己的 AI Persona？

#### 方法 1：使用 Scout Agent（生成 Tier 1）
1. 描述目标用户群体："30 岁职场女性，关注健康和健身"
2. Scout Agent 自动观察社交媒体（小红书、抖音等）
3. 生成多个 Tier 1 人设（评分 4-5 分）
4. 保存到人设库，供后续研究使用

#### 方法 2：导入真人访谈（生成 Tier 3）
1. 准备真人访谈记录（PDF、音频转录等）
2. 使用 personaimport 功能上传
3. 系统自动分析并生成人设
4. 如果信息不完整，系统会建议补充问题
5. 持续优化（可以多次补充访谈数据）

#### 方法 3：手工构建（生成 Tier 0-2，取决于信息完整度）
1. 直接输入人设描述
2. 系统自动评分并确定 Tier
3. 如果评分较低，可以补充更多信息

## 技术实现细节

### 代码位置
- **人设创建**：`src/app/(persona)/lib.ts`
- **评分系统**：`src/app/(persona)/types.ts`
- **前端展示**：`src/app/(persona)/personas/`

### 评分实现
```typescript
// src/app/(persona)/lib.ts:105
export async function scorePersona(persona: Persona) {
  // 如果是私有导入，自动设为 Tier 3
  if (persona.personaImportId) {
    await prisma.persona.update({
      where: { id: persona.id },
      data: { tier: PersonaTier.Tier3 },
    });
    return;
  }

  // 使用 LLM 评估 7 个维度
  const result = await generateObject({
    model: llm("gpt-4.1-mini"),
    system: personaScoringPrompt({ locale }),
    schema: personaScoringSchema,
    messages: [
      {
        role: "user",
        content: `Prompt: ${persona.prompt}\n\nTags: ${persona.tags.join(", ")}`,
      },
    ],
  });

  // 计算总分
  const totalScore =
    result.object.demographic +
    result.object.geographic +
    result.object.psychological +
    result.object.behavioral +
    result.object.needsPainPoints +
    result.object.techAcceptance +
    result.object.socialRelations;

  // 确定 Tier
  const tier =
    totalScore >= 6
      ? PersonaTier.Tier2
      : totalScore >= 4
        ? PersonaTier.Tier1
        : PersonaTier.Tier0;

  // 更新数据库
  await prisma.persona.update({
    where: { id: persona.id },
    data: { tier, scoringResult: result.object },
  });

  // 如果 tier > 0，创建 embedding（用于相似度搜索）
  if (tier > 0) {
    await createPersonaEmbedding(persona);
  } else {
    await clearPersonaEmbedding(persona);
  }
}
```

### 为什么这样设计？

**自动评分而非人工标注**：
- 可扩展：可以处理 30 万+ 人设
- 一致性：避免人工标注的主观差异
- 实时更新：新构建的人设立即评分

**7 维度而非更多或更少**：
- 7 个维度覆盖了用户研究的核心需求
- 评分简单：每个维度 0/1，总分 0-7
- 易于理解：用户可以快速判断人设质量

**Tier 3 特殊处理**：
- 私有人设不参与公开评分（保护隐私）
- 用户可以看到自己导入的人设评分（内部使用）

## 常见问题

### Q1: AI Persona 真的能替代真人吗？
**A**: 不能完全替代，但可以在特定场景下提供接近真人的反馈。

**AI Persona 的优势**：
- 速度快：30 分钟 vs 2-4 周
- 成本低：50-100 元 vs 5-10 万元
- 规模化：30 万+ 人设池，小众人群也能快速找到
- 一致性高：深访人设 85 分，超过人类基线 81%

**AI Persona 的局限**：
- 不能观察真实行为（如产品使用行为）
- 不能捕捉非语言信息（如表情、肢体语言）
- 不能完全模拟复杂的情感反应

**使用建议**：
- **探索阶段**：用 AI Persona（快速、便宜）
- **验证阶段**：用真人研究（深度、真实）
- **组合使用**：AI Persona 筛选方向 → 真人深度验证

### Q2: 为什么深访人设只有 1 万，社媒人设有 30 万？
**A**: 深访人设需要完整的深度访谈数据，构建成本高。

**深访人设（Tier 2）**：
- 需要 45-60 分钟完整访谈记录
- 需要手工审核和优化
- 包含完整的 7 维度信息
- 构建成本：约 500-1000 元/个（人工成本 + 时间成本）

**社媒人设（Tier 1）**：
- 通过 Scout Agent 自动构建
- 基于社交媒体公开内容
- 无需人工干预
- 构建成本：约 50-80 元/个（纯 AI 成本）

**为什么不全部构建为 Tier 2**：
- 成本太高（30 万 × 500 元 = 1.5 亿元）
- 大部分研究不需要 Tier 2 的高一致性
- Tier 1 已经足够满足 80% 的研究需求

### Q3: 如何判断我的研究需要用哪个 Tier？
**A**: 根据研究目标和决策重要性选择。

**决策树**：
```
是否是关键产品决策？
  ├─ 是 → 用 Tier 2（深访人设）
  └─ 否 → 是否需要多样性样本？
      ├─ 是 → 用 Tier 1（社媒人设）
      └─ 否 → 是否有企业专用数据？
          ├─ 是 → 用 Tier 3（私有人设）
          └─ 否 → 用 Tier 1（社媒人设）
```

**示例**：
- "测试 3 个文案，看哪个更吸引人" → Tier 1（快速验证）
- "新功能是否开发，影响 100 万用户" → Tier 2（关键决策）
- "使用公司自己的客户访谈数据" → Tier 3（企业专用）

### Q4: Tier 3 私有人设的安全性如何？
**A**: 完全私有，其他用户无法访问。

**安全机制**：
1. **数据库隔离**：Tier 3 人设标记 personaImportId
2. **搜索隔离**：搜索人设时，Tier 3 不会出现在公开结果中
3. **访问控制**：只有导入的用户可以使用自己的 Tier 3 人设
4. **不参与公开评分**：Tier 3 评分不会公开显示

**数据保护**：
- 用户上传的访谈记录存储在加密数据库中
- 只有用户自己可以查看、修改、删除
- atypica 团队无法访问用户的 Tier 3 人设内容

### Q5: AI Persona 的一致性是如何测试的？
**A**: 通过重测相关性（Test-Retest Reliability）测试。

**测试方法**：
1. 选择同一个 AI 人设
2. 在不同时间（间隔 1 周）问相同的问题
3. 对比两次回答的一致性
4. 计算相关系数

**结果**：
- 深访人设（Tier 2）：85 分一致性
- 社媒人设（Tier 1）：79 分一致性
- 人类基线：81% 重测一致性（同一个人两次回答相同问题）

**结论**：
- Tier 2 深访人设**超过**人类基线（85 > 81）
- Tier 1 社媒人设接近人类基线（79 ≈ 81）
