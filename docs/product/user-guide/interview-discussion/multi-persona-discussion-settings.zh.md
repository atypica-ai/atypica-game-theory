# 如何让多个AI人设一起讨论？

## 问题类型
用户手册问题

---

## 快速开始（3步）

### Step 1：创建Discussion项目

**入口**：
- 首页 → "创建新访谈"
- 选择 **"Discussion Chat（群体讨论）"**

**填写项目信息**：
- 项目名称：如"气泡咖啡定价讨论"
- 研究目标：简短描述你想探讨的议题

---

### Step 2：选择AI人设（3-8人）

**推荐数量**：5-6人

**选人原则**：
- ✅ 确保观点多样性（不同立场）
- ✅ 不同背景和消费心理
- ❌ 避免选择观点相似的人

**示例**：测试气泡咖啡定价（5人）
- 2个价格敏感型（学生党、工薪族）
- 2个品质优先型（白领、高收入）
- 1个社交尝鲜型（Z世代、小红书用户）

---

### Step 3：选择讨论类型并开始

**3种讨论类型**：

#### 1. Focus Group（焦点小组）- 最常用

**适用**：
- 产品概念测试
- 用户态度调研
- 功能接受度

**特点**：
- AI主持人引导讨论
- 自由表达观点
- 寻找共识和分歧

---

#### 2. Debate（辩论模式）

**适用**：
- 方案对比（A vs B）
- 争议议题探讨
- 明确正反立场

**特点**：
- 分为正反方
- 观点对立碰撞
- 深度论证

---

#### 3. Roundtable（圆桌讨论）

**适用**：
- 探索性研究
- 开放式探讨
- 无明确立场

**特点**：
- 平等发言
- 无预设立场
- 多角度洞察

---

**点击"开始讨论"**

---

## 讨论执行过程

### AI主持人的作用

**1. 开场和引导**：
```
AI主持人：大家好，今天我们讨论的主题是"气泡咖啡定价"。
首先请问，大家觉得¥18的气泡咖啡贵吗？
```

---

**2. 控制发言节奏**：
- 主动点名："XX，你怎么看？"
- 邀请沉默者："XX还没发言，你的看法呢？"
- 控制话痨："谢谢XX的分享，我们也听听其他人的想法"

---

**3. 引导观点碰撞**：
```
AI主持人：我注意到你们对价格的看法差异很大，
A觉得贵，B觉得不贵，能说说具体原因吗？
```

---

**4. 深挖争议点**：
```
AI主持人：大家对"气泡+咖啡"这个组合有不同看法，
有人觉得是创新，有人觉得是噱头，为什么？
```

---

**5. 总结共识和分歧**：
```
AI主持人：看来大家对0糖概念都认可（共识），
但对价格接受度差异很大（分歧）。
```

---

### 讨论进行中

**实时观看**：
- 可以实时查看讨论进展
- 看到每个人设的发言

**中途调整**：
- 插入新问题
- 调整讨论方向
- 延长或结束讨论

**预计时长**：
- 标准：30-40分钟
- 深度讨论：50-60分钟

## 实际案例详解

### 案例1：Focus Group - 气泡咖啡定价（5人）

**人设组合**：
- Linda（28岁运营，价格敏感）
- Emma（32岁PM，健康焦虑）
- Chloe（26岁设计师，社交尝鲜）
- Sarah（30岁财务，理性决策）
- Mia（29岁市场，品牌控）

**讨论流程**：

**开场**：
```
AI主持人：大家觉得¥18的气泡咖啡贵吗？

Linda：太贵了，我一般买¥12的美式
Emma：我觉得还好，0糖很有吸引力
Chloe：价格可以接受，关键看颜值
Sarah：需要看性价比，和竞品比如何？
Mia：¥18不算贵，但这个品牌知名度不够
```

---

**深挖分歧**：
```
AI主持人：为什么你们对价格的看法差异这么大？

Linda：我预算有限，¥18超出日常预算
Emma：我更关注健康，0糖值得多花点钱
Chloe：我不在乎价格，能拍照发小红书就值了
```

---

**探索共识**：
```
AI主持人：大家对0糖这个概念怎么看？

Emma：0糖很吸引我，但担心代糖安全
Sarah：我也关注0糖，但要看成分表
Mia：0糖是趋势，这个卖点不错

（共识：0糖概念有吸引力，但需要打消代糖疑虑）
```

---

**关键争议点**：
```
AI主持人："气泡+咖啡"这个组合，大家觉得是创新还是噱头？

Chloe：我觉得很新颖，愿意尝试
Linda：听起来怪怪的，担心味道不好
Emma：气泡会不会刺激胃？
```

---

**讨论结果**：
- **共识**：0糖概念有吸引力
- **分歧**：价格接受度差异大（取决于收入和消费心理）
- **争议点**："气泡+咖啡"组合是否创新
- **关键洞察**：
  - 需要打消代糖疑虑（标注"天然甜味"）
  - 价格定位要明确（¥15-18区间，避免¥20+）
  - 包装设计要好看（吸引社交尝鲜型）

---

### 案例2：Debate - 订阅制 vs 买断制（6人）

**人设组合**：
- **正方**（支持订阅制）：3人
  - 小企业决策者（关注灵活性）
  - 个人用户（不想一次性付太多）
  - 重度用户（持续使用，订阅更划算）

- **反方**（支持买断制）：3人
  - 中型企业决策者（关注总成本）
  - 轻度用户（不常用，买断更合算）
  - 隐私敏感者（不想绑定信用卡）

**讨论流程**：

**开场**：
```
AI主持人：今天我们讨论订阅制和买断制，哪个更好。
正方认为订阅制更优，反方认为买断制更优。
先请正方第一位代表发言。
```

---

**正方论证**：
```
正方-小企业：订阅制灵活性高，不想一次性付¥5000
正方-个人用户：每月¥99可以接受，买断¥2999太贵
正方-重度用户：我每天用，订阅2年只需¥2376，比买断划算
```

---

**反方论证**：
```
反方-中型企业：我们用3年，买断¥2999总成本更低
反方-轻度用户：我一个月用几次，订阅每月¥99不划算
反方-隐私敏感：订阅要绑信用卡，我不想绑定支付
```

---

**交锋**：
```
AI主持人：双方论证都有道理，现在请自由辩论。

正方-重度用户：买断没有更新，订阅有持续更新
反方-中型企业：我们不需要频繁更新，稳定就好

正方-小企业：订阅可以随时取消，买断一次性风险大
反方-轻度用户：订阅容易忘记取消，反而花更多钱
```

---

**讨论结果**：
- **共识**：两种模式都有需求，不能二选一
- **正方优势**：灵活性、持续更新、低门槛
- **反方优势**：总成本低、无绑定、稳定性
- **关键洞察**：
  - 提供两种选项（订阅 + 买断）
  - 重度用户推订阅，轻度用户推买断
  - 订阅提供"年付8折"，降低总成本顾虑

---

### 案例3：Roundtable - 探索健身App需求（5人）

**人设组合**：
- 健身新手、资深健身者、减脂人群、老年健身者、康复训练者

**讨论流程**：

**开场**：
```
AI主持人：今天我们自由讨论健身App的需求和痛点。
大家平时用健身App遇到什么问题？
```

---

**自由讨论**：
```
新手：我最大的问题是动作不标准，担心受伤
资深：我觉得App功能太基础，不够专业
减脂：我想看到实时卡路里消耗，但App没有
老年：界面太复杂，我不会用
康复：我膝盖有伤，App没有适合我的训练计划
```

---

**主持人引导**：
```
AI主持人：看来大家的需求很不同，
新手关注易用性，资深用户关注专业性。
如果App要优先解决一个问题，应该是什么？
```

---

**讨论结果**：
- **共识**：动作纠正是刚需（新手和老年都提到）
- **分歧**：新手要简单，资深要专业
- **关键洞察**：
  - 分层设计：新手模式 + 进阶模式
  - 核心功能：动作纠正AI
  - 差异化：个性化训练计划（考虑伤病）

## 常见问题

### Q1：讨论时长可以控制吗？

**可以**。你可以设置：
- 最短时长：如30分钟（确保讨论充分）
- 最长时长：如60分钟（避免冗长）
- 自动判断：AI主持人识别"观点充分碰撞"后自动结束

---

### Q2：可以中途加人或换人吗？

**不可以**。讨论开始后不能更换人设。

**原因**：
- 讨论是连续的，中途换人会打断
- 新人设缺少上下文，难以融入

**建议**：
- 提前确认人设选择
- 确保人设多样性

---

### Q3：如果某个人设一直不发言怎么办？

**AI主持人会主动邀请**：
- "XX还没发言，你的看法呢？"
- 确保每个人都有表达机会

---

### Q4：讨论会不会偏离主题？

**AI主持人会控制**：
- 识别偏离，及时拉回
- 确保讨论聚焦核心议题

**你也可以**：
- 中途插入新问题
- 调整讨论方向

## 最佳实践

### 建议1：确保观点多样性

**不要**：
- ❌ 5个价格敏感型用户（没有碰撞）

**要**：
- ✅ 2个价格敏感 + 2个品质优先 + 1个品牌控
- ✅ 确保观点对立，才有讨论价值

---

### 建议2：先Interview后Discussion

**最佳流程**：
1. 先用Interview深度挖掘（5-8人）
2. 发现关键争议点
3. 再用Discussion观察观点碰撞（3-5人）

**优势**：
- Discussion的讨论主题更精准
- 先理解个体，再观察群体

---

### 建议3：使用Focus Group测试接受度

**场景**：新品概念测试、功能接受度

**优势**：
- 快速发现共识和分歧
- 观察观点在讨论中的演变

---

### 建议4：使用Debate测试方案

**场景**：A/B方案对比

**设置**：
- 正方3人支持A
- 反方3人支持B
- 观察哪方论证更充分

---

## 最后一句话

> "让AI人设一起讨论，观察观点碰撞，
> 快速发现共识、分歧和争议点。"

**记住**：
- ✅ 3步开始：创建项目 → 选人设 → 选讨论类型
- ✅ 确保观点多样性（不同立场）
- ✅ 最佳实践：先Interview深度挖掘 + 后Discussion观点碰撞
