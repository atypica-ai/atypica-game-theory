# brief写得简单和详细，对结果影响大吗？如何写出好的brief？有模板吗？

## 问题类型
用户手册问题

---

## 核心回答

**影响不大，因为Plan Mode会帮你补全和优化。**

- ✅ 简单描述：Plan Mode通过对话澄清细节
- ✅ 详细描述：Plan Mode直接生成精准计划
- ❌ 错误描述：Plan Mode会发现并纠正

**关键**：说清楚想研究什么，剩下的交给Plan Mode。

---

## 什么是Brief？

**Brief（研究简报）**：描述研究背景和目标的简短说明

**示例**：
```
我们是一个健身App，最近用户流失率上升，
想了解用户为什么流失，以及如何挽回。
```

**作用**：
- 让AI理解背景
- 帮助AI设计研究方向
- 确定研究范围

---

## 简单 vs 详细的影响

### 简单描述（推荐新手）

**示例**：
```
Brief："测试气泡咖啡新品"
```

**Plan Mode会做什么**：
1. 对话澄清：
   - "目标用户是谁？"
   - "想了解哪些方面？接受度、定价还是包装？"
   - "有参考材料吗？"

2. 补全细节：
   - 定义目标用户画像
   - 设计多个研究问题
   - 选择合适的研究方法

**优点**：
- 不需要想太多
- Plan Mode引导你思考
- 适合不确定的情况

**缺点**：
- 需要3-5轮对话澄清
- 多花5-10分钟

---

### 详细描述（推荐熟手）

**示例**：
```
Brief：
"我们计划推出一款0糖气泡咖啡，定位健康提神饮品，
目标用户是25-35岁一线城市白领女性，
主要竞品是星巴克、manner和元气森林。

想了解：
1. 目标用户对'气泡+咖啡'组合的接受度如何？
2. ¥15/¥18/¥20三个定价方案，哪个最合适？
3. 健康定位 vs 社交定位，哪个更有吸引力？
4. 包装设计需要注意什么？"
```

**Plan Mode会做什么**：
- 直接生成精准的研究计划
- 可能只问1-2个确认问题
- 立即进入审核阶段

**优点**：
- 节省澄清时间
- 计划更精准
- 效率更高

**缺点**：
- 需要事先想清楚
- 如果描述错误，可能方向偏差

## 如何写好Brief？

### 模板1：产品测试Brief

```
产品背景：
- 我们是[行业]公司
- 计划推出[产品名称]
- 主打[核心卖点]

目标用户：
- [年龄段] [城市]
  [职业/特征]

研究目标：
- 测试产品接受度
- 验证定价策略
- [其他目标]

竞品情况：
- 主要竞品：[品牌A、品牌B]
- 我们的差异：[差异化优势]
```

**填写示例**：
```
产品背景：
- 我们是咖啡行业创业公司
- 计划推出"气泡咖啡"
- 主打0糖健康

目标用户：
- 25-35岁一线城市
  白领女性

研究目标：
- 测试产品接受度
- 验证定价策略
- 收集包装设计反馈

竞品情况：
- 主要竞品：星巴克、manner、元气森林
- 我们的差异：气泡+咖啡创新组合
```

---

### 模板2：用户洞察Brief

```
业务背景：
- 我们的产品/服务：[名称]
- 当前遇到的问题：[问题描述]
- 想了解：[核心疑问]

目标用户：
- [用户画像]

研究目标：
- 理解[某个行为/态度]
- 挖掘[某个痛点/需求]
- 发现[机会点]

已知信息：
- [现有数据或假设]
```

**填写示例**：
```
业务背景：
- 我们的产品：健身App
- 当前问题：用户流失率上升至30%
- 想了解：为什么流失？如何挽回？

目标用户：
- 流失用户（使用3个月后不再活跃）

研究目标：
- 理解流失的真实原因
- 挖掘用户不满的痛点
- 发现挽回策略

已知信息：
- 数据显示：流失主要发生在第3-4个月
- 假设：可能是内容重复，缺乏新鲜感
```

---

### 模板3：市场分析Brief

```
市场背景：
- 我们想进入[市场/行业]
- 目标：[市场目标]

研究问题：
- 市场规模和增长潜力？
- 主要竞品和竞争格局？
- 目标用户需求和痛点？
- 进入机会和风险？

已知信息：
- [现有认知]

限制条件：
- 时间：[时间限制]
- 预算：[预算限制]
```

**填写示例**：
```
市场背景：
- 我们想进入老年健身市场
- 目标：开发适合老年人的健身产品

研究问题：
- 老年健身市场规模和增长潜力？
- 主要竞品和差异化空间？
- 老年用户的真实需求和痛点？
- 支付意愿如何？

已知信息：
- 老龄化趋势明显，老年人健康意识增强
- 市场上产品多为年轻人设计

限制条件：
- 时间：需要1周内完成
- 预算：中等
```

## Brief写作原则

### 原则1：说清背景和目标

✅ **好的Brief**：
```
"我们是健身App，流失率上升，想知道为什么流失，如何挽回"
```

❌ **不好的Brief**：
```
"做个研究"
```

---

### 原则2：包含关键信息

**必备信息**：
- 你的产品/服务是什么
- 遇到什么问题 or 想测试什么
- 目标用户是谁

**可选信息**：
- 竞品情况
- 已知数据
- 时间预算

---

### 原则3：聚焦核心问题

❌ **问题太多**：
```
想了解用户的年龄、性别、职业、收入、喜好、
购买渠道、使用频率、满意度、推荐意愿......（20个问题）
```

✅ **聚焦核心**：
```
想了解：
1. 用户对产品的接受度
2. 合适的定价策略
3. 如何优化产品
```

**建议**：核心问题控制在3-5个

---

### 原则4：开放而非封闭

❌ **封闭问题**：
```
"用户会买吗？"（只能回答是/否）
```

✅ **开放问题**：
```
"用户对这个产品的态度如何？什么因素影响购买决策？"
```

**原因**：开放问题能挖掘深层动机

## 常见错误和修正

### 错误1：过于简单

❌ **错误示例**：
```
Brief："测试产品"
```

✅ **修正**：
```
Brief："测试气泡咖啡新品，目标用户是25-35岁白领女性，
      想了解产品接受度和合适的定价策略"
```

---

### 错误2：过于复杂

❌ **错误示例**：
```
Brief：3000字的详细产品文档 + 30个细节问题
```

✅ **修正**：
```
Brief：500字概括核心背景和目标 + 3-5个核心问题
```

**原因**：AI会自动延伸细节，过于复杂反而影响理解

---

### 错误3：假设答案

❌ **错误示例**：
```
"用户肯定喜欢我们的产品吧？怎么提升销量？"
```

✅ **修正**：
```
"用户对我们产品的态度如何？是什么影响他们的购买决策？"
```

**原因**：不要假设结论，让研究发现真相

- 逐步增加细节

- 产品特性
- 目标用户
- 竞品情况
