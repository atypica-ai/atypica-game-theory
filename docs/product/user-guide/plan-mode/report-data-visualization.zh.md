## 为什么暂时没有可视化？

### 原因1：定性研究为主

**atypica的核心价值**：
- 深度挖掘"为什么"
- 理解用户动机和态度
- 提供洞察而非统计

**定性 vs 定量**：
- 定性：5-10人深度访谈，挖掘洞察
- 定量：100+人问卷，统计分布

**atypica偏定性**：
- 样本量：5-10人（不适合统计可视化）
- 输出：洞察和建议（文字更合适）

---

### 原因2：小样本量

**统计意义**：
- 5-10人的数据不适合做百分比统计
- "2/5人喜欢"比"40%喜欢"更准确
- 避免误导（小样本不代表总体）

**例外**：
- 如果是大规模测试（50+人）
- 未来可考虑可视化

---

### 原因3：洞察 > 数字

**重要的是洞察**：
```
❌ 数字：40%的用户觉得贵
✅ 洞察：价格敏感型用户觉得贵，
        但健康焦虑型用户愿意为0糖多付钱，
        关键是打消代糖疑虑。
```

**可视化的风险**：
- 过度关注数字
- 忽略背后的"为什么"

## 常见问题

### Q1：为什么不像问卷工具那样有图表？

**核心区别**：
- 问卷工具：定量研究（100+人），适合统计
- atypica：定性研究（5-10人），适合洞察

**类比**：
- 问卷：体检报告（血压、血糖数据）
- atypica：医生诊断（为什么血压高？怎么治疗？）

---

### Q2：5-10人的数据做图表有意义吗？

**统计意义有限**：
- 5人中2人 = 40%（听起来很多）
- 但样本太小，不能代表总体

**更合适的表达**：
- "5人中2人觉得贵"
- 而不是"40%觉得贵"

**建议**：
- 如需统计分析 → 使用问卷工具（100+人）
- 如需洞察挖掘 → 使用atypica（5-10人）

---

## 最佳实践

### 建议1：专注洞察而非数字

**不要问**：
- "多少百分比的用户喜欢？"

**要问**：
- "用户为什么喜欢/不喜欢？"
- "不同用户群体的差异在哪？"
- "如何优化产品？"

---

### 建议2：需要统计用问卷

**场景分工**：
- atypica：探索"为什么"（5-10人深度）
- 问卷工具：验证"多少人"（100+人统计）

**组合使用**：
1. atypica探索洞察
2. 问卷大规模验证
3. atypica深挖异常点

---

### 建议3：用文字讲故事

**好的报告**：
```
不是："60%用户接受¥18定价"

而是："5位用户中3位接受¥18，2位觉得偏贵。
      接受的用户都是健康焦虑型，认为0糖值得多花钱；
      觉得贵的是价格敏感型，平时买¥12的美式。
      建议定价¥15-16，在两者之间取平衡。"
```

**故事 > 数字**
