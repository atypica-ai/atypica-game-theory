# AI访谈创建的项目后怎么在公众人设库里选择最合适的AI人设？公众人设库应该输入什么标签来查找？

## 问题类型
用户手册问题

---

## 搜索策略

### 方法1：关键词搜索（最常用）

**输入内容**：
- 人群特征：年龄、性别、职业、城市
- 行为特征：健身、咖啡、护肤、理财
- 场景特征：通勤、办公、居家、社交

**示例**：
```
搜索："25-35岁 女性 白领 上海 咖啡"
结果：返回符合条件的人设

搜索："健身爱好者 增肌"
结果：返回关注健身和增肌的人设

搜索："价格敏感 网购"
结果：返回注重性价比、常网购的人设
```

---

### 方法2：标签筛选

**常用标签分类**：

**人口特征**：
- 年龄：18-25、25-35、35-45、45-55、55+
- 性别：男性、女性
- 职业：产品经理、程序员、设计师、学生、宝妈
- 收入：5-10K、10-20K、20-30K、30K+

**生活方式**：
- 健康派、效率控、社交达人、宅家族
- 早起型、夜猫子
- 运动爱好者、读书爱好者

**消费特征**：
- 价格敏感、品质优先、品牌控
- 早期采用者、谨慎观望者
- 网购为主、线下购物

**兴趣领域**：
- 科技、美妆、健身、美食、旅行
- 游戏、音乐、阅读、摄影

---

### 方法3：相似度搜索

**适用**：已有目标用户画像

**步骤**：
1. 输入目标用户描述（200字以内）
2. 系统自动匹配相似人设
3. 按相似度排序
4. 选择最接近的3-10个

**示例**：
```
输入描述：
"28岁产品经理，月入15K，住上海，注重性价比，
 喜欢在小红书看评价后购买，工作忙但追求生活品质"

系统返回：
- 张丽（相似度92%）
- 李悦（相似度88%）
- 王静（相似度85%）
```

## 选择建议

### 数量建议

| 研究类型 | 推荐数量 |
|---------|---------|
| **Interview（一对一）** | 5-10人 |
| **Discussion（群体讨论）** | 3-8人 |
| **大规模测试** | 50-100人 |

### 多样性建议

**目标**：覆盖不同类型用户

**示例：测试气泡咖啡**
```
选择5个人设：
- 2个价格敏感型（关注性价比）
- 2个品质优先型（关注成分和体验）
- 1个社交分享型（关注颜值和拍照）
```

## 常见问题

### Q1：搜索不到合适的人设怎么办？

**解决方案**：
1. **扩大范围**：放宽筛选条件
2. **使用Scout**：让社交媒体观察相关社交媒体，生成新人设
3. **创建自定义人设**：导入自己的用户访谈数据

### Q2：为什么搜到很多人设，怎么筛选？

**建议**：
1. 按相似度排序
2. 查看人设详情页，确认7维度信息
3. 聚类：将相似人设归为一类，每类选1-2个代表

### Q3：公众人设库的质量如何？

**质量保证**：
- ✅ 基于真实的研究背景构建（30万+真实研究问题）
- ✅ 基于真实社交媒体观察构建（每个人设3000字观察记录）
- ✅ 质量稳定，一致性达到80分（接近真人基准81分）
- ✅ 覆盖90%的常规研究场景
- ✅ 如需更高质量或模拟特定用户，可创建自定义人设（一致性85分）

**数据来源透明**：
- 社交媒体平台：小红书、抖音、Twitter、Instagram、TikTok
- 观察深度：10-15轮深度观察
- 数据规模：每个人设对应15次工具调用
