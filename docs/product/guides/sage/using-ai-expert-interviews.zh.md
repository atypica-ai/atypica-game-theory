# 怎么用AI专家智能体来访谈AI角色？

## 问题类型
用户手册问题

---

## 快速回答

**Sage（AI专家系统）不是用来"访谈AI角色"的。**

**Sage的真正用途**：
- ✅ 创建**可进化的AI专家顾问**（基于你的专业资料）
- ✅ 随时咨询专家（像咨询真人专家一样）
- ✅ 通过**补充访谈**让专家持续进化

**如果你想"访谈AI角色"**：
- → 使用**Interview**功能（AI Persona Interview）
- → 而不是Sage

**Sage vs Interview的区别**：
- Interview：你访谈AI人设（用户模拟），了解用户需求
- Sage：你咨询AI专家（专家模拟），获取专业建议

---

## Sage是什么？

### 核心定位：培养你的AI专家顾问

**不是**：
- ❌ 用来访谈用户的工具
- ❌ 用来做市场研究的工具

**而是**：
- ✅ 培养一位AI专家，基于你的专业资料
- ✅ 随时咨询这位专家，获取专业建议
- ✅ 通过对话和访谈，让专家持续进化

---

### Sage vs Interview对比

| 维度 | Sage（AI专家系统） | Interview（AI人设访谈） |
|------|------------------|---------------------|
| **角色** | AI专家顾问 | AI用户模拟 |
| **你的身份** | 咨询者 | 访谈者/研究员 |
| **对话目的** | 获取专业建议 | 了解用户需求 |
| **知识来源** | 你上传的专业资料 | atypica的30万+人设库 |
| **使用场景** | 企业知识传承、个人学习、行业研究 | 产品测试、市场研究、用户洞察 |
| **输出** | 专业建议和指导 | 用户反馈和洞察 |
| **进化方式** | 补充访谈填补知识盲点 | 不进化（人设固定） |

## Sage的完整使用流程

### Step 1：创建AI专家

**上传专业资料**：

**支持格式**：
- PDF、TXT、Markdown、Audio、DOCX（最多10个）
- 文本粘贴
- URL导入（自动抓取网页内容）

**示例场景**：企业UX设计专家

```
上传知识源：
1. design-system-v2.3.pdf（50页设计规范）
2. ux-principles.md（UX设计原则）
3. https://company.com/design-tokens（设计Token文档）

AI自动处理：
→ 提取纯文本：65,000字专业知识
→ 构建Memory Document（记忆文档）
→ 生成专家简介和推荐问题
```

---

### Step 2：AI分析并构建专家

**AI自动分析**：

```
使用Claude Sonnet 4.5分析知识源：
1. 提取专家简介（bio）
2. 识别专长领域（expertise）
3. 生成推荐问题
4. 构建结构化的Markdown记忆文档
5. 识别知识盲点（Knowledge Gaps）

输出：
✓ Memory Document v1已创建
✓ 专家简介：我是贵公司的UX设计专家，熟悉设计系统v2.3...
✓ 专长领域：设计系统规范、组件库使用、设计Token...
✓ 知识盲点：3个（移动端规范、动效规范、设计决策案例）
```

---

### Step 3：咨询AI专家（对话）

**像咨询真人专家一样**：

```
你："我们的按钮圆角应该设置为多少？"

Sage："根据我们的设计系统v2.3，按钮圆角统一使用6px。

      具体规则：
      - Primary Button: 6px圆角 + 16px内边距
      - Secondary Button: 6px圆角 + 12px内边距
      - Icon Button: 8px圆角（圆形按钮）

      这是为了与我们的卡片圆角（8px）和输入框圆角（4px）
      形成统一的视觉层次。"

→ 下次对话，Sage仍然记得设计规范，无需重复提供
```

---

### Step 4：补充访谈（让专家进化）

**当Sage发现知识盲点时**：

**触发条件**：
- 对话中发现无法回答的问题
- 初始分析识别的知识空白
- 用户主动创建补充访谈

**补充访谈流程**：

```
Sage发现知识盲点：
- Gap 1: 缺少移动端设计规范
- Gap 2: 动效规范不完整
- Gap 3: 缺少设计决策案例

你创建补充访谈：
→ AI访谈员生成访谈计划
→ AI访谈员向你提问
→ 你回答（扮演专家）
→ AI自动更新知识库（创建Memory Document v2）
→ 标记已解决的Knowledge Gaps

访谈示例：
AI："你好！我想补充一些关于移动端设计规范的知识。
    第一个问题：我们公司的iOS和Android设计规范有哪些核心差异？"

你："主要差异有三点：
    1. 导航模式：iOS用底部Tab Bar，Android用Bottom Navigation
    2. 返回按钮：iOS在左上角，Android用系统返回键
    3. 圆角：iOS偏好更大的圆角（8-12px），Android 4-6px"

AI："明白了。你提到iOS用底部Tab Bar，能详细说说它的设计规范吗？
    比如高度、图标大小、激活状态的样式？"

你："底部Tab Bar：
    - 高度：80px（包含安全区域）
    - 图标：24×24px，激活时颜色为primary-500
    - 文字：12px，激活时font-weight: 600
    - 间距：图标和文字间距4px"

[访谈结束后，AI自动更新知识库]

Memory Document v2已创建 ✓
- 新增内容：移动端设计规范（详细）
- 解决Gaps：3个
- 变更说明：补充iOS/Android移动端设计规范
```

## Sage vs NotebookLM

### 核心差异

| 维度 | Sage | Google NotebookLM |
|------|------|------------------|
| **核心定位** | 可进化的AI专家代理系统 | AI研究助手和知识管理工具 |
| **核心价值** | 构建持续学习的专业AI顾问 | 快速理解和利用已有文档 |
| **知识管理** | 版本化记忆文档 + 主动学习 | 基于上传内容的静态分析 |
| **学习机制** | 主动识别知识盲点并通过访谈补充 | 被动响应用户查询 |
| **独特功能** | Knowledge Gap追踪、补充访谈、知识演进 | Audio Overview（文档转播客）、Deep Research |

**Sage强调"培养专家"**：
- 知识是**动态演进**的
- 系统**主动发现**知识盲点
- 通过**结构化访谈**补全知识
- **完整的知识演进**追踪

**NotebookLM强调"理解材料"**：
- 知识是**静态的**（基于上传文档）
- **被动响应**用户查询
- **创新的输出形式**（播客、视频总结）
- **快速上手**，使用门槛低

## "访谈AI角色"的正确工具：Interview

如果你想**"访谈AI角色"来了解用户需求**，应该使用**Interview**功能：

### Interview功能介绍

**核心用途**：
- 访谈AI人设（用户模拟）
- 了解用户需求、痛点、决策过程
- 测试产品概念、定价、包装等

**使用流程**：

```
Step 1: 创建Interview项目
描述："测试气泡咖啡新品的接受度"

Step 2: 选择AI人设（从30万+人设库）
- Linda（28岁，产品经理，价格敏感）
- Emma（32岁，会计师，健康焦虑）
- Chloe（26岁，设计师，社交尝鲜）

Step 3: 开始访谈
你："你会购买¥18的气泡咖啡吗？"

Linda："可能不会，有点贵。我平时买¥12的美式。"

你："为什么觉得贵？"

Linda："我平时买¥12的美式，¥18超出预算了。"

你："如果是¥12，你会买吗？"

Linda："也不会，主要是气泡+咖啡听起来怪怪的..."

→ 深度挖掘用户需求和痛点
```

**区别**：
- Sage：咨询专家，获取专业建议
- Interview：访谈用户，了解用户需求

## 常见问题

### Q1：Sage能用来做市场研究吗？

**不能，Sage不是市场研究工具**：

**Sage的用途**：
- ✅ 创建专家顾问（基于你的专业资料）
- ✅ 咨询专家建议
- ❌ 不能做市场研究

**市场研究应该用**：
- 社交媒体观察（观察社交媒体）
- Interview/Discussion（访谈AI人设）
- Plan Mode（综合研究）

---

### Q2：Sage能访谈真实用户吗？

**不能**：

**Sage的角色**：
- AI专家顾问（你咨询它）
- 不是访谈工具

**访谈真实用户应该用**：
- AI Persona Interview（访谈AI人设）
- 或线下真人访谈

---

### Q3：Sage的"补充访谈"是什么意思？

**补充访谈的目的：填补Sage的知识盲点**

**不是**：
- ❌ Sage访谈用户
- ❌ Sage访谈其他AI

**而是**：
- ✅ AI访谈员向你提问
- ✅ 你回答（扮演专家）
- ✅ 补充Sage缺少的知识

**流程**：
```
Sage发现知识盲点：
"我的知识库中没有移动端设计规范"

你创建补充访谈：
AI访谈员："我们的iOS设计规范有哪些？"
你："iOS设计规范有以下几点..."

访谈结束后，Sage自动更新知识库
```

---

### Q4：Sage和Memory System有什么区别？

**完全不同的功能**：

**Memory System**：
- 记住你的研究需求和历史
- 个性化AI建议
- 自动提取和整理

**Sage**：
- 创建AI专家顾问
- 基于你上传的专业资料
- 随时咨询专家建议

**关系**：
- Memory：用户级记忆
- Sage：专家级记忆
- 双层架构，互补

---

### Q5：Sage适合什么场景？

**适合**：
- ✅ 企业知识传承（资深员工经验）
- ✅ 个人学习助手（课程笔记整理）
- ✅ 行业研究专家（持续跟踪行业）
- ✅ 专业咨询（法律、医疗、技术领域）

**不适合**：
- ❌ 市场研究（用Scout/Interview）
- ❌ 用户访谈（用Interview）
- ❌ 实时数据查询（如股票价格）
- ❌ 通用问答（ChatGPT更合适）
