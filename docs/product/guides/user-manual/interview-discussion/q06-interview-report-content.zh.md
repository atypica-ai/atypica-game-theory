# 访谈报告里有什么内容？在哪里查看？

## 问题类型
用户手册问题

---

## 快速回答

**访谈报告包含**：
- 访谈总结：每个人设的核心发现
- 精彩对话片段：作为证据
- 共识和分歧：自动识别
- 用户画像：7维度详细信息
- 改进建议：可执行的行动方案

**查看位置**：
- 访谈项目详情页 → "查看报告"
- 或首页 → "我的项目" → 选择项目 → "查看报告"

---

## Interview Chat报告结构

### 1. 执行摘要（Executive Summary）

**内容**：
- 研究目标
- 访谈人设数量
- 关键发现（3-5个核心洞察）

**示例**：
```
研究目标：测试气泡咖啡接受度和定价策略
访谈人设：5位（价格敏感型、健康焦虑型、社交尝鲜型）

关键发现：
1. 用户在"下午续命场景"不愿冒险，建议调整为早餐尝鲜场景
2. 0糖概念有吸引力，但需要打消代糖疑虑
3. 价格敏感点在¥15-18，避免超过¥20
```

---

### 2. 个人访谈总结（每人3000+字）

**结构**：

#### 用户画像
- 基本信息：姓名、年龄、性别、职业、城市
- 7维度详细信息：
  - 人口统计：收入、教育、家庭
  - 心理特征：价值观、生活方式
  - 行为特征：消费习惯、决策逻辑
  - 痛点和需求：真实痛点、期望
  - 技术接受度：对新产品的态度
  - 社交关系：社交圈影响
  - 地理位置：城市、居住环境

---

#### 核心发现

**示例**：Linda的访谈总结
```
核心发现：
1. 价格敏感：¥18超出日常预算（通常买¥12美式）
2. 场景限制：下午续命场景不愿冒险尝试新口味
3. 风险规避："如果失望了，一整天状态都毁了"
4. 尝试场景：早上或周末愿意尝试，能承担失望风险

关键洞察：
- 表面：价格贵
- 深层：下午续命场景下的风险规避心理
```

---

#### 精彩对话片段

**作为证据的原声对话**：

**示例**：
```
Q：你会购买¥18的气泡咖啡吗？
A：可能不会。

Q：为什么不会？
A：感觉有点贵，而且气泡+咖啡听起来怪怪的。

Q：怪怪的？你担心味道不好吗？
A：对，而且下午那个时间点，我没心情冒险。
   咖啡是续命的，不是尝鲜的。如果失望了，我一整天的状态都毁了。

Q：那什么场景下你会愿意尝试新口味？
A：早上或周末吧，那时候可以承担失望的风险。
```

**价值**：
- 真实对话，可信度高
- 可直接用于汇报和演示
- 展示深度追问过程

---

### 3. 跨用户分析

#### 共同点（共识区域）

**自动识别多个用户的共同观点**：

**示例**：
```
共识1：0糖概念有吸引力
- 5人中4人提到"0糖很健康"
- 但都担心代糖安全性

共识2：包装颜值很重要
- 5人中3人提到"如果好看会拍照发小红书"
- 社交价值影响购买决策

共识3：价格敏感点¥15-18
- 5人中4人表示"¥18可以接受但接近上限"
- 超过¥20会"犹豫不决"
```

---

#### 差异点（分歧区域）

**不同用户的对立观点**：

**示例**：
```
分歧1：价格接受度
- 价格敏感型：¥18太贵，预算¥12以内
- 品质优先型：¥18可以接受，关键看品质
- 社交尝鲜型：不在乎价格，能拍照就值

分歧2：尝试场景
- Linda：下午续命不愿冒险
- Chloe：任何时候都愿意尝试新口味

分歧3：关注点
- Emma：关注成分和健康
- Chloe：关注颜值和社交价值
- Linda：关注价格和性价比
```

---

#### 细分用户群体

**基于访谈自动聚类**：

**示例**：
```
群体1：价格敏感型（40%）
- 代表：Linda、Sarah
- 特征：预算有限，¥12-15区间
- 决策因素：性价比第一

群体2：健康焦虑型（40%）
- 代表：Emma、Mia
- 特征：关注成分和健康
- 决策因素：0糖、天然、无添加

群体3：社交尝鲜型（20%）
- 代表：Chloe
- 特征：关注颜值和新鲜感
- 决策因素：能拍照、有话题性
```

---

### 4. 可执行的改进建议

**基于洞察的具体行动方案**：

**示例**：
```
建议1：调整场景定位
- 问题：下午续命场景用户不愿冒险
- 行动：定位为"早餐活力场景"或"周末尝鲜场景"
- 预期效果：降低尝试门槛，提升转化率

建议2：打消代糖疑虑
- 问题：用户担心代糖安全性
- 行动：标注"天然甜味，无添加"，展示成分表
- 预期效果：提升健康焦虑型用户的信任度

建议3：优化包装设计
- 问题：社交尝鲜型用户关注颜值
- 行动：设计高颜值包装，适合拍照分享
- 预期效果：提升社交传播，增加购买意愿

建议4：价格策略
- 问题：¥18接近用户心理上限
- 行动：定价¥15-16，或推出¥12的小杯装
- 预期效果：覆盖更多价格敏感型用户
```

---

### 5. 附录

**完整访谈记录**：
- 每个人设的完整对话记录
- 访谈问题列表
- 访谈时长和日期

## Discussion Chat报告结构

### 1. 讨论总结

**内容**：
- 讨论主题
- 参与人设（3-8人）
- 讨论时长
- 核心观点汇总

---

### 2. 观点碰撞关键时刻

**精彩对话片段**：

**示例**：
```
【争议点：订阅制 vs 买断制】

小企业A：我更喜欢月付，灵活性高
中型企业B：我们更倾向年付，有折扣

AI主持人：为什么你们的偏好差异这么大？

小企业A：我们现金流紧张，月付不会给财务压力
中型企业B：我们有预算规划，年付更划算，减少采购频次

（关键洞察：企业规模决定了支付偏好）
```

---

### 3. 共识和分歧清单

**自动识别**：

#### 共识区域
```
共识1：功能强大是核心价值
- 所有6人都认同
- 但对"强大"的定义不同（小企业关注易用性，中型企业关注扩展性）

共识2：试用期很重要
- 所有6人都提到"需要试用后才能决定"
- 建议延长试用期到30天
```

---

#### 分歧区域
```
分歧1：支付方式偏好
- 小企业（3人）：倾向月付
- 中型企业（3人）：倾向年付

分歧2：价格敏感度
- 小企业：对价格极度敏感
- 中型企业：更关注ROI，不在乎价格
```

---

#### 争议点（讨论最激烈）
```
争议点："AI私教能替代真人教练吗？"
- 支持方（2人）：AI已经够用，真人太贵
- 反对方（3人）：AI不懂我的身体，不信任

（关键洞察：用户不是不接受AI，而是担心"AI不懂我"）
```

---

### 4. 观点演变追踪

**讨论前 vs 讨论后**：

**示例**：
```
AI人设A的观点演变：
- 讨论前："气泡咖啡听起来怪怪的，不会买"
- 讨论中：听到其他人说"其实还不错"
- 讨论后："听你们这么说，我可能会试试看"

（关键发现：社交讨论可以改变观点，口碑很重要）
```

---

### 5. 群体动态分析

**发言统计**：
- 每个人设的发言次数
- 发言时长占比
- 主导者 vs 沉默者

**影响力分析**：
- 谁的观点最有说服力？
- 谁被说服了？
- 关键说服时刻

---

### 6. 可执行的改进建议

**与Interview类似**，基于讨论发现提供行动方案。

## 报告生成时间

**自动生成**：
- Interview：所有访谈完成后5-10分钟
- Discussion：讨论结束后5-10分钟

**通知方式**：
- 系统通知
- 邮件通知（可选）

---

## 报告查看位置

### 方式1：项目详情页

**步骤**：
1. 首页 → "我的项目"
2. 选择访谈项目
3. 点击"查看报告"

---

### 方式2：直接链接

**访谈完成后**：
- 系统会提供报告直接链接
- 点击即可查看

---

### 方式3：报告列表

**步骤**：
1. 首页 → "报告中心"
2. 查看所有历史报告
3. 筛选、搜索、排序

---

## 报告导出

### 支持格式

**PDF**：
- 完整报告，包含所有内容
- 适合打印和分享
- 保持原始格式

**HTML**：
- 网页格式
- 适合在线查看
- 可用浏览器打开

---

### 导出步骤

1. 打开报告详情页
2. 点击"导出"按钮
3. 选择格式（PDF/HTML）
4. 下载文件

---

## 报告分享

### 团队内分享

**步骤**：
1. 报告详情页 → "分享"
2. 选择团队成员
3. 设置权限（只读/可编辑）

---

### 外部分享

**生成分享链接**：
- 有效期：7天/30天/永久
- 密码保护（可选）

---

## 常见问题

### Q1：报告生成失败怎么办？

**常见原因**：
- 访谈未完成
- 系统繁忙

**解决方案**：
- 刷新页面重试
- 联系客服

---

### Q2：报告可以修改吗？

**不可以直接修改系统生成的报告**。

**但可以**：
- 导出为Word格式
- 手动修改后分享

---

### Q3：报告会过期吗？

**不会**。所有历史报告都会保存。

**存储限制**：
- 免费版：最多保存10份
- 付费版：无限制

---

### Q4：如何找到历史报告？

**步骤**：
1. 首页 → "报告中心"
2. 使用搜索或筛选
3. 按日期、项目名称排序

---

---

### 建议2：重点看"可执行建议"

**报告最有价值的部分**：
- 可执行的改进建议
- 基于洞察的具体行动方案
- 预期效果评估

---

### 建议3：导出PDF存档

**重要报告建议**：
- 导出PDF格式

- 方便长期追溯
