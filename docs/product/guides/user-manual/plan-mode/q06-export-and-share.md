# How to export and share my research results?

## Question Type
User Manual Question

---

## Quick Answer

**Supports 2 export formats**:
- PDF: Complete report, suitable for printing and presentations
- HTML: Web format, suitable for online viewing and sharing

**Sharing Methods**:
- Team internal sharing (permission control)
- Generate share link (external sharing)
- Direct email sending

---

## Export Reports

### Step 1: Access Report Details Page

**Location**:
- Homepage → "My Studies"
- Select project → "View Report"

---

### Step 2: Select Export Format

**Click "Export" Button**

**Choose Format**:

#### Format 1: PDF (Recommended)

**Content**:
- Complete research report
- Executive summary
- Interview details
- Highlight conversation excerpts
- Actionable recommendations

**Use Cases**:
- ✅ Report to boss
- ✅ Team meeting presentation
- ✅ Print for archiving
- ✅ External sharing

**Advantages**:
- Fixed format, won't break
- Easy to read and print
- Compatible with all devices

---

#### Format 2: HTML

**Use Cases**:
- Online viewing
- Quick sharing
- Maintain complete format

**Content Included**:
- Complete report text
- All data tables
- Quotes and excerpts
- Opens in browser

**Steps**:
1. Report page → "Export"
2. Select "HTML format"
3. Save locally
4. Open with browser to view

---

### Step 3: Download File

**Generation Time**:
- PDF: 10-30 seconds
- HTML: 5-15 seconds

**File Size**:
- Usually 5-20MB
- Includes complete content

---

## Share Research

### Method 1: Team Internal Sharing

**Applicable**: Team members all have atypica accounts

**Steps**:
1. Report details page → "Share"
2. Select team members
3. Set permissions:
   - Read-only: Can only view
   - Can edit: Can modify and export

**Advantages**:
- Real-time sync
- Controllable permissions
- Convenient collaboration

---

### Method 2: Generate Share Link

**Applicable**: Share with external people

**Steps**:
1. Report details page → "Share"
2. Select "Generate Share Link"
3. Settings:
   - Validity: 7 days/30 days/permanent
   - Password protection (optional)
   - Allow download (optional)

**Get Link**:
```
https://atypica.ai/share/report/abc123
Password: 1234 (if set)
```

**Advantages**:
- No registration needed
- Access anytime
- Can set password protection

**Note**:
- Cannot modify permissions after link shared
- Automatically expires after expiration
- Recommend password protection for sensitive content

---

### Method 3: Direct Email Sending

**Steps**:
1. Report details page → "Share"
2. Select "Send Email"
3. Enter recipient email
4. Choose format: PDF/HTML
5. Send

**Advantages**:
- Delivers directly
- No login needed
- Attachment form

---

## Export/Share Scenarios

### Scenario 1: Report to Boss

**Recommended Method**: Export PDF

**Steps**:
1. Export PDF format report
2. Open to review, confirm content
3. Print or email to boss

**Tips**:
- Highlight core findings
- Prepare verbal explanation

---

### Scenario 2: Team Discussion

**Recommended Method**: Team internal sharing

**Steps**:
1. Share with team members (read-only permission)
2. Let everyone read before team meeting
3. Discuss core findings in meeting
4. Create action plan

**Tips**:
- Share in advance, give time to read
- Focus on actionable recommendations in meeting
- Record discussion results

---

### Scenario 3: External Partners

**Recommended Method**: Generate share link

**Steps**:
1. Generate share link (set 7-day validity)
2. Set password protection
3. Disable download (prevent redistribution)
4. Send link and password to partner

**Tips**:
- Must use password protection for sensitive content
- Set shorter validity period
- Remind not to screenshot and redistribute

---

### Scenario 4: Archive Backup

**Recommended Method**: Export PDF

**Steps**:
1. Export PDF (complete report)
2. Save to company server or cloud storage
3. Establish archive directory

**Tips**:
- File naming convention:
  `[Project Name]_[Date]_Research Report.pdf`
- Regular backups

---

### Scenario 5: Client Delivery

**Recommended Method**: Export PDF

**Steps**:
1. Export PDF format
2. Check content completeness
3. Send to client

**Tips**:
- Completeness: Check no missing content
- Professionalism: Confirm format standards

## Permission Management

### Team Member Permissions

**Read-only Permission**:
- ✅ View report
- ✅ Export PDF
- ❌ Modify content
- ❌ Share with others

**Edit Permission**:
- ✅ View report
- ✅ Export all formats
- ✅ Share with team members

**Admin Permission**:
- ✅ All edit permissions
- ✅ Manage share permissions
- ✅ Delete report
- ✅ Generate external share links

---

### Share Link Permissions

**Read-only Link**:
- ✅ View report
- ❌ Export
- ❌ Modify
- ❌ Reshare

**Downloadable Link**:
- ✅ View report
- ✅ Export PDF
- ❌ Modify
- ❌ Reshare

**Note**:
- Cannot modify permissions once share link generated
- Recommendation: Default to read-only, generate downloadable link when needed

## Common Questions

### Q1: Can exported reports be modified?

**Depends on Format**:
- PDF: Cannot modify directly (needs PDF editor)
- HTML: Can view in browser, but not recommended to modify directly

**Recommendation**:
- Need to share for viewing → Export PDF or HTML
- PDF more suitable for printing and archiving

---

### Q2: Will share links leak?

**Security Measures**:
- Set password protection
- Set validity period (7 days/30 days)
- Disable download

**If Worried About Leaks**:
- Don't generate permanent links
- Must set password
- Use team internal sharing (more secure)

---

### Q3: Can I share only partial content?

**Currently not supported**. Sharing is complete report.

---

### Q4: Can I still view after exporting?

**Yes**. Export doesn't affect online viewing.

**Online Report and Export File**:
- Completely independent
- Online report doesn't change
- Export file is snapshot

---

### Q5: What if share link expires?

**Regenerate**:
1. Report details page → "Share"
2. Generate new share link
3. Send to recipient

**Or**:
- Directly invite to register
- Team internal sharing (permanently valid)

## Best Practices

### Recommendation 1: Export Backup

**Important Project Backup**:
- PDF: Complete report (easy to read)
- Save to cloud storage

**Benefits**:
- Prevent data loss
- Convenient for long-term review

---

### Recommendation 2: Check Before Sharing

**Check Content**:
- Any sensitive information?
- Is format correct?
- Is content complete?

**Avoid**:
- Leak internal data
- Format disorder
- Information omission

---

### Recommendation 3: Establish Naming Convention

**File Naming**:
```
[Project Name]_[Date]_[Type].pdf

Examples:
Sparkling Coffee New Product Test_20260130_Research Report.pdf
User Churn Analysis_20260115_Interview Records.xlsx
```

**Benefits**:
- Easy to find
- Version control
- Team collaboration

---

### Recommendation 4: Minimize Permissions

**Principle**:
- Only give to necessary people
- Only give necessary permissions
- Set necessary protection

**Examples**:
- Team discussion → Read-only permission
- External sharing → Password protection + validity period
- Client delivery → Export PDF (cannot modify)

---

## Bottom Line

> "Export with PDF/HTML, share via team/link/email.
> Export backup for important reports, password protection required for external sharing."

**Remember**:
- ✅ Export formats: PDF (presentations/printing), HTML (online viewing)
- ✅ Sharing methods: Team internal (most secure), link (most convenient), email (most direct)
- ✅ Permission control: Read-only/Edit/Admin
- ✅ Security measures: Password protection + validity period
- ✅ Backup archiving: Export PDF + cloud storage
