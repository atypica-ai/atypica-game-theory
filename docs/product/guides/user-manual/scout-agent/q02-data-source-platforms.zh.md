## AI Research数据来源（非社交媒体）

### 核心来源：30万+ AI人设库

**AI人设不是从社交媒体实时抓取，而是提前构建好的**：

**人设来源**：
1. **真人访谈记录**：
   - 真实用户的访谈内容
   - 真实的态度、观点、决策过程
   - 转化为结构化的AI人设

2. **真实用户画像**：
   - 人口统计信息
   - 心理特征
   - 行为模式

3. **社交媒体观察生成**：
   - 社交媒体观察 → 生成自定义人设
   - 基于真实用户内容构建

**AI人设质量**：
- 自定义人设：一致性达到85分（超过真人基准81分，基于社交媒体观察/访谈记录构建）
- 公众人设库：一致性达到80分（接近真人基准81分，基于通用画像构建）
- 临时生成：快速验证用

---

### Interview/Discussion不依赖社交媒体

**区别**：

```
社交媒体观察：
- 实时观察社交媒体
- 提取真实用户内容
- 生成人群画像和AI人设

AI Interview/Discussion：
- 使用已有的AI人设库
- AI人设模拟真实用户反馈
- 不依赖社交媒体实时数据
```

**案例**：

```
场景：测试气泡咖啡新品

Step 1: 社交媒体观察（使用社交媒体）
- 观察小红书/抖音讨论气泡咖啡
- 生成人群画像 + 3个自定义人设 AI人设

Step 2: AI Interview（不使用社交媒体）
- 用3个AI人设进行深度访谈
- AI人设基于自身的人设背景回答
- 不会去查社交媒体的实时数据
```

## 数据隐私和安全

### 只观察公开内容

**社交媒体观察**：
- ✅ 只观察社交媒体上公开的内容
- ❌ 不访问任何私密信息
- ❌ 不访问账号私信或私密内容

**公开内容定义**：
- 小红书公开笔记
- 抖音公开内容
- 其他社交平台公开内容
- 其他平台的公开发布内容

---

### 不存储原始个人信息

**处理流程**：
```
Step 1: 观察公开内容
- 提取内容：用户发布的文字、图片（不含个人身份信息）

Step 2: AI分析
- 提取观点、态度、行为模式
- 不记录具体的用户ID或账号

Step 3: 生成洞察
- 输出：人群画像、行为模式
- 不输出：具体的用户身份信息
```

**示例**：

```
❌ 不会记录：
- "@张三 说：我喜欢喝咖啡"
- "小红书用户ID：123456"

✅ 会提取：
- "年轻女性用户讨论咖啡时，高频提到'颜值'和'拍照'"
- "价格接受度：¥25-28最优"
```

---

### 符合平台政策

**合规性**：
- ✅ 遵守各平台的爬虫政策
- ✅ 只访问公开API或公开页面
- ✅ 不违反任何平台的用户协议

---

## 实时性和数据新鲜度

### 实时观察

**社交媒体观察**：
- 观察最新的社交媒体内容
- 不是历史数据库
- 反映当前用户态度

**示例**：

```
2026年1月观察：
- 观察到"情绪价值"成为健康零食新卖点
- 这是最新趋势，行业报告可能还没覆盖

vs 传统行业报告：
- 通常滞后3-6个月
- 可能错过最新趋势
```

---

### 定期更新建议

**市场在变化**：
- 建议3-6个月重新社交媒体观察
- 发现趋势变化
- 及时调整策略

**案例**：

```
2025年10月观察：
- 健康零食用户关注"低糖低脂"

2026年1月重新观察：
- 用户开始关注"情绪价值"和"治愈感"
- 趋势变化：功能性健康 → 情绪健康

建议：
- 调整产品定位和营销策略
```

## 常见问题

### Q1：所有平台都会同时观察吗？

**不是**，根据任务选择：

**默认**：AI根据人群特征推荐2-3个平台
**手动指定**：可以指定特定平台

**示例**：

```
观察目标："25-35岁女性对健康零食的态度"

AI推荐：
- 小红书（主）：年轻女性聚集地
- 抖音（辅）：快速种草
- 抖音（可选）：快消品种草

原因：
- 小红书用户最匹配
- 抖音覆盖更广，种草效果好
- 抖音可以补充下沉市场视角
```

---

### Q2：可以只观察特定地域的用户吗？

**可以**：

**设置方式**：
```
观察目标："成都地区用户讨论咖啡消费"

Scout会：
- 聚焦成都地区的用户
- 分析地域特色
- 输出成都用户画像
```

**支持的地域筛选**：
- 一线城市（北京、上海、广州、深圳）
- 新一线城市（成都、杭州、武汉等）
- 省份（四川、浙江等）
- 海外地区（美国、欧洲等）

---

### Q3：社交媒体观察的内容会过期吗？

**会过期**：

**原因**：
- 社交媒体内容实时变化
- 用户态度随时间演变
- 市场趋势不断更新

**建议**：
- 3-6个月后重新Scout
- 或在关键节点（产品上市前、重大市场变化后）重新观察

---

### Q4：可以观察历史数据吗？

**当前不支持**：
- 社交媒体观察最新的实时内容
- 不支持指定历史时间段

**替代方案**：
- 如需历史趋势分析 → 使用传统的社交聆听工具
- Scout专注：当前用户态度和最新趋势

---

### Q5：AI Interview/Discussion会实时查社交媒体吗？

**不会**：

**AI人设回答基于**：
- 人设自身的背景和特征
- 不会实时查社交媒体

**示例**：

```
Interview中问AI人设："你会买¥18的气泡咖啡吗？"

AI人设（Linda，价格敏感型）：
- 基于人设背景回答："可能不会，有点贵。我平时买¥12的美式"
- 不会去查社交媒体上其他人怎么说

区别：
- Scout：观察社交媒体上真实用户讨论
- Interview：AI人设基于自身特征回答
```
