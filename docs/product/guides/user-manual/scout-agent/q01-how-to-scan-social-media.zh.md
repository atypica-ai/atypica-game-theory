# 如何让你们帮我观察社交媒体上的目标用户？

## 问题类型
用户手册问题

---

## 快速回答

**3步开始社交媒体观察**：
1. **创建Scout任务**：描述你想观察什么人群
2. **AI开始观察**：Scout自动在社交媒体上观察5-10轮
3. **查看输出**：人群画像 + 3个高质量自定义AI人设（可直接用于Interview）

**时间**：1-2天自动完成
**输出**：结构化人群画像 + 高质量AI人设

---

## 完整操作流程

### Step 1：进入社交媒体观察功能

**入口位置**：
- 首页 → "开始新研究"
- 或侧边栏 → "社交媒体观察"
- 或直接访问 `/scout/new`

---

### Step 2：描述观察目标

**输入框**："你想观察哪些用户？"

**示例输入**：

**例子1**：产品测试场景
```
"观察小红书上讨论气泡咖啡的用户，
了解他们对这类产品的态度"
```

**例子2**：新市场进入
```
"观察成都地区讨论咖啡消费的用户，
了解成都和北京用户的差异"
```

**例子3**：人群洞察
```
"观察25-35岁女性讨论健康零食的内容，
了解她们的消费心理和痛点"
```

**输入技巧**：

✅ **好的描述**：
- 明确人群：年龄、地域、特征
- 明确主题：讨论什么内容
- 明确目的：想了解什么

❌ **避免的描述**：
- 太宽泛："观察年轻人"（什么叫年轻？什么内容？）
- 太窄："观察买过我们产品的用户"（社交媒体上找不到特定用户）
- 太模糊："看看市场怎么样"（需要具体方向）

---

### Step 3：Plan Mode澄清（如需要）

**AI可能会问**：

**问题1**："想观察哪些平台？"
```
回答示例：
- "小红书和微博"
- "B站和抖音"
- "你推荐"（AI会根据人群特征推荐）
```

**问题2**："观察多少轮？"
```
回答示例：
- "10轮"（推荐，更全面）
- "5轮"（快速了解）
- "你决定"（AI会根据复杂度建议）
```

**问题3**："有什么特殊要求吗？"
```
回答示例：
- "重点关注价格敏感度"
- "想知道和竞品的对比态度"
- "没有"
```

```

```
正在推理用户特征...

1. 核心人群：25-32岁一线城市女性
2. 主要动机：社交分享 > 提神
3. 价格敏感度：¥20-30可接受

```

**阶段3：验证（继续观察）**
```

→ 验证结果：✅ 确认

→ 正在生成人群画像和AI人设...

**你可以**：
- 实时查看观察进度
- 中途暂停或调整
- 等待完成通知

**输出内容**：

```

- 性别：女性为主（约80%）
- 地域：一线城市
- 收入：中高收入（月入8K-20K）
```

**心理特征**：
```
- 愿意尝试新产品
- 注重生活品质
- 关注产品颜值
- 喜欢在社交媒体分享
```

**行为模式**：
```
- 购买场景：下午茶、逛街
- 决策因素：颜值 > 口味 > 健康
- 价格接受度：¥25-28最优，超过¥30犹豫
- 复购意愿：如果好喝会复购，但不是刚需

**痛点**：
```
- 担心"气泡感太强"
- 担心"咖啡味不够"
- 希望"包装好看"
```

```

## 高级设置（可选）

### 设置1：指定观察平台

**默认**：AI根据人群特征自动选择平台

**手动指定**：
```
平台选择：
- 小红书（年轻女性消费品）
- 微博（科技产品、热点话题）
- B站（年轻人、二次元、科技）
- 抖音（Z世代、短视频内容）
- X/Twitter（海外市场、科技圈）
- Reddit（海外市场、深度讨论）
```

**建议**：
- 不确定选哪个 → 让AI推荐
- 有明确目标人群 → 手动指定2-3个平台

---

### 设置2：观察轮次

**默认**：10轮（推荐）

**可调整**：
- 5轮：快速了解（适合时间紧急）
- 10-15轮：深度理解（推荐）
- 20轮+：不推荐（收益递减）

**如何选择**：
- 简单人群（如年轻女性消费品）→ 10轮
- 复杂人群（如企业决策者）→ 15轮
- 小众市场（如手办收藏）→ 15轮

---

### 设置3：聚焦维度

**默认**：全面观察

**可聚焦**：
```
维度选择：
- 价格敏感度（重点观察价格相关讨论）
- 使用场景（重点观察什么时候、哪里使用）
- 竞品对比（重点观察和竞品的对比态度）
- 痛点挖掘（重点观察抱怨和不满）
```

**示例**：
```
用户："观察气泡咖啡用户，重点关注价格敏感度"

Scout会：
- 优先观察讨论价格的内容
- 分析不同价格区间的接受度
- 输出详细的定价建议
```

## 实战案例

### 案例1：新产品测试（最常见）

**背景**：
某公司计划推出0糖气泡咖啡，想了解目标用户态度

**操作流程**：

**Step 1**：创建Scout任务
```
输入："观察小红书/微博讨论气泡咖啡的用户，
了解他们对这类产品的接受度和价格敏感度"
```

**Step 2**：AI自动观察10轮
```
耗时：1-2天
观察平台：小红书、微博、抖音
观察内容：用户自发讨论的气泡咖啡相关内容
```

**Step 3**：查看输出
```
人群画像：
- 25-32岁一线城市女性
- 关注颜值和社交分享
- 价格接受度¥25-28
- 担心气泡感太强

生成AI人设：3个自定义人设
→ 直接用于后续Interview测试产品概念
```

**价值**：
- 快速了解目标用户（1-2天）
- 基于真实用户内容，不是想象
- 生成可用的AI人设

---

### 案例2：区域扩展研究

**背景**：
北京咖啡品牌想进入成都市场

**操作流程**：

**Step 1**：创建Scout任务
```
输入："观察成都地区用户讨论咖啡消费的内容，
了解成都和北京用户的差异"
```

**Step 2**：AI观察成都用户
```
平台：小红书（成都地区）、大众点评（成都）
重点观察：
- 咖啡消费频次
- 价格接受度
- 偏好的咖啡类型
- 消费场景
```

**Step 3**：发现差异
```
成都 vs 北京差异：
1. 价格敏感度：成都更关注性价比
2. 消费场景：成都喜欢"坐着聊天"，北京快节奏外带
3. 产品偏好：成都偏好"茶咖结合"，北京纯咖啡
4. 消费频次：北京每天，成都每周2-3次

建议：
- 成都店需要更大空间（适合社交）
- 推出茶咖产品线
- 定价更亲民（¥20-25 vs 北京¥25-30）
```

**价值**：
- 避免"北京经验"直接复制
- 降低市场进入风险
- 精准定位区域策略

---

### 案例3：竞品用户研究

**背景**：
某健身App想了解竞品用户的真实态度

**操作流程**：

**Step 1**：创建Scout任务
```
输入："观察讨论Keep健身App的用户，
了解他们为什么选择Keep、有什么不满"
```

**Step 2**：AI观察竞品讨论
```
平台：小红书、微博、B站
观察内容：
- 用户为什么选择Keep
- 用户的使用体验
- 用户的抱怨和痛点
```

**Step 3**：发现机会
```
Keep用户画像：
- 核心用户：健身新手（占60%+）
- 选择原因："课程多"、"免费"
- 不满痛点："课程质量参差"、"广告多"、"缺少个性化"

差异化机会：
1. 聚焦中高级用户（Keep偏新手）
2. 提供高质量精选课程（vs Keep课程多但杂）
3. 个性化训练计划（Keep缺少）
4. 无广告体验（Keep广告多）
```

**价值**：
- 了解竞品用户真实态度
- 发现差异化机会
- 避免同质化竞争

## 常见问题

### Q1：社交媒体观察需要多久？

**时间**：
- 观察过程：1-2天（10轮观察 + AI推理）
- 可以后台自动进行，不需要人工参与

**如需加急**：
- 5轮观察：半天到1天
- 但深度和准确性会下降

---

### Q2：可以中途暂停或调整吗？

**可以**：
- 暂停观察：保存当前进度
- 调整方向：修改观察重点
- 增加轮次：如果觉得还不够深入

**示例**：
```
用户观察5轮后：
"我想增加5轮，重点观察价格敏感度"

Scout会：
- 保留前5轮的观察结果
- 再观察5轮，聚焦价格讨论
- 整合输出
```

---

### Q3：观察的内容是公开的吗？

**完全公开**：
- Scout只观察社交媒体上公开的内容
- 不访问任何私密信息
- 不涉及隐私问题

**数据来源**：
- 小红书公开笔记
- 微博公开讨论
- B站公开评论
- 其他社交平台公开内容

---

### Q4：生成的AI人设质量如何？

**质量等级**：高质量自定义人设
- 一致性高：接近真人表现
- 深度：基于真实用户内容构建
- 可用性：直接用于Interview/Discussion

**vs 临时生成**：
- 临时生成：一致性较低，反馈浅薄
- 自定义人设：一致性高，反馈真实

**建议**：
- 关键研究：用Scout生成的自定义人设
- 快速验证：可以用临时临时生成人设

---

### Q5：可以指定观察特定人群吗？

**可以，通过筛选条件**：

**示例1**：年龄筛选
```
"观察25-35岁女性讨论健康零食"

Scout会：
- 聚焦25-35岁女性发布的内容
- 排除其他年龄段
```

**示例2**：地域筛选
```
"观察成都地区用户讨论咖啡"

Scout会：
- 聚焦成都地区的用户
- 分析地域特色
```

**注意**：
- 不要筛选太窄（找不到足够内容）
- 不要筛选太宽（噪音太多）

---

### Q6：Scout发现的趋势可靠吗？

**可靠性**：
- ✅ 基于真实用户内容
- ✅ 多轮观察交叉验证
- ⚠️ 但仍是定性研究，不是统计显著性

**适用场景**：
- ✅ 快速发现趋势方向
- ✅ 理解用户态度和动机
- ❌ 不能替代大规模定量研究

**建议**：
- Scout发现趋势 → AI Interview验证 → (可选)大规模问卷确认

---

### Q7：可以观察竞品的用户吗？

**可以**：
- 观察讨论竞品的用户
- 了解竞品用户特征
- 发现差异化机会

**案例**：
```
"观察讨论星巴克的用户"

Scout发现：
- 高频词："方便"、"稳定"、"无功无过"
- 机会：用户觉得星巴克"没有惊喜"
- 定位方向："有特色的咖啡体验"
```

## 最佳实践

### 建议1：先Scout后Interview

**推荐流程**：
```
Step 1: 社交媒体观察10轮 → 生成人群画像
Step 2: 基于画像选择AI人设
Step 3: Interview深度访谈验证
```

**价值**：
- Scout确保方向正确
- Interview基于真实人群深挖
- 避免"对着空气访谈"

---

### 建议2：不要过度依赖单一平台

**建议**：
- 观察2-3个平台
- 交叉验证发现
- 避免平台偏差

**例子**：
- 小红书 + 微博 + B站
- 看到一致的趋势 → 可信度高
- 看到矛盾 → 需要进一步验证

---

### 建议3：定期重新观察

**市场在变化**：
- 3-6个月后重新Scout
- 发现趋势变化
- 及时调整策略

**价值**：
- 避免基于过时认知决策
- 抓住新兴机会

---

### 建议4：关注"为什么"而非"是什么"

**常见错误**：
- 只记录"用户喜欢A"
- 不深挖"为什么喜欢A"

**正确做法**：
- 观察时追问"为什么"
- 分析背后的动机
- 输出洞察而非现象
