# 怎么创建我自己的AI人设？

## 问题类型
用户手册问题

---

## 快速回答

**两种主要方式：上传访谈记录 或 使用社交媒体观察。**

**推荐选择**：
- ✅ **已有访谈数据**：直接上传访谈记录（支持TXT、PDF、Word、音频）
- ✅ **没有访谈数据**：使用社交媒体观察社交媒体上的目标人群

**简单说**：大多数情况下，上传访谈记录最快；如果想研究特定社交媒体人群，用社交媒体观察。

---

## 详细说明

### 方式1：上传访谈记录（推荐）

**适用场景**：
```
✅ 已有真人访谈记录
✅ 有客户访谈数据想复用
✅ 需要模拟特定真实用户
✅ 历史研究数据想要重复使用
```

**操作步骤**：

```
Step 1: 进入人设管理页面
→ 点击「创建人设」

Step 2: 选择「上传访谈记录」
→ 上传文件

Step 3: 选择文件类型
→ 支持格式：
  - 文字：TXT、PDF、Word、Markdown
  - 音频：MP3、M4A、WAV（自动转录）
  - 多个文件可批量上传

Step 4: AI自动分析
→ 提取关键信息（人口统计、价值观、消费习惯等）
→ 生成人设

Step 5: 预览并保存
→ 检查人设信息
→ 保存到「我的人设库」
→ 后续可用于任何研究
```

---

**文件要求**：

```
理想的访谈记录包含：
✅ 基本信息（年龄、职业、收入、城市）
✅ 生活方式和价值观
✅ 消费习惯和决策逻辑
✅ 痛点和需求
✅ 日常行为和态度

文件长度：
- 最少：1000字（基础人设）
- 理想：3000-5000字（高质量人设）
- 支持多个短访谈合并
```

---

**案例**：

**案例1：上传单个深度访谈**

```
场景：已做过1小时深度访谈，有录音和转录文字

操作：
Step 1: 上传访谈转录文档（5000字）
Step 2: AI分析并生成人设「张丽」
Step 3: 保存到人设库

人设内容包含：
- 基本信息：28岁，产品经理，月入2万，上海
- 价值观：追求工作生活平衡，注重健康
- 消费习惯：愿意为品质付费，偏好线上购物
- 痛点：工作压力大，需要便捷的健康解决方案

后续使用：
→ 用于测试健康零食新品
→ 验证包装设计偏好
→ 测试定价敏感度
```

**案例2：批量上传多个简短访谈**

```
场景：有10个客户的简短访谈记录（每个1000-2000字）

操作：
Step 1: 批量上传10个访谈文件
Step 2: AI分别生成10个人设
Step 3: 保存到人设库

价值：
→ 一次性创建10个真实客户人设
→ 后续可多次复用，无需重复访谈
→ 团队可共享使用
```

### 方式2：使用社交媒体观察

**适用场景**：
```
✅ 没有访谈数据
✅ 想研究社交媒体上的特定人群
✅ 公众人设库找不到的小众群体
✅ 需要了解最新用户行为
```

**操作步骤**：

```
Step 1: 进入社交媒体观察功能
→ 点击「创建新Scout任务」

Step 2: 描述观察目标
→ 例如："小红书上的露营爱好者"
→ 例如:"微博上关注环保的年轻人"

Step 3: Scout三阶段观察
→ 阶段1：观察（收集社交媒体内容）
→ 阶段2：推理（分析行为模式）
→ 阶段3：验证（确认人群特征）

Step 4: 生成人设
→ Scout完成后，自动生成人设
→ 保存到「我的人设库」

Step 5: 使用人设
→ 在Interview中选择该人设
→ 进行访谈和测试
```

---

**案例**：

**案例1：研究小众人群**

```
场景：想研究"咖啡爱好者"，但公众库中覆盖不足

操作：
Step 1: 创建Scout任务"观察小红书上的精品咖啡爱好者"
Step 2: 社交媒体观察3-5天
Step 3: 生成5-10个人设
Step 4: 保存到人设库

生成的人设特点：
- 基于真实社交媒体行为
- 反映最新的消费趋势
- 包含具体的品牌偏好和痛点

后续使用：
→ 测试咖啡新品
→ 验证定价策略
→ 测试营销文案
```

**案例2：跨平台观察**

```
场景：想全面了解"健身人群"在不同平台的差异

操作：
Step 1: 创建3个Scout任务
  - 小红书上的健身爱好者
  - B站上的健身Up主粉丝
  - 抖音上的健身内容消费者

Step 2: 分别观察和生成人设

Step 3: 对比分析不同平台用户的差异

价值：
→ 了解不同平台用户的特点
→ 针对性制定营销策略
→ 选择合适的投放渠道
```

---

### 方式3：手动创建（不推荐）

**为什么不推荐？**

```
手动创建的问题：
⚠️ 需要自己编写详细的人设信息
⚠️ 耗时长，效率低
⚠️ 质量依赖你的写作能力
⚠️ 可能缺少关键维度

相比之下：
✅ 上传访谈：AI自动提取，快速准确
✅ 社交媒体观察：基于真实数据，质量有保证
```

**仍然需要手动创建的场景**：
- 非常特殊的假设人设（如"未来用户"）
- 需要精确控制每个细节
- 教学或演示用途

## 创建后如何使用？

### 使用流程

```
创建人设后：

Step 1: 人设保存在「我的人设库」
→ 点击侧边栏「人设库」
→ 切换到「我的人设」标签
→ 查看所有自定义人设

Step 2: 在Interview中使用
→ 创建新的Interview项目
→ 在人设选择器中切换到「我的人设库」
→ 选择自定义人设
→ 开始访谈

Step 3: 多次复用
→ 同一个人设可用于多个研究
→ 一致性稳定，结果可对比
→ 边际成本极低
```

---

### 管理人设

```
人设管理功能：

✅ 查看：点击人设查看详细信息
✅ 编辑：修改人设的基本信息
✅ 删除：不需要的人设可删除
✅ 分享：团队版可分享给团队成员
✅ 标签：添加标签方便分类和搜索
✅ 备注：添加备注说明人设用途
```

详细管理方法见：[怎么管理和整理我的人设库？](./q06-管理人设库.md)

## 常见问题

### Q1：上传访谈记录需要什么格式？

**支持多种格式，无需特别处理。**

```
文字格式：
✅ TXT、PDF、Word、Markdown
✅ 直接粘贴文字也可以

音频格式：
✅ MP3、M4A、WAV
✅ 自动转录为文字后生成人设

访谈记录格式：
✅ 对话形式（Q&A）：推荐
✅ 叙述形式：也可以
✅ 混合形式：没问题

无需特别处理：
→ 不需要标准化格式
→ AI会自动识别和提取关键信息
```

---

### Q2：访谈记录太短怎么办？

**建议补充更多信息，或使用多个访谈。**

```
如果访谈记录<1000字：
⚠️ 信息可能不足
⚠️ 生成的人设可能不够详细

解决方案：
✅ 上传多个访谈记录（合并成一个人设）
✅ 先创建基础人设，后续补充访谈
✅ 使用社交媒体观察补充信息

推荐长度：
→ 最少：1000字
→ 理想：3000-5000字
→ 多个短访谈合并也可以
```

---

### Q3：可以批量创建人设吗？

**可以，支持批量上传。**

```
批量创建方式：

方式1：批量上传访谈文件
→ 一次上传多个访谈文件
→ 每个文件生成一个人设
→ 自动保存到人设库

方式2：批量社交媒体观察
→ 创建多个Scout任务
→ 同时观察不同人群
→ 分别生成人设

方式3：导入Excel（如支持）
→ 上传用户数据表格
→ 批量生成人设

适用场景：
→ 有大量历史访谈数据
→ 需要研究多个细分人群
→ 企业客户数据批量导入
```

详细操作见：[可以支持上传多个用户的访谈数据吗？](./q05-批量创建人设.md)

---

### Q4：创建的人设可以修改吗？

**可以，支持编辑和补充。**

```
修改方式：

方式1：直接编辑
→ 进入「我的人设库」
→ 点击人设 → 编辑
→ 修改基本信息、标签、备注

方式2：补充访谈
→ 上传新的访谈记录
→ 关联到现有人设
→ AI自动更新人设信息

方式3：重新生成
→ 如果原始数据不满意
→ 重新上传更好的访谈记录
→ 生成新的人设
```

---

### Q5：自定义人设会被其他用户看到吗？

**不会，完全私有。**

```
隐私保护：
✅ 只有创建者可见
✅ 其他用户无法搜索或使用
✅ 数据完全隔离

团队版例外：
→ 可以选择分享给团队成员
→ 需要手动设置共享权限
→ 默认仍是私有的
```

---

### Q6：创建人设需要额外付费吗？

**取决于创建方式和套餐。**

```
上传访谈记录：
→ 免费版/专业版：都可以上传
→ 无额外费用
→ 数量无限制

社交媒体观察：
→ 需要消耗Scout额度
→ 免费版：2次/月
→ 专业版：无限制

建议：
→ 优先使用已有访谈数据（免费）
→ Scout用于特殊需求（消耗额度）
```

---

### Q7：创建人设需要多长时间？

**取决于创建方式。**

```
上传访谈记录：
→ 上传文件：1-2分钟
→ AI分析：2-5分钟
→ 预览确认：1分钟
→ 总计：5-10分钟

社交媒体观察：
→ 设置任务：5分钟
→ 观察期：3-5天（后台运行）
→ 生成人设：5-10分钟
→ 总计：3-5天（但无需你盯着）

手动创建：
→ 填写信息：20-30分钟
→ 但质量可能不如其他方式
```
