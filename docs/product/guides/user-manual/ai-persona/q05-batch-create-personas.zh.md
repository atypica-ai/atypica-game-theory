# 可以支持上传多个用户的访谈数据，创建多个AI人设吗？

## 问题类型
用户手册问题

---

## 核心回答

**可以。支持批量导入，一次创建多个AI人设。**

---

## 批量创建方式

### 方式1：ZIP压缩包（多个访谈记录）

**适用**：已有多个访谈记录文件

**步骤**：
1. 准备访谈记录文件（TXT/PDF/Word）
   ```
   访谈记录/
   ├── 张丽_访谈记录.pdf
   ├── 李明_访谈记录.docx
   ├── 王悦_访谈记录.txt
   └── ...
   ```

2. 压缩为ZIP文件

3. 上传到atypica：
   - 进入Persona管理
   - 点击"批量导入"
   - 上传ZIP文件

4. 系统自动处理：
   - 为每个文件创建一个自定义人设
   - 显示处理进度
   - 完成后显示质量提示

5. 审核并确认：
   - 查看每个人设的信息完整度
   - 删除或补充信息不完整的人设
   - 保存到"我的人设库"

**限制**：
- 单次最多500个文件
- 每个文件不超过10MB
- ZIP总大小不超过100MB

---

### 方式2：Excel批量导入（结构化数据）

**适用**：CRM导出、问卷结果

**Excel模板**：
```
| ID | 姓名 | 年龄 | 性别 | 职业 | 城市 | 收入 | 价值观 | 消费习惯 | 痛点 |
|----|------|------|------|------|------|------|--------|----------|------|
| 001| 张丽 | 28   | 女   | PM   | 上海 | 15K  | 性价比 | 网购看评价 | 时间紧张 |
| 002| 李明 | 32   | 男   | 创业 | 北京 | 30K  | 效率  | 直接买贵的 | 选择困难 |
| ... |
```

**步骤**：
1. 下载Excel模板
2. 填写用户数据（支持50-500行）
3. 上传Excel
4. 系统批量生成自定义人设
5. 审核并确认

**质量建议**：
- 完整7维度信息 → 高质量人设（推荐）
- 只有基础信息 → 质量一般（建议补充信息）

## 处理时间

| 数量 | 方式1（访谈记录） | 方式2（Excel） |
|------|-----------------|---------------|
| 10个 | 约5-10分钟 | 约2-3分钟 |
| 50个 | 约20-30分钟 | 约5-10分钟 |
| 100个 | 约40-60分钟 | 约10-15分钟 |

**提示**：
- 处理期间可以离开，系统会发送邮件通知完成
- 可以查看处理进度

## 质量管理

### 信息完整度检查

系统会自动检查每个人设的信息完整度：
- **信息完整**：包含7个维度完整信息，质量稳定
- **信息部分完整**：缺少部分维度，建议补充
- **信息不完整**：只有基础信息，建议补充或删除

### 批量操作

**按完整度筛选**：
```
筛选：信息完整
结果：显示所有信息完整的人设

操作：
- 批量保存
- 批量补充信息
- 批量删除信息不完整的人设
```

## 实际案例

### 案例1：餐饮品牌会员研究

**需求**：测试新菜品

**操作**：
1. 从CRM导出500个会员数据（Excel）
2. 包含：姓名、年龄、性别、消费频次、偏好菜系
3. 批量上传
4. 生成500个自定义人设
5. 筛选信息完整的300个人设
6. 用于新菜品概念测试

**结果**：
- 反馈贴合真实客户
- 发现高频客户vs低频客户的需求差异

## 常见问题

### Q1：批量导入后，可以分组管理吗？

**可以**：
- 创建"文件夹"或"标签"
- 例如："2024Q1会员"、"VIP客户"、"流失用户"
- 方便后续查找和使用

### Q2：批量导入的人设可以修改吗？

**可以**：
- 单个修改：进入人设详情页编辑
- 批量修改：导出Excel，修改后重新上传

### Q3：批量导入失败怎么办？

**常见原因**：
- 文件格式不支持 → 转换为支持格式
- 文件损坏 → 重新导出
- 文件过大 → 分批上传

**系统会显示**：
- 成功：XX个
- 失败：XX个（附失败原因）
