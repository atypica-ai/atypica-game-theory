# 如何设置让AI记住我的研究需求？

## 问题类型
用户手册问题

---

## 快速回答

**Memory System自动工作，无需手动设置。**

**AI自动提取和记住**：
- ✅ 用户画像（行业、角色、公司）
- ✅ 研究偏好（喜欢的方法和风格）
- ✅ 研究历史（做过的研究和核心发现）
- ✅ 品牌定位（你的独特定位和差异化）

**你只需正常对话，AI自动学习和记忆。**

## Memory System如何工作？

### 自动提取：无需手动输入

**你不需要**：
- ❌ 填写"个人资料"表单
- ❌ 手动设置"研究偏好"
- ❌ 主动告诉AI"记住这个"

**AI自动从对话中提取**：

**第1次对话**：
```
你："我想了解健康零食市场的新趋势"

AI："好的，让我先了解一下背景：
    - 你是什么行业的？
    - 你的目标用户是谁？
    - 你关注哪个细分市场？"

你："我是某消费品公司的品牌经理，负责健康零食产品线。
    目标用户是25-35岁女性。"

→ AI自动提取并记住：
  - 行业：消费品 / 健康零食
  - 角色：品牌经理
  - 目标用户：25-35岁女性
```

**第5次对话**（AI已经记住）：
```
你："我想做一个情绪零食产品"

AI：（无需重复询问背景）
"基于你之前对健康零食赛道的关注，
 我发现情绪零食是一个新兴趋势。
 你的目标用户（25-35岁女性）对'治愈感'的需求很强..."

→ AI调用记忆：
  - 行业：健康零食
  - 目标用户：25-35岁女性
  - 研究历史：市场趋势分析
```

**价值**：
- 节省时间：不需要每次重复背景
- 自然对话：无需特殊格式或命令
- 零负担：正常对话即可

---

### 自动整理：保持记忆精简

**问题**：随着使用时间增长，记忆会越来越多

**解决方案**：AI自动整理

**整理机制**：
```
记忆超过阈值时（如100条）：
1. 去重：删除重复信息
   - "目标用户25-35岁女性"被提到10次 → 只保留1次

2. 压缩：合并相似记忆
   - 5次研究都提到"情绪价值" → 压缩为"核心关注：情绪价值"

3. 优先级：保留最重要的，删除过时的
   - 6个月前的临时决策 → 删除
   - 核心品牌定位 → 保留

4. 版本管理：可追溯历史
   - 保留完整的变更历史
```

**案例**：
```
未整理前（100条记忆）：
- 记忆1：目标用户是25-35岁女性
- 记忆5：我们关注25-35岁都市女性
- 记忆12：主要人群是25-35岁职场女性
- 记忆18：核心用户还是25-35岁女性
（重复4次）

自动整理后（30条核心记忆）：
- 核心目标人群：25-35岁都市职场女性（基于4次研究确认）
```

## 加速Memory建立的技巧

虽然AI自动学习，但你可以**加速**这个过程：

### 技巧1：初期多提供背景信息

**前5次对话，主动介绍自己**：

**好的示例**：
```
"我是某消费品公司的品牌经理，负责健康零食产品线。
 我们的目标用户是25-35岁都市女性，注重健康但不想太极端。
 我喜欢先快速验证方向，再深度研究。"

→ AI提取到6个关键信息：
  1. 角色：品牌经理
  2. 行业：消费品 / 健康零食
  3. 目标用户：25-35岁都市女性
  4. 用户特征：注重健康但不极端
  5. 研究偏好：快速验证优先
  6. 公司类型：消费品公司
```

**不好的示例**：
```
"我想做市场调研"

→ AI只知道：用户想做市场调研
→ 需要多轮对话才能建立完整认知
```

---

### 技巧2：明确说明你的品牌定位

**品牌定位是Memory的核心**：

**示例**：
```
你："我们的品牌定位是'好吃的健康零食 + 情绪治愈'，
    不是功能性健康零食，也不是极端健身零食。
    我们想让用户觉得'吃得健康又不内疚'。"

→ AI记住：
  - 品牌定位：好吃的健康 + 情绪治愈
  - 核心差异：情绪价值，而非功能性
  - 目标心理：吃得健康又不内疚
```

**价值**：
- AI后续建议都会基于这个定位
- 避免AI给出不符合定位的建议

---

### 技巧3：说明你的研究目标

**每次研究开始时，说明目标**：

**示例**：
```
你："我想测试新包装设计，
    希望包装能体现我们的品牌定位（情绪治愈），
    同时吸引25-35岁女性购买。"

→ AI记住：
  - 当前研究：包装设计测试
  - 核心目标：体现品牌定位 + 吸引目标用户
```

**价值**：
- AI的研究计划更精准
- AI的建议更符合你的目标

```

- 角色：产品经理、品牌经理、创业者、研究员...
- 公司规模：创业公司、中型企业、大型企业...
- 关注领域：健康食品、消费电子、SaaS...
```

**案例**：
```
第1次对话："我是某消费品公司的品牌经理，负责健康零食产品线"

Memory提取：
- 行业：消费品 / 健康零食
- 角色：品牌经理
- 产品线：健康零食
- 公司类型：消费品公司

第10次对话：
AI："作为健康零食品牌经理，你可能关注..."
（无需重复介绍）
```

---

### 2. 研究偏好

**记住你喜欢的工作方式**：

**记忆内容**：
```
- 研究方法偏好：喜欢Discussion快速验证，还是Interview深度挖掘？
- 报告风格：喜欢详细数据，还是简洁洞察？
- 人设偏好：更信任高质量自定义人设，还是公众人设库也可以？
- 节奏偏好：快速输出，还是深度研究？
```

**案例**：
```
第3次研究：用户多次选择"先Discussion后Interview"的流程

Memory学习：
- 用户偏好：快速验证 → 深度挖掘

第10次研究：
AI："基于你的研究习惯，我建议先用Discussion快速测试3个方向，
    再深度访谈最有潜力的方向"
```

---

### 3. 研究历史

**完整记录你的研究轨迹**：

**记忆内容**：
```
- 过往项目：做过哪些研究？
- 核心发现：每次研究的关键洞察是什么？
- 研究方向：从市场分析到产品验证到包装测试...
- 时间线：按时间排列的完整研究历史
```

**案例**：
```
用户的研究历史（Memory自动记录）：

2026-01：健康零食市场趋势分析
- 核心发现：情绪价值成为新卖点

2026-02：情绪零食产品创意验证
- 核心发现：用户愿意为"治愈感"付溢价

2026-03：包装设计测试
- 核心发现：精致 + 情绪标签的设计最受欢迎

2026-04（现在）：定价策略研究
- AI："基于你之前的发现（情绪价值、治愈感、精致包装），
       我建议定价在¥8-12，强调情绪体验而非健康功能..."
```

**价值**：
- AI理解你的完整研究上下文
- 新研究可以基于旧发现，不是从零开始
- 形成知识积累，而不是碎片化研究

**记忆内容**：
```
- 如果你多次研究"年轻女性消费者"，AI知道这是你的核心人群
- 如果你反复提到"情绪价值"，AI理解这是你的品牌定位
- 如果你常问"25-35岁用户"，AI记住这是你的目标年龄段
```

**案例**：
```
用户在5次研究中都提到"25-35岁都市女性"

Memory识别：
- 核心目标人群：25-35岁都市女性

第6次研究：
AI：（自动使用25-35岁都市女性作为默认人群，无需再问）
"我将基于你的核心目标人群（25-35岁都市女性）进行研究..."
```

### 5. 品牌和产品定位

**理解你的核心定位**：

**记忆内容**：
```
- 品牌调性：健康、治愈、活力、高端...
- 产品特点：功能性、情绪价值、性价比...
- 差异化策略：和竞品的核心差异是什么？
```

**案例**：
```

    我建议产品包装强调'吃得开心又不内疚'，
```

## 查看和管理Memory

### 查看当前记忆

**入口**：
- 账号设置 → "Memory"标签
- 或直接访问 `/settings/memory`

**查看内容**：
```
核心记忆（Core Memory）：
- 用户画像
- 品牌定位
- 核心目标人群
- 研究偏好

工作记忆（Working Memory）：
- 最近研究历史
- 临时决策
- 当前关注点
```

---

### 手动编辑记忆（可选）

**如果AI理解错误，可以手动修正**：

**步骤**：
1. 进入Memory管理页面
2. 点击"编辑核心记忆"
3. 修改或补充内容
4. 保存

**示例**：
```
AI记住：
- 目标用户：25-35岁女性

你想修正为：
- 目标用户：25-40岁女性（扩展年龄段）

手动编辑后：
- AI后续建议会基于25-40岁女性
```

**建议**：
- 一般情况不需要手动编辑
- 只在AI明显理解错误时修正
- 或在战略方向大调整时更新

---

### 删除记忆（可选）

**场景**：
- 品牌策略调整：删除旧定位的记忆
- 换了目标市场：删除旧市场的记忆
- 隐私考虑：删除敏感信息

**步骤**：
1. 进入Memory管理页面
2. 选择要删除的记忆
3. 点击"删除"
4. 确认删除

**批量清除**：
- 删除某个时间段的所有记忆
- 删除某个主题的所有记忆
- 一键清空所有记忆（重新开始）

## 团队共享：团队级Memory

### 个人版 vs 团队版

**个人版**：
- 只有你自己的记忆
- 适合个人用户、独立研究员

**团队版**：
- 整个团队共享同一个Memory
- 所有人的研究都会累积到团队记忆中
- 新人加入无需重复介绍

---

### 团队Memory的价值

**场景1：新人入职**

**传统方式**：
```
新人："我们是什么定位？目标用户是谁？"
需要老员工花时间解释
新人还是不能完全理解
```

**Memory System（团队版）**：
```
新人直接问AI："我们的品牌定位是什么？"

AI："基于团队过去6个月的30次研究，你们的品牌定位是：
    - 好吃的健康零食 + 情绪治愈价值
    - 目标用户：25-35岁都市女性
    - 核心差异：情绪治愈，而非功能性健康
    - 定价策略：中高端（¥8-12）
    - 核心发现：用户愿意为情绪价值付溢价"

新人5分钟理解完整背景，无需老员工解释
```

---

**场景2：跨部门协作**

**传统方式**：
```
市场部做了用户研究
产品部不知道研究结果
重复做研究，浪费时间
```

**Memory System（团队版）**：
```
市场部完成研究 → 自动记录到团队Memory

产品部问AI："用户对我们产品的真实态度是什么？"

AI调用市场部的研究结果，直接回答

避免重复研究
```

---

**场景3：长期知识积累**

**传统方式**：
```
研究报告散落在各处
几个月后忘记之前的发现
新研究无法基于旧发现
```

**Memory System（团队版）**：
```
所有研究自动累积到团队Memory

AI理解团队的完整研究历史

新研究可以基于旧发现，形成知识网络
```

---

### 团队Memory的隐私保护

**隔离机制**：
- 每个团队的Memory完全隔离
- 团队A看不到团队B的记忆
- 个人用户看不到团队用户的记忆

**权限管理**：
- 团队管理员可以控制哪些成员可以访问Memory
- 可以设置只读权限（查看但不能修改）
- 成员离职可以撤销访问权限

## 常见问题

### Q1：Memory会影响AI的客观性吗？

**不会，Memory只提供上下文，不改变判断逻辑**：

**案例1**：
```
用户记忆："我们认为产品A很好"
新研究发现：用户不喜欢产品A

AI回应：
"虽然你们之前认为产品A很好，
 但根据本次研究，用户反馈显示产品A存在以下问题..."

→ AI不会因为Memory而改变客观判断
```

**案例2**：
```
用户记忆："目标用户是25-35岁女性"
新研究范围：40-50岁用户

AI回应：
"注意：本次研究对象（40-50岁）与你们的核心目标人群（25-35岁）不同。
 研究结果可能不能直接应用于核心人群..."

→ Memory帮助AI提供更准确的上下文提醒
```

---

### Q2：Memory会不会太多，导致AI混乱？

**不会，有自动整理机制**：

**触发条件**：
- 记忆超过100条
- 距离上次整理超过30天
- 用户可以随时手动整理

**整理内容**：
- 去重：删除重复的记忆
- 合并：将相似记忆合并
- 压缩：提取核心信息，删除冗余细节
- 归档：将过时但可能有用的记忆归档

**效果**：
- 记忆保持精简高效
- 关键信息不会被淹没
- AI响应速度不受影响

---

### Q3：可以导出Memory吗？

**可以，支持导出**：

**导出格式**：
- Markdown文件（适合阅读）
- JSON文件（适合程序处理）
- PDF报告（适合分享）

**导出内容**：
- 完整记忆列表
- 按类别整理（用户画像、研究偏好、研究历史...）
- 包含版本历史

**使用场景**：
- 向老板汇报：导出研究历史
- 团队会议：导出团队共识
- 备份：定期导出记忆备份

---

### Q4：如果换了工作，Memory怎么办？

**个人记忆 vs 团队记忆分开处理**：

**个人记忆**：
- 跟着你的账号走
- 换工作后依然保留
- 可以选择删除（如果不想保留旧公司信息）

**团队记忆**：
- 属于团队，不属于个人
- 离职后自动失去访问权限
- 不会带走团队记忆

**隐私保护**：
- 离职时可以选择"清空个人贡献的记忆"
- 只删除你个人添加的内容，不影响其他人

---

### Q5：多久会触发一次自动整理？

**动态触发机制**：

**触发条件**：
1. 数量阈值：记忆超过100条
2. 时间阈值：距离上次整理超过30天
3. 手动触发：用户可以随时手动整理

**整理流程**：
```
整理前：AI会提醒
"你的记忆即将整理，预计压缩30%内容"

整理后：AI会总结
"已整理完成，从100条压缩到70条核心记忆"
```

## 最佳实践

### 建议1：初期多提供背景信息

**前5次对话，主动介绍**：
- 行业、角色、目标
- 品牌定位和差异化
- 研究偏好

**价值**：
- 让Memory快速建立认知
- AI建议更精准

---

### 建议2：定期查看记忆，确保准确

**建议频率**：每月查看一次

**检查内容**：
- AI记住的信息是否准确？
- 有没有过时的信息需要删除？
- 有没有新的核心信息需要强调？

**价值**：
- 确保AI理解是准确的
- 及时纠正偏差
- 提升建议质量

---

### 建议3：团队版定期对齐Memory

**问题**：团队成员可能对某些概念理解不一致

**建议**：每季度开一次"记忆对齐会"

**流程**：
1. 导出团队Memory
2. 团队一起审核
3. 讨论不一致的地方
4. 统一认知并更新Memory

**价值**：
- 确保团队认知一致
- Memory成为"团队共识"的载体

---

### 建议4：不要过度依赖Memory

**Memory的作用**：
- ✅ 提供上下文，节省时间
- ✅ 个性化建议
- ❌ 不替代独立思考

**建议**：
- 每次重要决策，重新验证假设
- 不要因为"Memory中是这样"就不去验证
- 市场在变，Memory可能过时

**案例**：
```
Memory："目标用户是25-35岁女性"

6个月后，市场变化，Z世代成为新机会

正确做法：
- 用社交媒体观察重新观察市场
- 验证Z世代是否真的是机会
- 更新Memory："扩展到Z世代"

错误做法：
- 因为Memory中是"25-35岁"就不看Z世代
```
