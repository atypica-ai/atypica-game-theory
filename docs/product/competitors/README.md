## 文档结构

每个竞品对比文档包含：

1. **为什么要对比**
   - 表面相似性
   - 用户困惑场景
   - 核心差异预览

2. **产品定位差异**
   - 核心价值
   - 典型工作流
   - 关键特点

3. **详细功能对比**（5-7 个维度）
   - 研究流程
   - 核心能力
   - 分析深度
   - 输出形式
   - 等等

4. **5 个典型场景对比**
   - 每个场景包含：时间、成本、质量分析
   - 结论和建议

5. **核心优劣势分析**
   - 各产品的优势（3-5 点）
   - 各产品的局限（3-5 点）

6. **使用场景建议**
   - 什么时候用哪个产品
   - 清晰的使用边界

7. **组合使用策略**
   - 如何配合使用两个产品
   - 具体工作流程
   - 成本和价值分析

8. **10 个常见问题**
   - 用户真实疑问
   - 详细解答
   - 决策建议

9. **成本对比分析**
   - 订阅费用对比
   - ROI 计算
   - 隐性成本分析

10. **总结和选择建议**
    - 核心差异表格
    - 选择建议
    - 最佳实践

## 使用建议

### 销售场景
当客户问"为什么不用 XX"时：
- 打开对应竞品文档
- 查看"核心差异"和"使用场景建议"
- 重点讲解"组合使用策略"（不是替代，而是互补）

### 产品定位
- 用于理解 atypica.AI 的差异化价值
- 明确哪些场景是我们的优势
- 哪些场景不适合（建议用户使用竞品）

### 市场教育
- 帮助用户理解不同工具的定位
- 避免用户选错工具导致失望
- 建立专业形象（诚实推荐合适工具）

### 决策支持
- 帮助用户评估需求
- 提供组合使用方案
- 计算 ROI 和成本

## 竞品对比策略

### 核心原则
1. **不贬低竞品**：客观说明差异，尊重竞品价值
2. **明确定位**：清晰说明各自适用场景
3. **推荐组合**：主动建议如何配合使用
4. **诚实建议**：某些场景直接推荐竞品

### 差异化重点
- **Claude Projects**：通用 vs 专业（垂直化价值）
- **NotebookLM**：转化 vs 创造（主动研究能力）
- **Otter.ai**：记录 vs 理解（深度洞察能力）
- **UserTesting**：真人 vs AI（速度和成本优势）
- **SurveyMonkey**：量化 vs 质性（深度理解能力）
