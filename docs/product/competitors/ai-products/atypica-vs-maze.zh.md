## atypica的独特价值

### 1. 验证方向，不只是测试原型

**Maze的前提**：产品方向已确定，测试原型好不好用。

**atypica的能力**：
- 在设计原型前验证需求
- 避免设计"好用但没人要"的功能

**案例**：
产品团队有3个功能方向。

**用Maze**：
- 制作3个原型（1周）
- 测试可用性（3天）
- 结果：功能C可用性最好（90%完成率）
- → 设计功能C

**用atypica**：
```
深度访谈发现：
- 功能C：可用性好，但需求弱（3/10）
  "几乎不导出数据，给我这个功能干嘛？"

- 功能A：可用性未知，但需求强（9/10）
  "这是我最需要的！我愿意多付30%。"

→ 应该做功能A（强需求）
→ 避免浪费2周设计+4周开发功能C
```

**区别**：Maze告诉你"哪个原型好用"，atypica告诉你"该不该做这个功能"。

---

### 2. 开发前即可研究

**Maze需要**：已有原型（Figma/Sketch）才能测试

**atypica可以**：
- 只需要一个想法就能研究
- 无需设计原型
- 无需写代码

**价值场景**：
- 测试50个产品概念，筛选最有潜力的5个
- Maze：需要制作50个原型（8-10周）
- atypica：直接访谈验证（3-5天）

**节省**：8周设计时间 + 避免错误方向

---

### 3. 理解深层"为什么"

**Maze的输出**：
```
任务：完成购买
- 成功率：65%
- 热图：40%用户先点"收藏"（误以为是购买）
- 建议：调整按钮位置和样式
```

**atypica的输出**：
```
深度访谈发现：
不是"找不到按钮"，而是"不确定要不要买"

用户A："按钮位置无所谓，关键是没有尺码推荐。
       我不知道是否适合我，不敢下单。"

用户B："不是找不到购买按钮，是看到运费$15
       立刻放弃了。商品$30，运费一半？"

→ 真正的问题：信任缺失 + 定价策略
→ 优化方向：
  1. 增加尺码推荐+用户评价
  2. 满$50包邮

预期：转化率从5%提升到12%
```

**区别**：Maze发现表层问题，atypica找到根本原因。
