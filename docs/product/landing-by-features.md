## 10 大核心功能

### 🎯 1. Plan Mode - 让 AI 先想清楚再动手

**一句话**：自动生成完整研究计划，展示方法和步骤，一键确认开始执行。

**解决什么问题**：
- ❌ 传统 AI：直接执行，用户不知道它会怎么做
- ✅ Plan Mode：先展示计划（调研谁、问什么、用什么方法），你满意了再开始

**适用场景**：
- 初次使用 atypica，不确定如何提需求
- 复杂研究项目，需要确认研究路径
- 团队协作，需要让同事看到研究计划

**真实案例**：
> "气泡咖啡产品创意"需求 → Plan Mode 生成 4 步计划（定义目标用户 → 讨论口味偏好 → 测试包装设计 → 市场定位建议）→ 用户确认 → 自动执行访谈和讨论 → 输出完整报告

**5 秒理解**：像有个研究顾问先帮你设计方案，你点头了再干活。

### 💬 2. Interview - 一对一深度访谈

**一句话**：与 5-10 个 AI 人设进行深度对话，挖掘未表达需求和深层动机。

**核心优势**：
- **真实行为模式**：基于 7 维度人设数据（人口统计、心理、行为、技术接受度等）
- **自然对话**：支持追问、澄清、探索，不是填问卷
- **即时执行**：选好人设立即开始，无需等待招募

**vs. Discussion**：
- Interview：1v1 深度，适合探索个人动机（如"为什么选择这个品牌"）
- Discussion：3-8 人群体，适合观点碰撞（如"气泡咖啡 vs 传统咖啡哪个更有前景"）

**典型流程**：
1. AI 自动推荐人设（如"25-34 岁一线城市白领女性，注重健康"）
2. 你审核或调整人设筛选条件
3. 自动生成访谈提纲并执行
4. 获得访谈记录 + 洞察总结

**成本对比**：
- 传统访谈：5 人 × $200/人 = $1,000 + 2 周招募
- atypica.AI：数小时完成，包含在订阅费用中

### 👥 3. Discussion - 群体讨论碰撞观点

**一句话**：3-8 个 AI 人设同时讨论，模拟焦点小组，激发新洞察。

**为什么需要群体讨论**：
- 观点碰撞产生新想法（"我觉得气泡咖啡太甜" → "但这正是年轻人喜欢的"）
- 看到不同人群的分歧点（健康派 vs 口味派）
- 揭示隐性共识（大家都提到包装要有设计感）

**Discussion 独有能力**：
- 自动控制讨论节奏，防止跑题
- 平衡发言权，避免某个角色主导
- 提取共识和分歧，生成结构化洞察

**适用场景**：
- 新产品概念测试："大家觉得这个产品怎么样？"
- 市场定位选择："A 定位和 B 定位哪个更吸引你？"
- 品牌形象探索："你们对这个品牌的第一印象是什么？"

**真实案例**：
> 气泡咖啡讨论，8 人 45 分钟 → 发现：健康概念不如"轻松社交饮料"定位受欢迎，包装需要年轻化设计，价格敏感度比预期低

### 🔍 4. Scout Agent - 社交媒体深度观察

**一句话**：3 阶段观察社交媒体内容，构建目标人群画像和行为模式。

**三阶段工作流**：
1. **观察（5 轮）**：分析社交媒体内容（小红书/微博/X/Reddit 等），提取用户行为
2. **推理（自动触发）**：5 次观察后，用 reasoningThinking 工具深度分析，形成假设
3. **验证（继续观察）**：带着假设继续观察，修正认知

**vs. 直接问 AI**：
- 直接问：基于 AI 预训练知识，可能脱离实际
- Scout Agent：基于真实社交内容，发现最新趋势

**输出**：
- 目标人群画像（人口统计 + 心理特征 + 行为模式）
- 可直接用于后续 Interview/Discussion 的 Tier2/Tier3 人设

**适用场景**：
- 新市场进入前，理解目标用户
- 发现新兴消费趋势
- 品牌定位研究

**技术亮点**：
- 自动判断何时触发推理（5 次观察后）
- 提取 7 维度人设数据，确保后续研究的人设质量

### ⚡ 5. Fast Insight - 播客优先自动化

**一句话**：从研究需求到播客内容，一条龙自动化，几小时完成。

**标准工作流**：
需求 → 自动规划 → 网络搜索 → 生成报告 → 生成播客脚本（无需人工干预）

**vs. 常规研究（Study Agent）**：
- Study Agent：灵活度高，支持 Interview/Discussion/Scout 等深度工具，适合复杂项目
- Fast Insight：速度优先，自动化程度高，适合快速输出播客内容

**核心价值**：
- **内容创作者**：快速产出播客素材
- **品牌团队**：快速响应热点，生成市场洞察播客
- **咨询顾问**：快速输出客户可听的报告

**真实案例**：
> "OpenAI GPT-5 对市场的影响" → 自动搜索最新新闻 → 生成 5000 字报告 → 生成 15 分钟播客脚本（双人对话格式，包含开场、核心观点、案例、结尾）

**输出格式**：
- Markdown 报告（结构化洞察）
- 播客脚本（主持人 A + 嘉宾 B 对话）
- 可直接用于 TTS 生成音频

**适用场景**：

**vs. 传统头脑风暴**：

### 🎭 7. AI Persona System - 三层人设体系

**一句话**：30 万+ AI 人设库，从普通 AI 到真人级模拟，保证研究质量。

**三层品质体系**：
| Tier | 数量 | 特点 | 适用场景 |
|------|------|------|---------|
| **Tier 0** | 无限 | 临时生成 | 快速验证想法 |
| **Tier 1** | ~30 万 | 基础 7 维度 | 常规研究 |
| **Tier 2** | ~1 万 | 真人级模拟（85 点一致性） | 关键决策 |
| **Tier 3** | 用户自定义 | 私有人设 | 特定项目 |

**品质保证**：
- **7 维度评分系统**：人口统计、地理、心理、行为、痛点、技术接受度、社交关系
- **一致性测试**：Tier2 人设在重复测试中保持 85% 一致性（vs 真人 81%）

**三种构建方式**：
1. **Scout Agent 生成**：社交观察自动生成高质量 Tier2 人设
2. **导入真实数据**：团队已有用户画像可导入
3. **手动创建**：精细定义特定人设（Tier3）

**为什么重要**：
- 低质量人设 → 无意义的"AI 幻觉"反馈
- 高质量人设 → 接近真人的洞察

### 🧠 8. Memory System - 越用越懂你

**一句话**：双层持久记忆，自动提取用户偏好和研究历史，AI 无需重复问。

**双层架构**：
- **Core Memory**：长期稳定记忆（Markdown 格式），如"用户所在行业：消费品"
- **Working Memory**：短期工作记忆（JSON 格式），如"本周关注气泡咖啡项目"

**自动提取 5 类信息**：
1. **Profile（用户画像）**：行业、角色、公司规模
2. **Preference（偏好）**：喜欢的研究方法、报告风格
3. **ResearchHistory（研究历史）**：过往项目、核心发现
4. **RecurringTheme（反复主题）**：持续关注的议题
5. **UnexploredInterest（未探索兴趣）**：提到但未深入的方向

**自动重组**：
- 当记忆超过 8000 tokens → 自动压缩去重
- 保留最重要信息，删除过时内容
- 版本管理，可追溯历史

**用户体验提升**：
- 第 1 次对话：需要介绍背景（"我是做消费品的"）
- 第 5 次对话：AI 直接说"基于你之前关注的健康食品赛道..."
- 第 20 次对话：AI 理解你的品牌定位偏好和目标用户特征

### 📚 9. Sage - 可进化的领域专家

**一句话**：上传文档/网页，AI 自动构建领域专家，可持续对话和进化。

**完整生命周期**：
1. **Create**：上传知识源（PDF/网页/文本）
2. **Extract**：自动提取知识 → 生成 Memory Document（类似 CLAUDE.md）
3. **Chat**：基于知识库对话，提供专业建议
4. **Discover Gaps**：AI 分析对话，发现知识盲区
5. **Evolve**：针对盲区提问专家（你），更新 Memory Document

**vs. 普通 RAG**：
- 普通 RAG：检索相关片段 → 回答
- Sage：理解整体知识 → 发现不足 → 主动进化

**适用场景**：
- 构建企业内部专家（如"我们公司的品牌指南专家"）
- 将顾问/导师的知识固化（上传他们的文章/书籍）
- 持续学习型专家（如"巧克力行业专家"不断吸收新知识）

**真实案例**：
> 上传 UX 设计书籍 + 设计博客 → Sage 自动提取设计原则 → 用户咨询"如何设计表单" → Sage 回答 → 发现缺少"移动端表单"知识 → 提问用户 → 更新记忆 → 下次能回答移动端问题

**技术亮点**：
- Memory Document 是 Markdown 格式，人类可读可编辑
- 支持团队级 Sage，整个团队共享专家知识

### 🔌 10. MCP Integration - 团队自定义工具

**一句话**：通过 MCP 协议接入团队自有数据源和工具，无需改代码。

**支持场景**：
- 接入公司内部数据库（如"查询我们的用户行为数据"）
- 集成第三方 API（如"调用我们的 CRM 系统"）
- 自定义分析工具（如"运行我们的市场模型"）

**两步执行机制**（createSubAgent）：
1. **工具选择**：AI 自动从团队配置的 N 个 MCP 中选择合适工具
2. **子智能体执行**：创建临时子智能体，使用选中的工具完成任务

**配置示例**：
```json
{
  "mcp": {
    "servers": {
      "company-analytics": {
        "command": "node",
        "args": ["/path/to/mcp-server.js"],
        "env": { "API_KEY": "xxx" }
      }
    }
  }
}
```

**vs. 固定工具**：
- 固定工具：所有用户共享，无法定制
- MCP Integration：每个团队配置自己的工具，数据隔离

**真实案例**：
> 数据分析团队：配置 MCP 连接 Snowflake 数据库 → 用户提问"分析上季度用户留存" → AI 自动选择 Snowflake MCP → 生成 SQL 查询 → 返回分析结果

### 🔗 11. Reference & Attachments - 研究延续

**一句话**：关联历史研究 + 上传文档，AI 基于已有洞察继续深挖。

**Reference Study（参考研究）**：
- 关联之前的研究项目 → AI 自动读取研究日志（studyLog）
- 避免重复工作，基于已有发现继续探索

**File Attachments（文件附件）**：
- 支持类型：PDF/TXT/CSV/图片（最多 10MB/文件）
- 自动处理：提取文本 → 压缩到 ≤20K tokens → 注入研究上下文
- 图片支持：用 Vision API 理解图表、原型、包装设计等

**适用场景**：
- **连续研究**：先做市场分析，再做产品验证，两者关联
- **文档分析**：上传竞品报告，AI 提取关键信息
- **原型测试**：上传产品原型图，讨论用户反应

**真实案例**：
> 第 1 次研究：气泡咖啡市场机会分析 → 输出报告 A
> 第 2 次研究：关联报告 A + 上传竞品包装图 → AI 基于市场机会做包装设计建议

**成本优化**：
- 文本压缩使用 gpt-5-mini（便宜快速）
- 只在需要时调用 Vision API
