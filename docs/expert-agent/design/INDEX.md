# 专家智能体系统 - UI 设计原型

本目录包含专家智能体系统的核心页面设计原型。所有设计均使用纯 HTML/CSS 实现，可以直接在浏览器中打开预览。

---

## 🎯 完整用户流程

### 创建专家流程
```
01. 添加知识源 → AI 自动分析 → 02. 记忆管理界面（查看 + 分析）
                                    ↓
                               (可选) 06. 补充访谈 → 回到 02（更新状态）
```

### 使用专家
```
03. 专家对话界面 - 与专家实时交互
```

### 分享专家
```
04. 专家公开页面 - Landing Page 展示
```

### 持续优化
```
02. 记忆管理界面 ⇄ 06. 补充访谈
- 随时查看知识状态
- 编辑 Memory Document
- 触发新的补充访谈
- 持续改进专家能力
```

---

## 📁 页面详细说明

### 01. [添加知识源](./01-expert-creation.html)
**场景**: 创建新专家的第一步

**功能**:
- 上传文件（PDF, .txt, Markdown, Audio, .docx）
- 添加网站链接
- 粘贴文本内容
- 知识源数量限制（最多 10 个）

**核心交互**:
- 拖拽上传文件
- 点击选择文件
- 弹窗添加 Website / Text
- 实时显示已添加的源列表
- Source limit 进度条

**设计亮点**:
- 极简设计，类似 NotebookLM
- 清晰的文件类型提示
- 动态进度条显示源数量
- 下一步按钮智能启用/禁用

**下一步**: 
- 点击"下一步"→ AI 后台处理（生成 Memory Document）→ 进入 02 页面

---

### 02. [记忆管理界面](./02-memory-management.html)
**场景**: 
- 创建专家后首次查看（显示刚生成的结果）
- 日常管理和查看专家知识状态

**功能**:
- **Tab 1: Memory Document** - 查看和编辑专家的核心记忆文档
  - Markdown 编辑器（左侧编辑，右侧实时预览）
  - 保存修改
  - 历史版本（未来功能）
  
- **Tab 2: 知识分析** - 7 维度知识完整度评估
  - 整体评分（圆形进度条）
  - 7 个维度的详细评分（基础理论、实践经验、行业洞察等）
  - 改进建议卡片
  - "开始补充访谈"按钮（智能推荐）

**核心交互**:
- 两个 Tab 页切换
- Markdown 实时预览
- 保存文档
- 一键开启补充访谈

**设计亮点**:
- 侧边栏显示专家基本信息和统计
- 分栏编辑器，实时预览
- 知识完整度可视化（圆形进度 + 维度柱状图）
- 智能推荐：评分低时突出"补充访谈"按钮

**智能化体现**:
- 首次进入时，显示 AI 刚生成的 Memory Document
- 知识分析 Tab 智能评估当前知识完整度
- 根据评分和空白智能推荐是否需要补充访谈
- 可随时回到这个页面查看最新状态

---

### 03. [专家对话界面](./03-expert-chat.html)
**场景**: 用户与专家进行实时对话咨询

**功能**:
- 实时对话交互
- 工具调用展示（Deep Research）
- 相关记忆展示
- 多模态输入（文本、语音、文件）

**核心交互**:
- 消息发送和接收
- 打字指示器
- 语音录入
- 文件附件上传
- 快速问题选择

**设计亮点**:
- 三栏布局（专家信息 / 对话区 / 上下文侧边栏）
- 消息气泡设计
- 工具调用卡片（显示 AI 使用工具的过程）
- 相关记忆实时展示（右侧栏显示调用的记忆片段）
- 欢迎消息和快速问题推荐

---

### 04. [专家公开页面](./04-expert-public-page.html)
**场景**: 专家对外分享的 Landing Page

**功能**:
- 专家主页展示
- 能力和知识领域介绍
- 用户评价展示
- 分享和嵌入功能

**核心交互**:
- 导航栏
- Hero 区域展示
- 分享按钮
- CTA 按钮（开始对话）

**设计亮点**:
- Landing Page 风格设计
- 渐变色背景
- 特性卡片网格
- 知识领域卡片
- 用户评价展示
- 响应式设计

---

### 06. [补充访谈界面](./06-supplementary-interview.html)
**场景**: 
- 创建专家后，选择进行补充访谈
- 日常优化专家时，从 02 触发补充访谈

**功能**:
- 专注的全屏访谈体验
- 基于 FocusedInterviewChat 的设计
- 语音和文本双输入支持
- 实时语音转写显示
- 倒计时提醒机制

**核心交互**:
- 欢迎页到问答的流畅过渡
- 麦克风录音（点击开始/停止）
- 键盘输入切换
- 实时转写文本显示
- 打字机效果展示问题
- 思考状态动画

**设计亮点**:
- 极简的全屏对话界面
- 专注力引导设计
- 语音录入的脉冲动画
- 问题上下文提示
- 进度指示器
- 响应式计时器
- 平滑的状态切换动画

**完成后**: 
- 访谈结束 → AI 更新 Memory Document → 返回 02 页面（显示更新后的状态）

## 🎨 设计系统

### 颜色体系

本设计使用项目的 Tailwind CSS v4 颜色系统（oklch 色彩空间）：

```css
/* 主要颜色 */
--primary: oklch(0.87 0.29 142.57)      /* 绿色 - 主要操作 */
--background: oklch(0.985 0 0)          /* 浅灰背景 */
--foreground: oklch(0.145 0 0)          /* 深灰文字 */
--border: oklch(0.922 0 0)              /* 边框颜色 */
--muted: oklch(0.556 0 0)               /* 次要文字 */

/* 辅助颜色 */
--destructive: oklch(0.577 0.245 27.325) /* 红色 - 警告/删除 */
--secondary: oklch(0.6 0.118 184.704)    /* 蓝色 - 辅助 */
--accent: oklch(0.769 0.188 70.08)       /* 橙色 - 强调 */
```

### 圆角规范

```css
--radius-sm: 6px
--radius-md: 8px
--radius-lg: 12px
--radius-xl: 16px
```

### 字体系统

```css
--font-sans: -apple-system, BlinkMacSystemFont, "Segoe UI", Roboto, "Helvetica Neue", Arial, sans-serif
```

### 间距系统

遵循 4px 基准间距:
- 4px, 8px, 12px, 16px, 20px, 24px, 32px, 40px, 48px, 80px

## 🔄 交互模式

### 1. 按钮状态
- **Normal**: 默认状态
- **Hover**: 背景色略深，轻微抬起效果
- **Active**: 按压效果
- **Disabled**: 50% 透明度，禁用光标

### 2. 表单输入
- **Focus**: 边框高亮 + 阴影
- **Error**: 红色边框 + 错误提示
- **Success**: 绿色边框 + 成功图标

### 3. 卡片交互
- **Hover**: 边框高亮 + 轻微阴影 + 向上 2px
- **Active**: 按压效果

### 4. 模态框
- **打开**: 淡入动画 + 背景遮罩
- **关闭**: 淡出动画
- **点击外部**: 关闭模态框

## 📱 响应式设计

所有页面都支持响应式设计：

### 断点
- **移动端**: < 768px
- **平板**: 768px - 1024px
- **桌面**: > 1024px

### 适配策略
- 移动端：单栏布局，堆叠展示
- 平板：2栏布局，适当调整间距
- 桌面：3栏布局，充分利用空间

## 🚀 使用方法

### 本地预览

1. **直接打开文件**
   ```bash
   # 在浏览器中打开任意 HTML 文件
   open 01-expert-creation.html
   ```

2. **使用本地服务器**
   ```bash
   # 在当前目录启动服务器
   python -m http.server 8000
   # 或使用 Node.js
   npx serve .
   ```

3. **在浏览器访问**
   ```
   http://localhost:8000/01-expert-creation.html
   ```

### 开发工具

推荐使用浏览器开发者工具：
- **Chrome DevTools**: 查看元素、调试样式
- **响应式模式**: 测试不同屏幕尺寸
- **Console**: 查看交互逻辑

   ```

   │   ├── KnowledgeAnalysis.tsx
   │   └── index.tsx
   ├── ExpertChat/
   │   ├── ChatHeader.tsx
   │   ├── MessageList.tsx
   │   ├── MessageInput.tsx
   │   └── ContextSidebar.tsx
   └── SupplementaryInterview/
       ├── InterviewWelcome.tsx
       ├── InterviewQuestion.tsx
       └── index.tsx
   ```

2. **使用项目规范**
   - 遵循 `CLAUDE.md` 中的代码规范

## 🔮 AI 智能化体现

### 创建流程的智能化
1. **自动分析**: 上传源后，AI 自动提取和结构化知识
2. **智能评估**: 自动评估知识完整度，7 维度打分
3. **智能推荐**: 根据评分智能推荐是否需要补充访谈
4. **问题生成**: 自动生成针对性的补充问题

### 使用过程的智能化
1. **工具调用**: 对话中自动调用 Deep Research 等工具
2. **记忆检索**: 实时检索相关记忆片段
3. **上下文展示**: 显示 AI 使用的记忆和工具
4. **持续学习**: 从对话中发现新的知识空白

### 优化流程的智能化
1. **动态分析**: 持续更新知识完整度评分
2. **主动建议**: 主动提示需要改进的领域
3. **访谈优化**: 智能生成补充问题
4. **自动更新**: 访谈后自动更新 Memory Document
