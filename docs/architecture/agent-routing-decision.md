# Agent Routing Decision Matrix

## 概述

Plan Mode通过分析用户brief，自动判断使用哪个执行agent。本文档定义清晰的决策规则。

## 三种执行Agent

### 1. Fast Insight Agent (kind: `fastInsight`)

**核心定位**：快速内容生成 - 播客优先

**使用场景**：
- 时事热点分析
- 商业趋势解读
- 快速生成可听的内容
- 内容营销素材

**关键特征**：
- ⚡ **速度优先**：maxSteps=10，快速完成
- 🎙️ **播客为主**：主要产出是播客音频
- 📊 **报告可选**：可生成结构化报告
- 🔍 **浅层研究**：webSearch限1次 + deepResearch

**Brief触发词**：
```
强触发：
- "生成播客"
- "快速了解X趋势"
- "制作音频内容"
- "热点分析"
- "最新动态"

弱触发：
- "介绍X话题"
- "解读X现象"
- "分享X观点"
```

**Brief模式**：
```
模式1: 直接要求播客
"帮我生成一期播客，聊聊最近AI Agent的发展趋势"

模式2: 快速洞察 + 内容产出
"快速分析一下露营经济的最新动态，制作成可听的内容"

模式3: 时事解读
"解读一下最近火热的小红书电商模式"
```

**典型工具流程**：
```
webSearch(1次) → planPodcast → deepResearch → generatePodcast → (generateReport可选)
```

---

### 2. Product R&D Agent (kind: `productRnD`)

**核心定位**：产品创新机会发现 - 社交趋势驱动

**使用场景**：
- 产品功能创新
- 包装/设计创新
- 营销策略创新
- 目标用户拓展
- 使用场景创新

**关键特征**：
- 🎨 **创新导向**：寻找创新机会和灵感点
- 📱 **社交驱动**：使用scoutSocialTrends观察社交媒体
- 🔗 **灵感结合**：原产品 + 灵感点 → 创新机会
- 🎯 **商业价值**：关注可落地的商业机会

**Brief触发词**：
```
强触发：
- "产品创新"
- "新产品机会"
- "功能创新"
- "包装设计创新"
- "营销创新"
- "使用场景创新"
- "目标用户拓展"
- "寻找灵感"
- "创意发散"

弱触发：
- "优化X产品"
- "提升X体验"
- "X的新方向"
```

**Brief模式**：
```
模式1: 明确创新类型
"为星巴克咖啡找到使用场景创新的机会，看看其他品牌有什么可以借鉴"

模式2: 产品+方向
"我们的运动鞋品牌想要拓展年轻用户，帮我从社交媒体上找找灵感"

模式3: 开放式创新
"帮我为冰淇淋产品寻找口味和包装上的创新机会"
```

**典型工具流程**：
```
理解需求 → scoutSocialTrends(观察社交趋势) → 灵感点分析 → 创新机会评估 → generateReport
```

**创新维度**（引导用户思考）：
- 口味风味创新
- 包装设计创新
- 功能创新
- 使用场景创新
- 目标用户创新
- 营销策略创新

---

### 3. Study Agent (kind: `testing | insights | creation | planning | misc`)

**核心定位**：深度商业研究 - AI人设模拟

**使用场景**：
- 用户行为研究
- 决策机制分析
- 产品测试与比较
- 策略规划与框架设计
- 创意方案生成与验证

**关键特征**：
- 🧠 **深度研究**：构建AI人设，模拟真实用户
- 💬 **访谈/讨论**：interview或discussion获取洞察
- 📋 **框架支持**：JTBD/KANO/STP/用户旅程等
- 📊 **结构化产出**：深度报告，为主观世界建模

**5种研究子类型**：

#### 3.1 Testing (测试型)

**Brief触发词**：
```
- "比较X和Y"
- "测试X效果"
- "A/B测试"
- "哪个方案更好"
- "验证假设"
- "用户更喜欢"
```

**Brief示例**：
```
"对比星巴克和Luckin Coffee在一线城市年轻人心中的品牌形象"
"测试三种包装设计，看看用户更喜欢哪个"
```

#### 3.2 Insights (洞察型)

**Brief触发词**：
```
- "了解用户对X的看法"
- "发现问题"
- "用户痛点"
- "行为分析"
- "需求洞察"
- "为什么用户"
```

**Brief示例**：
```
"了解18-28岁年轻人对日常咖啡消费的决策因素"
"发现用户在使用健身App时的核心痛点"
```

#### 3.3 Creation (创意型)

**Brief触发词**：
```
- "设计新功能"
- "产生创意"
- "头脑风暴"
- "创意方案"
- "如何设计"
```

**Brief示例**：
```
"为我们的社交App设计一个新的社区功能"
"帮我设计针对银发族的智能手机新功能"
```

#### 3.4 Planning (规划型)

**Brief触发词**：
```
- "制定策略"
- "规划方案"
- "实施计划"
- "战略设计"
- "路线图"
```

**Brief示例**：
```
"为我们的新产品制定上市策略"
"规划一套完整的用户增长方案"
```

#### 3.5 Misc (综合型)

**Brief触发词**：
```
其他不符合上述类型的研究需求
```

**典型工具流程**：
```
webSearch(背景调研) → planStudy(规划) → buildPersona(构建人设) →
interview/discussion(访谈/讨论) → generateReport(生成报告)
```

**研究框架**：
- JTBD（Jobs To Be Done）- 用户决策/需求
- KANO - 功能优先级
- STP - 市场定位
- GE-McKinsey - 业务机会评估
- 用户旅程地图 - 使用流程分析
- 情感地图 - 情感体验分析

## 决策树

```
用户Brief
  │
  ├─ 包含"播客"/"快速"/"热点"? ───Yes──→ Fast Insight Agent
  │                             │
  │                             No
  │                             ↓
  ├─ 包含"创新"/"灵感"/"社交趋势"? ──Yes──→ Product R&D Agent
  │                                 │
  │                                 No
  │                                 ↓
  └─────────────────────────────→ Study Agent
                                    │
                                    ├─ 比较/测试? → testing
                                    ├─ 了解/发现? → insights
                                    ├─ 设计/创意? → creation
                                    ├─ 制定/规划? → planning
                                    └─ 其他 → misc
```

## 判断优先级

### Level 1: Agent选择（三选一）

1. **Fast Insight优先** - 如果明确要求播客或快速内容
2. **Product R&D其次** - 如果是产品创新相关
3. **Study兜底** - 其他所有深度研究

### Level 2: Study子类型判断（五选一）

只有选择了Study Agent后，才判断具体的kind：

1. **Testing** - 明确的比较、测试、验证诉求
2. **Insights** - 了解、发现、分析诉求
3. **Creation** - 设计、创意、头脑风暴诉求
4. **Planning** - 策略、规划、实施计划诉求
5. **Misc** - 无法归类的综合研究

## Brief质量标准

### 好的Brief特征

✅ **明确触发词**：包含关键动词（比较/了解/设计/生成播客）
✅ **清晰对象**：明确研究谁（目标用户群）
✅ **具体场景**：说明在什么情况下（使用场景）
✅ **期望产出**：暗示想要什么结果

### Brief优化示例

❌ **模糊Brief**：
```
"研究一下咖啡市场"
→ 无法判断是要播客、创新还是深度研究
```

✅ **优化后**：
```
选项1（Fast Insight）：
"快速分析咖啡市场最新趋势，生成一期播客"

选项2（Product R&D）：
"为我们的咖啡品牌寻找使用场景创新机会，从社交媒体观察年轻人的新消费习惯"

选项3（Study - Insights）：
"深入了解一线城市25-35岁白领对精品咖啡的消费决策因素，分析他们选择咖啡店的核心考量"
```

## Plan Mode自动判断策略

### 策略1: 关键词优先匹配

```typescript
// Fast Insight触发词权重最高
const fastInsightKeywords = ["播客", "podcast", "快速", "热点", "音频", "听"];

// Product R&D次之
const productRnDKeywords = ["创新", "灵感", "新产品", "社交趋势", "功能创新", "包装设计"];

// Study兜底（默认）
```

### 策略2: 场景模式识别

```typescript
// 识别Brief模式
if (brief.includes("生成播客") || brief.includes("制作音频")) {
  return "fastInsight";
}

if (brief.match(/创新|灵感|新.*机会|拓展.*用户/)) {
  return "productRnD";
}

// 否则根据动词判断Study子类型
if (brief.match(/比较|测试|A\/B|哪个更/)) {
  return "testing";
}
// ... 其他子类型判断
```

### 策略3: 上下文补充

Plan Mode应该：
1. 先理解用户核心诉求
2. 必要时通过requestInteraction澄清
3. 根据澄清结果自动判断kind
4. 在makeStudyPlan中展示判断结果让用户确认

## 边界情况处理

### 情况1: Brief包含多个agent特征

```
例："从社交媒体寻找咖啡产品创新机会，并生成播客"
```

**处理策略**：
- 识别主要目标（创新机会）→ Product R&D Agent
- 次要目标（播客）可在报告生成后单独处理
- 或者建议用户分成两个研究

### 情况2: Brief过于简单

```
例："研究一下年轻人"
```

**处理策略**：
- Plan Mode多轮澄清：
  - "你想了解年轻人的什么方面？"
  - "是想快速了解某个趋势，还是深入研究某个决策？"
  - "最终需要什么产出？播客、报告还是策略方案？"

### 情况3: Brief不包含明确触发词

```
例："帮我分析一下露营市场"
```

**处理策略**：
- 默认选择Study Agent（insights）
- Plan Mode在澄清过程中探测真实意图：
  - "你是想快速了解露营市场的最新动态（播客）"
  - "还是想为你的产品寻找创新机会（产品创新）"
  - "还是深入研究消费者的露营决策（深度研究）？"

```markdown
**优先级2: Product R&D Agent (kind=productRnD)**

• planning: 制定("制定策略")、规划("规划方案")、战略("实施计划")
• misc: 无法归类的综合研究

判断要点：
- 是否需要**构建AI人设**进行深度访谈或讨论？
- 是否需要使用**分析框架**（JTBD/KANO等）？
- 是否期望**为主观决策建模**？
```

## 总结

**三种agent的本质区别**：

| 维度 | Fast Insight | Product R&D | Study |
|------|-------------|-------------|-------|
| **核心价值** | 快速内容生成 | 创新机会发现 | 深度行为建模 |
| **主要产出** | 播客为主 | 创新报告 | 深度研究报告 |
| **研究深度** | 浅层（快速） | 中层（灵感+分析） | 深层（AI人设模拟） |
| **时间投入** | 最短 | 中等 | 最长 |
| **数据来源** | 网络搜索+深度研究 | 社交媒体观察 | AI人设访谈/讨论 |
| **适用场景** | 内容营销、热点追踪 | 产品创新、R&D | 用户研究、策略规划 |

**关键判断原则**：
1. **有"播客"诉求** → Fast Insight
2. **有"创新"诉求 + 产品上下文** → Product R&D
3. **需要深度洞察 + 行为分析** → Study

用户不需要理解这些区别，Plan Mode会自动判断。但Brief质量越高，判断越准确。
