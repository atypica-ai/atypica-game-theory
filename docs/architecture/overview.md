# 系统架构概览

本文档介绍 atypica.AI 的整体架构设计和技术选型。

## 架构总览

atypica.AI 是一个基于 AI 多智能体的商业研究平台，采用现代化的全栈架构：

```
┌─────────────────────────────────────────────────────────────┐
│                        用户界面层                              │
│   Next.js 15 App Router + React 19 + Tailwind CSS v4        │
└─────────────────────────────────────────────────────────────┘
                              ↓
┌─────────────────────────────────────────────────────────────┐
│                        应用服务层                              │
│   • 研究助手 (Study)    • Sage 专家智能体                      │
│   • 用户发现 (Scout)    • 访谈模拟 (Interview)                │
│   • AI 人设库 (Persona) • 播客生成 (Podcast)                 │
│   • 深度研究 (MCP)      • 管理后台 (Admin)                    │
└─────────────────────────────────────────────────────────────┘
                              ↓
┌─────────────────────────────────────────────────────────────┐
│                        AI 服务层                               │
│   Vercel AI SDK v5 + 多提供商支持                             │
│   • Prompt 管理     • 工具定义     • 流式处理                 │
└─────────────────────────────────────────────────────────────┘
                              ↓
┌─────────────────────────────────────────────────────────────┐
│                        外部服务层                              │
│   • LLM: Claude, GPT, Gemini, Bedrock, DeepSeek            │
│   • 搜索: Tavily, 社交媒体 APIs                              │
│   • TTS: Volcano Engine                                     │
│   • 存储: AWS S3                                            │
└─────────────────────────────────────────────────────────────┘
                              ↓
┌─────────────────────────────────────────────────────────────┐
│                        数据持久层                              │
│   PostgreSQL 15 + Prisma ORM + pgvector 扩展                │
└─────────────────────────────────────────────────────────────┘
```

## 核心设计原则

### 1. 模块化架构

每个功能模块独立封装，通过清晰的接口交互：

```
src/
├── app/                    # 功能模块（按路由分组）
├── ai/                     # AI 服务层（工具和提示词）
├── components/             # 可复用 UI 组件
├── hooks/                  # 可复用 React Hooks
├── lib/                    # 工具函数和通用配置
└── prisma/                 # 数据层
```

### 2. 类型安全优先

- 全面使用 TypeScript
- Prisma 自动生成数据库类型
- AI SDK 提供完整的类型定义
- 严格的 ESLint 配置（零警告容忍）

### 3. 流式优先

- AI 响应使用流式传输（SSE）
- 平滑的中文字符流式渲染
- 实时的 UI 状态更新
- 渐进式加载体验

### 4. 多租户支持

- 个人用户和团队用户
- 团队级别的配置管理
- 独立的订阅和计费
- API Key 权限隔离

## 部署架构

### 生产环境

```
┌─────────────┐
│   用户请求    │
└──────┬──────┘
       ↓
┌─────────────┐
│  CDN/Edge   │  (Vercel Edge Network)
└──────┬──────┘
       ↓
┌─────────────────────┐
│  Next.js App        │
│  (Serverless)       │
│  - API Routes       │
│  - Server Actions   │
│  - Streaming        │
└──────┬──────────────┘
       ↓
┌──────────────┬──────────────┐
│   Database   │  External    │
│  PostgreSQL  │  Services    │
│  (AWS RDS)   │  (LLM/S3)    │
└──────────────┴──────────────┘
```

### Docker 部署

使用多阶段构建优化镜像大小：

```dockerfile
FROM node:22-alpine AS base
# deps stage - 安装依赖
# builder stage - 构建应用
# runner stage - 运行生产服务器
```

**特点**:

- 基于 Alpine Linux（轻量级）
- Standalone 输出模式
- 自动优化依赖
- 健康检查支持

### 区域部署

支持多区域部署：

- **全球区域** (Global) - 美东、欧洲
- **中国大陆** (Mainland) - 北京区域
- 自动路由和区域选择
- CDN 加速和缓存优化

## 数据流

### 典型的研究流程

```
1. 用户发起研究请求
   ↓
2. Study Agent 解析需求
   ↓
3. 调用专业工具:
   - Scout: 用户发现
   - Interview: 访谈模拟
   - Persona: 人设对话
   - Report: 生成报告
   ↓
4. 结果持久化到数据库
   ↓
5. 流式返回给用户界面
```

### AI 工具调用流程

```
用户输入 → AI SDK
          ↓
      工具选择 (Tool Selection)
          ↓
      工具执行 (Tool Execution)
      - 数据库查询
      - 外部 API 调用
      - 业务逻辑处理
          ↓
      结果格式化
          ↓
      流式返回用户
```

## 性能优化

### 前端优化

- **代码分割**: 按路由自动分割
- **懒加载**: 组件按需加载
- **图片优化**: Next.js Image 组件
- **字体优化**: 自托管字体
- **缓存策略**: SWR + React Query

### 后端优化

- **数据库连接池**: Prisma Accelerate
- **读写分离**: Read Replicas
- **查询优化**: 索引 + 复合查询
- **向量搜索**: pgvector 优化
- **API 缓存**: unstable_cache

### AI 优化

- **Prompt 缓存**: 复用系统提示词
- **流式传输**: 降低首字节时间
- **并行请求**: 多工具并行调用
- **Token 优化**: 控制上下文长度

## 安全性

### 认证和授权

- **用户认证**: NextAuth.js (Session + JWT)
- **API 认证**: Bearer Token (atypica_xxx)
- **权限控制**: 基于用户角色和团队
- **CORS 配置**: 跨域请求管理

### 数据安全

- **加密传输**: HTTPS/TLS
- **敏感数据**: 环境变量加密
- **SQL 注入防护**: Prisma 参数化查询
- **XSS 防护**: React 自动转义

### API 安全

- **速率限制**: 防止滥用
- **Token 消耗追踪**: 配额管理
- **审计日志**: 操作记录
- **健康检查**: 系统监控
